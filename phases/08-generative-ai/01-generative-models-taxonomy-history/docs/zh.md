# 生成模型——分类与历史

> 每个图像模型、文本模型、视频模型和 3D 模型都属于五个桶之一。选错桶，你会和数学搏斗好几周。选对桶，这个领域过去十二年的进展就会在你脑海中整齐堆叠。

**类型：** 学习
**语言：** Python
**前置条件：** Phase 2（ML 基础）、Phase 3（深度学习核心）、Phase 7 · 14（Transformer）
**时长：** 约 45 分钟

## 问题背景

生成模型做一件事：给定从某个未知分布 `p_data(x)` 中抽取的训练样本，输出看起来来自同一分布的新样本。面孔、句子、MIDI 文件、蛋白质结构——如果眯起眼睛看，都是同一个问题。

难点在于 `p_data` 生活在一个拥有数百万维度的空间中（512x512 的 RGB 图像约有 78.6 万维），样本位于该空间内的一个薄流形上，而你可能只有 1000 万个例子。暴力求解密度是没有希望的。每个生成模型都是一种妥协，用一个难题换一个稍微容易点的问题。

过去十二年，五个模型族存活了下来。了解每个族做出了什么妥协，就能明白它为什么在某些任务上获胜，在另一些任务上崩溃。

## 核心概念

![五类生成模型——按建模对象分类](../assets/taxonomy.svg)

**1. 显式密度，可处理。** 将 `log p(x)` 写成可以实际评估的求和。自回归模型（PixelCNN、WaveNet、GPT）将 `p(x) = ∏ p(x_i | x_<i)` 分解。归一化流（RealNVP、Glow）将 `p(x)` 构建为简单基分布的可逆变换。优点：精确似然，干净的训练损失。缺点：自回归推理是顺序的（长序列慢），流需要可逆架构（架构限制多）。

**2. 显式密度，近似。** 从下方约束 `log p(x)`（ELBO）并优化该约束。VAE（Kingma 2013）使用带变分后验的编码器-解码器。扩散模型（DDPM，Ho 2020）训练一个去噪器，隐式地优化加权 ELBO。扩散是 2026 年主导图像、视频和 3D 的骨干。

**3. 隐式密度。** 完全跳过密度；学习一个产生样本的生成器 `G(z)` 和一个分辨真假的判别器 `D(x)`。GAN（Goodfellow 2014）。推理快（一次前向传播），但训练时出了名地不稳定。StyleGAN 1/2/3 在 2026 年仍然是固定域真实感（面孔、卧室）的最先进方案。

**4. 基于分数 / 连续时间。** 直接学习对数密度的梯度 `∇_x log p(x)`（分数）。Song & Ermon（2019）表明分数匹配将扩散推广到 SDE。流匹配（Lipman 2023）是 2024-2026 年的热门：无模拟训练、更直的路径、比 DDPM 快 4-10 倍的采样。Stable Diffusion 3、Flux、AudioCraft 2 都使用流匹配。

**5. 离散码上的基于 token 的自回归。** 用 VQ-VAE 或残差量化器将高维数据压缩成短的离散 token 序列，然后用 Transformer 对 token 序列建模。Parti、MuseNet、AudioLM、VALL-E、Sora 的图块分词器都使用这种方式。这是桶 1 加上一个可学习的分词器。

## 五问分类法

当一篇新的生成模型论文发布时，在阅读方法部分之前先回答这五个问题。

1. **建模的是什么？** 像素、潜在表示、离散 token、3D 高斯、网格、波形？
2. **密度是显式还是隐式的？** 他们有没有写出 `log p(x)`？
3. **采样：一次性还是迭代？** 迭代意味着推理更慢；一次性通常意味着对抗性或蒸馏。
4. **条件：无条件、类别、文本、图像、姿态？** 这决定了损失和架构脚手架。
5. **评估：FID、CLIP 分数、IS、人类偏好、任务准确率？** 每种都有已知的失效模式（见第 14 课）。

在本阶段的每节课中你都会重新回答这五个问题。到最后，它们会成为你的反射。

## 动手实现

本课的代码是一个轻量级可视化：用三种玩具方法（核密度估计、离散直方图和最近邻"GAN 风格"生成器）从样本拟合一维高斯混合，让你在一屏幕内看到显式与隐式密度之间的差异。

运行 `code/main.py`。它从双模高斯混合中抽取 2000 个样本，然后打印：

```
显式密度（直方图）：p(x ∈ [-0.5, 0.5]) ≈ 0.38
近似密度（KDE）：   p(x ∈ [-0.5, 0.5]) ≈ 0.41
隐式（最近邻生成器）：打印 20 个新样本，无 p(x)
```

注意：前两种让你问"这个点有多可能？"第三种不能。这就是*显式与隐式*的区别，它在每节后续课中都会很重要。

## 生产使用

2026 年，哪种任务选哪个族？

| 任务 | 最佳族 | 原因 |
|------|--------|------|
| 窄域真实感面孔 | StyleGAN 2/3 | 仍然最锐利，推理最快。 |
| 通用文本到图像 | 潜在扩散 + 流匹配 | SD3、Flux.1、DALL-E 3。 |
| 快速文本到图像 | 修正流 + 蒸馏 | SDXL-Turbo、SD3-Turbo、LCM。 |
| 文本到视频 | 扩散 Transformer + 流匹配 | Sora、Veo 2、Kling。 |
| 语音 + 音乐 | 基于 token 的 AR（AudioLM、VALL-E、MusicGen）或流匹配（AudioCraft 2） | 离散 token 扩展成本低。 |
| 3D 场景 | 高斯 Splatting 拟合，扩散先验 | 3D-GS 用于重建，扩散用于新视角。 |
| 密度估计（无采样） | 流 | 唯一具有精确 `log p(x)` 的族。 |
| 仿真 / 物理 | 流匹配，分数 SDE | 直线路径，平滑向量场。 |

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| 生成模型（Generative model） | "它生成新东西" | 学习 `p_data(x)` 的采样器，可选地暴露 `log p(x)`。 |
| 显式密度（Explicit density） | "可以评估它" | 模型提供封闭形式或可处理的 `log p(x)`。 |
| 隐式密度（Implicit density） | "GAN 风格" | 只有采样器——无法评估给定点的 `p(x)`。 |
| ELBO | "证据下界" | `log p(x)` 的可处理下界；VAE 和扩散模型对其进行优化。 |
| 分数（Score） | "对数密度的梯度" | `∇_x log p(x)`；扩散和 SDE 模型学习这个场。 |
| 流形假设（Manifold hypothesis） | "数据生活在一个曲面上" | 高维数据集中在低维流形上；这就是降维有效的原因。 |
| 自回归（Autoregressive） | "预测下一块" | 将联合分布分解为条件分布的乘积。 |
| 潜在（Latent） | "压缩码" | 解码器可以从中重建输入的低维表示。 |

## 生产注意：五个族，五种推理形态

每个族对应不同的推理服务成本曲线。生产推理文献将 LLM 推理分为预填充 + 解码；同样的分解也适用于这里：

- **自回归（桶 1 和 5）。** 顺序解码主导延迟；KV 缓存、连续批处理和投机解码直接适用。
- **VAE / 扩散 / 流匹配（桶 2 和 4）。** 没有 LLM 意义上的解码。成本 = `步数 × 每步成本`，而 `每步成本` 是以全潜在分辨率进行的 Transformer 或 U-Net 前向传播。生产旋钮是步数（DDIM / DPM-Solver / 蒸馏）、批大小和精度（bf16 / fp8 / int4）。
- **GAN（桶 3）。** 一次前向传播。没有调度，没有 KV 缓存。TTFT ≈ 总延迟。这就是 StyleGAN 在窄域 UX 上仍然获胜的原因。

当你在论文摘要中看到"比扩散更快"，将其翻译为"更少步数 × 相同步成本"或"相同步数 × 更便宜的步成本"。其他都是营销。
