# 视频生成

> 图像是一个 2 维张量。视频是一个 3 维的。理论相同；计算量难 10-100 倍。OpenAI 的 Sora（2024 年 2 月）证明了这是可能的。到 2026 年，Veo 2、Kling 1.5、Runway Gen-3、Pika 2.0 和 WAN 2.2 已能从文本生成 1080p 生产级视频——开放权重栈（CogVideoX、HunyuanVideo、Mochi-1、WAN 2.2）落后约 12 个月。

**类型：** 构建
**语言：** Python
**前置条件：** Phase 8 · 07（潜在扩散）、Phase 7 · 09（ViT）、Phase 8 · 06（DDPM）
**时长：** 约 45 分钟

## 问题背景

24fps 下 10 秒的 1080p 视频是 240 帧的 1920×1080×3 像素。每个片段约 1.5 GB 的原始数据。像素空间扩散不可行。你需要：

1. **时空压缩。** 将视频而非帧编码为时空图块序列的 VAE。
2. **时间连贯性。** 帧需要在数秒内共享内容、光照和对象身份。网络必须建模运动。
3. **计算预算。** 相同模型大小下，视频训练比图像贵 10-100 倍。
4. **条件化。** 文本、图像（第一帧）、音频或另一个视频。大多数生产模型接受全部四种。

解决这个问题的架构是应用于时空图块的**扩散 Transformer（DiT）**，在大量（提示词、标注、视频）数据集上训练。与第 06 课相同的扩散损失。

## 核心概念

![视频扩散：图块化、DiT、解码](../assets/video-generation.svg)

### 图块化

用 3D VAE 编码视频（学习到的时空压缩）。潜在表示形状为 `[T_latent, H_latent, W_latent, C_latent]`。分割为大小为 `[t_p, h_p, w_p]` 的图块。对 Sora 风格的模型，`t_p = 1`（每帧图块）或 `t_p = 2`（每两帧）。10 秒 1080p 视频压缩为约 20,000-100,000 个图块。

### 时空 DiT

Transformer 处理图块的扁平序列。每个图块有 3D 位置嵌入（时间 + y + x）。注意力通常是分解的：

- **空间注意力**：在每帧的图块内。
- **时间注意力**：在相同空间位置的跨帧之间。
- **完整 3D 注意力**：成本高 16-100 倍；仅在低分辨率或研究中使用。

### 文本条件化

与大型文本编码器进行交叉注意力（Sora 和 CogVideoX-5B 都使用 T5-XXL）。长提示词很重要——Sora 的训练集有 GPT 生成的密集重标注，每个片段平均 200 个 token。

### 训练

标准扩散损失（ε 或 v 预测）在时空潜在表示上。数据：网络视频 + 约 1 亿个精选片段 + 合成文本标注。计算量：即使是小型研究运行也需要 10,000+ GPU 小时；Sora 规模是 100,000+。

## 动手实现

`code/main.py` 模拟了核心时空 DiT 思想：对小型合成视频进行图块化，添加每图块位置嵌入，并用 Transformer 风格的图块注意力对整个序列去噪。纯 Python，无 numpy。我们展示即使在一维中，当相邻帧图块共享去噪器和位置嵌入时，时间连贯性也会涌现。

### 步骤一：对合成一维"视频"进行图块化

```python
def make_video(T_frames=8, rng=None):
    # "视频"是遵循平滑轨迹的一维值序列
    base = rng.gauss(0, 1)
    return [base + 0.3 * t + rng.gauss(0, 0.1) for t in range(T_frames)]
```

### 步骤二：每帧位置嵌入

```python
def pos_embed(t, dim):
    return sinusoidal(t, dim)
```

### 步骤三：去噪器看到整个序列

与独立对每帧去噪不同，我们的小型网络拼接所有帧值 + 它们的位置嵌入，并联合预测所有帧的噪声。

### 步骤四：时间连贯性测试

训练后，采样一个视频。测量帧间差值。如果模型学到了时间结构，差值应该保持比独立采样每帧更小。

## 常见陷阱

- **独立逐帧采样 = 闪烁。** 如果对每帧单独运行图像扩散，由于每帧的噪声是独立的，输出会闪烁。视频扩散通过注意力或共享噪声耦合帧来解决这个问题。
- **朴素 3D 注意力 = 内存溢出。** 对 10 秒 1080p 潜在表示进行完整 3D 注意力需要数千亿次操作。分解为空间 + 时间注意力。
- **数据标注比规模更重要。** Sora 对先前工作的主要升级是在约 10 倍更详细的标注上训练（GPT-4 重新标注了片段）。OpenAI 的技术报告明确说明了这一点。
- **第一帧条件化。** 大多数生产模型也接受图像作为第一帧。这是"图像到视频"模式；训练包含此变体。
- **物理漂移。** 长片段（>10s）会积累微妙的不一致性。滑动窗口生成 + 关键帧锚定有帮助。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| 视频 VAE（Video VAE） | "3D VAE" | 将 `(T, H, W, C)` 压缩为时空潜在表示的编码器。 |
| 图块（Patches） | "那些 token" | 潜在表示的固定大小 3D 块；DiT 的输入。 |
| 分解注意力（Factorized attention） | "空间 + 时间" | 先在空间上运行注意力，再在时间上；跳过完整 3D 注意力。 |
| 图像到视频（I2V） | "让这张照片动起来" | 模型接受图像 + 文本，输出从该图像开始的视频。 |
| 关键帧条件化（Keyframe conditioning） | "锚定帧" | 固定特定帧以控制视频弧线。 |
| 运动刷（Motion brush） | "方向提示" | 用户在图像上绘制运动向量的 UI 输入。 |
| 重标注（Re-captioning） | "密集标注" | 使用 LLM 用详细提示词重新标记训练片段。 |
| 闪烁（Flicker） | "时间伪影" | 帧间不一致；通过耦合去噪修复。 |

## 生产注意：视频潜在表示是内存带宽问题

24fps 下 10 秒 1080p 片段是 240 帧 × 1920 × 1080 × 3 ≈ 1.5 GB 原始像素。经过 4× 视频 VAE 压缩（`2 × 空间 × 2 × 时间`）后，每个请求的潜在表示约为 100 MB。以批大小 1 通过时空 DiT 运行 30 步，每步通过 HBM 移动约 3 GB——瓶颈是内存带宽，而非 FLOP。

三个生产旋钮，均来自生产推理文献：

- **在 DiT 上进行张量并行（TP）。** 文本到视频模型通常 ≥10B 参数。TP=4 跨 4 块 H100 是标准；405B 级别模型使用 PP=2 × TP=2。每步延迟随 TP 大致线性下降，直到全归约墙。
- **帧批处理 = 连续批处理。** 生成时，视频在概念上是一批通过注意力连接的帧。连续批处理（飞行中调度）适用：在返回帧 `t-1` 的同时开始渲染帧 `t+1`（如果模型架构允许滑动窗口生成）。
- **片段级预填充缓存。** 对于图像到视频，第一帧条件化类似于 LLM 的提示词预填充：计算一次，在时间解码器传播中复用。这实际上是视频的 KV 缓存。
