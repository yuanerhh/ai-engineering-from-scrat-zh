# 视觉自回归建模（VAR）：下一尺度预测

> 扩散模型在时间上迭代采样（去噪步骤）。VAR 在尺度上迭代采样——它预测一个 1×1 的 token，然后是 2×2，然后是 4×4，一直到最终分辨率，每个尺度以前一个为条件。2024 年的论文表明，VAR 在图像生成方面遵循 GPT 风格的缩放定律，并在相同计算预算下胜过 DiT。本课构建核心机制。

**类型：** 构建
**语言：** Python（使用 PyTorch）
**前置条件：** Phase 7 第 03 课（多头注意力）、Phase 8 第 06 课（DDPM）
**时长：** 约 90 分钟

## 问题背景

自回归生成主导语言建模，因为它可预测地扩展：更多计算、更多参数、更低困惑度、更好的输出。2024 年之前，图像生成有两个主要的自回归尝试：PixelRNN/PixelCNN（逐像素）和 DALL-E 1 / Parti / MuseGAN（在 VQ-VAE 码上逐 token）。

两者都存在生成顺序问题。像素和 token 排列在 2D 网格中，但自回归模型必须以 1D 光栅顺序访问它们。早期的角落像素无法知道图像最终会变成什么样。生成质量的提升不如 GPT 在文本上那么顺畅，在相同计算量下也从未达到扩散模型的质量。

VAR 通过改变被生成的内容来修正生成顺序问题。与其在空间中逐一预测图像 token，VAR 以递增的分辨率预测整张图像。步骤 1：预测 1×1 的 token（图像的整体"摘要"）。步骤 2：预测 2×2 的 token 网格（更粗的特征）。步骤 3：预测 4×4 网格。步骤 K：预测最终的 (H/8)×(W/8) 网格。

每个尺度都关注所有前一个尺度（在"尺度顺序"上是因果的），并在其自身尺度内并行。顺序问题消失了：尺度 k 的整张图像在一次 Transformer 传播中生成。

## 核心概念

### VQ-VAE 多尺度分词器

VAR 需要一个**多尺度离散分词器**。对于图像 x，它生成一系列分辨率递增的 token 网格：

```
x -> 编码器 -> 潜在表示 f
f -> 在 1×1 分词: 形状为 (1, 1) 的 token 网格 z_1
f -> 在 2×2 分词: 形状为 (2, 2) 的 token 网格 z_2
...
f -> 在 (H/p)×(W/p) 分词: 形状为 (H/p, W/p) 的 token 网格 z_K
```

每个 z_k 使用相同的码本（典型大小 4096-16384）。每个尺度的分词不是独立的——它被训练为使每个尺度的残差之和重建 f：

```
f ≈ upsample(embed(z_1), target_size) + ... + upsample(embed(z_K), target_size)
```

这是**残差向量量化（Residual VQ）**的变体。尺度 k 捕获尺度 1..k-1 遗漏的内容。解码器取所有尺度嵌入的总和并生成图像。

多尺度 VQ 分词器只训练一次（类似 VQGAN），然后冻结。所有生成工作由其上的自回归模型完成。

### 下一尺度预测

生成模型是一个 Transformer，它看到所有前一个尺度的 token，并预测下一个尺度的 token。

输入序列结构：
```
[START, z_1 的 token, z_2 的 token, z_3 的 token, ..., z_K 的 token]
```

位置嵌入编码尺度索引和尺度内的空间位置。注意力在尺度顺序上是因果的：尺度 k、位置 (i, j) 的 token 可以关注尺度 1..k 的所有 token，以及尺度 k 本身中在使用的任何尺度内顺序中更早的 token（VAR 使用固定位置注意力，无尺度内因果性——尺度内的所有位置并行预测）。

训练损失：在每个尺度 k，给定所有前一尺度的 token 预测 token z_k。离散 VQ 码上的交叉熵损失。与 GPT 结构相同，只是"序列"现在是尺度结构化的。

### 生成

推理时：
```
生成 z_1 = 从 p(z_1) 采样                    # 1 个 token
生成 z_2 = 从 p(z_2 | z_1) 采样              # 并行 4 个 token
生成 z_3 = 从 p(z_3 | z_1, z_2) 采样         # 并行 16 个 token
...
解码: f = 尺度 1..K 的嵌入-上采样之和
图像 = VAE_解码器(f)
```

对于 K = 10 个尺度，生成需要 10 次 Transformer 前向传播。每次传播并行生成其整个尺度——尺度内无逐 token 自回归。对于 256×256 图像，这大约是 10 次传播，而 DiT 需要 28-50 次。

### 为什么下一尺度比下一 token 更好

三个结构性优势：
1. **粗到细与自然图像统计对齐。** 人类视觉感知和图像数据集都表现出尺度依赖的规律：低频结构稳定可预测；高频细节以低频内容为条件。下一尺度预测利用了这一点。
2. **尺度内并行生成。** 与 GPT 风格的 token 自回归不同，VAR 在一步中生成尺度上的所有 token。有效生成长度是对数级而非线性级。
3. **无生成顺序偏差。** 尺度 k 的 token 看到所有尺度 k-1 的内容；不存在迫使早期 token 在获得后期上下文之前做出决定的"左侧"或"上方"偏差。

### 缩放定律

Tian 等证明，VAR 在 ImageNet 上的 FID 遵循幂律缩放曲线——就像 GPT 对困惑度那样。参数或计算量翻倍，误差可靠地减半。这是第一个展现出与语言模型同样清晰缩放行为的图像生成模型。结果是，VAR 规模的预测可以从计算量中得出，而无需对每个架构进行经验猜测。

### 与扩散的关系

VAR 和扩散共享相同的数据压缩故事：两者都将生成问题分解为一系列更容易的子问题。

- 扩散：逐渐添加噪声，学习撤销一步。
- VAR：逐渐增加分辨率，学习预测下一个尺度。

它们是通过问题的不同轴。两者都产生可处理的条件分布。经验上，VAR 推理更快（更少传播，尺度内全部并行），在类别条件 ImageNet 上匹配或胜过 DiT。文本条件 VAR（VARclip、HART）是活跃的研究方向。

## 动手实现

在 `code/main.py` 中你将：
1. 在合成"图像"数据（2D 高斯环）上构建一个微型**多尺度 VQ 分词器**。
2. 训练一个 **VAR 风格 Transformer** 进行下一尺度预测。
3. 通过 4 次调用 Transformer（4 个尺度）并解码来采样。
4. 验证尺度有序训练确实使尺度内生成并行。

这是一个玩具实现。重点是看到尺度结构化注意力掩码和尺度内并行生成实际运行。

## 练习

1. **尺度数量消融。** 用 4、6、8、10 个尺度训练 VAR。测量重建质量 vs 自回归传播次数。尺度越多 = 残差越细 = 质量更好，但传播更多。

2. **码本大小。** 用码本大小 512、4096、16384 训练分词器。更大的码本提供更好的重建，但预测更难。找出拐点。

3. **尺度内并行性检验。** 对训练好的 VAR，明确测量注意力模式。在尺度 k 内，模型是否关注跨尺度位置但不关注尺度内位置？验证掩码实现。

4. **VAR vs DiT 缩放。** 对相同的 ImageNet 类别条件任务，以匹配的参数预算（例如 33M、130M、458M）训练 VAR 和 DiT。绘制 FID vs 计算量。VAR 应该在每个规模上领先 DiT——在小规模下复现论文结果。

5. **文本条件化。** 扩展 VAR 通过 adaLN 接受文本嵌入（CLIP 池化）作为额外条件输入。这是 HART 方案。文本对齐采样的 FID 改善了多少？

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| VAR | "视觉自回归" | 通过 VQ token 网格金字塔的下一尺度预测进行图像生成。 |
| 下一尺度预测（Next-scale prediction） | "先预测粗的，再预测细的" | 模型以递增分辨率尺度预测 token，以所有前一尺度为条件。 |
| 多尺度 VQ 分词器（Multi-scale VQ tokenizer） | "残差 VQ" | 生成 K 个递增分辨率 token 网格的 VQ-VAE，解码器对所有尺度求和。 |
| 尺度 k（Scale k） | "金字塔层级 k" | K 个分辨率层级之一，从 k=1 的 1×1 到 k=K 的 (H/p)×(W/p)。 |
| 尺度内并行（Parallel-within-scale） | "每尺度一次前向" | 尺度 k 的所有 token 在一次 Transformer 传播中预测，非自回归。 |
| 跨尺度因果（Causal-across-scales） | "尺度有序注意力" | 尺度 k 的 token 可以关注尺度 1..k 的所有内容，但不能关注尺度 k+1..K。 |
| 残差 VQ（Residual VQ） | "加法分词" | 每个尺度的 token 编码较低尺度遗漏的残差；解码器对所有尺度嵌入求和。 |
| VAR 缩放定律（VAR scaling law） | "图像 GPT 缩放" | FID 遵循计算量的可预测幂律，类似语言模型的困惑度。 |
| HART | "混合 VAR + 文本" | 结合 MaskGIT 风格迭代解码与 VAR 尺度结构的文本条件 VAR 变体。 |
| 尺度位置嵌入（Scale position embedding） | "(尺度, 行, 列) 三元组" | 位置编码携带尺度索引和尺度内空间坐标。 |
