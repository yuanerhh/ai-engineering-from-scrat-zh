# 范数与距离

> 你的距离函数定义了"相似"的含义。选错了，下游的一切都会崩溃。

**类型：** 实践
**语言：** Python
**前置条件：** 第 1 阶段，第 01 课（线性代数直觉）、第 02 课（向量、矩阵与运算）
**时间：** ~90 分钟

## 学习目标

- 从零实现 L1、L2、余弦、马氏、Jaccard 和编辑距离函数
- 为给定 ML 任务选择合适的距离度量，并解释为何其他选择会失败
- 将 L1 和 L2 范数与 LASSO 和 Ridge 正则化及其几何约束区域联系起来
- 演示相同数据集在不同度量下产生不同的最近邻

## 问题所在

你有两个向量。也许是词嵌入，也许是用户画像，也许是像素数组。你需要知道：它们有多近？

答案完全取决于你选择哪个距离函数。两个数据点在一种度量下可能是最近邻，在另一种度量下可能相距很远。你的 KNN 分类器、推荐引擎、向量数据库、聚类算法、损失函数——它们都依赖于这个选择。选错了，你的模型就在优化错误的事情。

没有普遍最优的距离。L2 适用于空间数据。余弦相似度主导 NLP。Jaccard 处理集合。编辑距离处理字符串。马氏距离考虑相关性。Wasserstein 移动概率质量。每一种都编码了不同的"相似"假设。

本课从零构建每种主要距离函数，告诉你每种工具的正确使用场景，并展示相同数据在不同度量下如何产生完全不同的最近邻。

## 核心概念

### 范数：测量向量大小

范数测量向量的"大小"。两个向量之间的每个距离函数都可以写成它们差的范数：d(a, b) = ||a - b||。所以理解范数就是理解距离。

### L1 范数（曼哈顿距离）

L1 范数对所有分量的绝对值求和。

```
||x||_1 = |x_1| + |x_2| + ... + |x_n|
```

它被称为曼哈顿距离，因为它测量在只能沿轴移动的城市网格上行走的距离。没有对角线。

```
点 A = (1, 1)
点 B = (4, 5)

L1 距离 = |4-1| + |5-1| = 3 + 4 = 7

在网格上，你向东走 3 个街区，向北走 4 个街区。
```

何时使用 L1：
- 高维稀疏数据（文本特征、独热编码）
- 当你想对异常值有鲁棒性时（单个巨大差异不会主导结果）
- 特征选择问题（L1 正则化促进稀疏性）

与 L1 正则化（Lasso）的联系：在损失函数中加入 ||w||_1 会惩罚权重绝对值的和。这会将小权重推向精确的零，执行自动特征选择。L1 惩罚在权重空间中创建菱形约束区域，菱形的角位于某些权重为零的轴上。

与损失函数的联系：平均绝对误差（MAE）是预测值与目标值之间 L1 距离的平均。它线性惩罚所有误差，与 MSE 相比对异常值更鲁棒。

### L2 范数（欧几里得距离）

L2 范数是直线距离。所有分量平方和的平方根。

```
||x||_2 = sqrt(x_1^2 + x_2^2 + ... + x_n^2)
```

这是你在几何课上学到的距离。n 维空间中的勾股定理。

```
点 A = (1, 1)
点 B = (4, 5)

L2 距离 = sqrt((4-1)^2 + (5-1)^2) = sqrt(9 + 16) = sqrt(25) = 5.0

穿过网格对角线的直线。
```

何时使用 L2：
- 低到中等维度的连续数据
- 当特征尺度可比时
- 物理距离（空间数据、传感器读数）
- 像素级的图像相似度

与 L2 正则化（Ridge）的联系：在损失函数中加入 ||w||_2^2 会惩罚大权重。与 L1 不同，它不会将权重推向零。它按比例将所有权重向零收缩。L2 惩罚创建圆形约束区域，所以轴上没有角。权重变小但很少精确为零。

与损失函数的联系：均方误差（MSE）是 L2 距离平方的平均。平方会比小误差更重地惩罚大误差。

```
MAE（L1 损失）：|y - y_hat|         线性惩罚。对异常值鲁棒。
MSE（L2 损失）：(y - y_hat)^2       二次惩罚。对异常值敏感。
```

### Lp 范数：一般族

L1 和 L2 是 Lp 范数的特殊情况：

```
||x||_p = (|x_1|^p + |x_2|^p + ... + |x_n|^p)^(1/p)
```

不同的 p 值产生不同形状的"单位球"（距原点距离为 1 的所有点的集合）：

```
p=1：    菱形      （角在轴上）
p=2：    圆/球     （通常的圆形球）
p=3：    超椭圆    （圆角正方形）
p=inf：  正方形/超立方体  （沿轴的平面）
```

### L 无穷范数（切比雪夫距离）

当 p 趋向无穷时，Lp 范数收敛到最大绝对分量。

```
||x||_inf = max(|x_1|, |x_2|, ..., |x_n|)
```

两点之间的距离由它们相差最大的单一维度决定。所有其他维度被忽略。

```
点 A = (1, 1)
点 B = (4, 5)

L 无穷距离 = max(|4-1|, |5-1|) = max(3, 4) = 4
```

何时使用 L 无穷：
- 当任何单一维度的最坏情况偏差重要时
- 棋盘游戏（国际象棋中的国王用 L 无穷移动：向任意方向走一步代价为 1）
- 制造公差（每个维度都必须在规格内）

### 余弦相似度与余弦距离

余弦相似度测量两个向量之间的角度，忽略它们的大小。

```
cos_sim(a, b) = (a · b) / (||a||_2 * ||b||_2)
```

范围从 -1（相反方向）到 +1（相同方向）。垂直向量的余弦相似度为 0。

余弦距离将其转换为距离：cosine_distance = 1 - cosine_similarity。范围从 0（相同方向）到 2（相反方向）。

```
a = (1, 0)    b = (1, 1)

cos_sim = (1*1 + 0*1) / (1 * sqrt(2)) = 1/sqrt(2) = 0.707
cos_dist = 1 - 0.707 = 0.293
```

为什么余弦主导 NLP 和嵌入：在文本中，文档长度不应影响相似度。一篇关于猫的文档如果比另一篇关于猫的文档长两倍，它们仍然应该"相似"。余弦相似度忽略大小（长度），只关心方向。具有相同词分布但长度不同的两个文档指向相同方向，余弦相似度为 1.0。

何时使用余弦相似度：
- 文本相似度（TF-IDF 向量、词嵌入、句子嵌入）
- 任何大小是噪声而方向是信号的领域
- 推荐系统（用户偏好向量）
- 嵌入搜索（向量数据库几乎总是使用余弦或点积）

### 点积相似度 vs 余弦相似度

两个向量的点积是：

```
a · b = a_1*b_1 + a_2*b_2 + ... + a_n*b_n
      = ||a|| * ||b|| * cos(角度)
```

余弦相似度是点积除以两个大小后的归一化。当两个向量都已经单位归一化（大小 = 1）时，点积和余弦相似度相同。

```
如果 ||a|| = 1 且 ||b|| = 1：
    a · b = cos(a 和 b 之间的角度)
```

何时它们不同：点积包含大小信息。大小更大的向量得到更高的点积分数。在某些检索系统中，当你希望"热门"项目排名更高时，这很重要。大小作为隐式的质量或重要性信号。

```
a = (3, 0)    b = (1, 0)    c = (0, 1)

dot(a, b) = 3     dot(a, c) = 0
cos(a, b) = 1.0   cos(a, c) = 0.0

两者在方向上一致，但点积还反映大小。
```

实践中：
- 当你想要纯方向相似度时使用余弦相似度
- 当大小携带有意义的信息时使用点积
- 许多向量数据库（Pinecone、Weaviate、Qdrant）让你在两者之间选择
- 如果你的嵌入是 L2 归一化的，选择无关紧要

### 马氏距离

欧几里得距离平等对待所有维度。但如果你的特征是相关的或有不同尺度，L2 会给出误导性结果。

马氏距离考虑数据的协方差结构。

```
d_M(x, y) = sqrt((x - y)^T * S^(-1) * (x - y))
```

其中 S 是数据的协方差矩阵。

直觉上：马氏距离首先对数据进行去相关和归一化（白化），然后在该变换空间中计算 L2 距离。如果 S 是单位矩阵（不相关、单位方差特征），马氏距离退化为欧几里得距离。

```
示例：身高和体重是相关的。
身高 6'2"、体重 180 磅的人并不罕见。
身高 5'0"、体重 180 磅的人很罕见。

欧几里得距离可能说他们离均值等距。
马氏距离正确识别第二个为异常值，
因为它考虑了身高-体重的相关性。
```

何时使用马氏距离：
- 异常值检测（到均值马氏距离大的点是异常值）
- 特征有不同尺度和相关性时的分类
- 当你有足够数据来估计可靠的协方差矩阵时
- 制造业的质量控制（多变量过程监控）

### Jaccard 相似度（用于集合）

Jaccard 相似度测量两个集合之间的重叠。

```
J(A, B) = |A ∩ B| / |A ∪ B|
```

范围从 0（无重叠）到 1（相同集合）。Jaccard 距离 = 1 - Jaccard 相似度。

```
A = {猫, 狗, 鱼}
B = {猫, 鸟, 鱼, 蛇}

交集 = {猫, 鱼}              大小 = 2
并集 = {猫, 狗, 鱼, 鸟, 蛇}  大小 = 5

Jaccard 相似度 = 2/5 = 0.4
Jaccard 距离 = 0.6
```

何时使用 Jaccard：
- 比较标签、类别或特征的集合
- 基于词出现（不是频率）的文档相似度
- 近重复检测（MinHash 近似 Jaccard）
- 比较二值特征向量（存在/缺失数据）
- 评估分割模型（交并比 = Jaccard）

### 编辑距离（Levenshtein 距离）

编辑距离计算将一个字符串转换为另一个字符串所需的最少单字符操作数。操作为：插入、删除或替换。

```
"kitten" -> "sitting"

kitten -> sitten  （替换 k -> s）
sitten -> sittin  （替换 e -> i）
sittin -> sitting （插入 g）

编辑距离 = 3
```

使用动态规划计算。填充一个矩阵，其中条目 (i, j) 是字符串 A 的前 i 个字符与字符串 B 的前 j 个字符之间的编辑距离。

```
        ""  s  i  t  t  i  n  g
    ""   0  1  2  3  4  5  6  7
    k    1  1  2  3  4  5  6  7
    i    2  2  1  2  3  4  5  6
    t    3  3  2  1  2  3  4  5
    t    4  4  3  2  1  2  3  4
    e    5  5  4  3  2  2  3  4
    n    6  6  5  4  3  3  2  3
```

何时使用编辑距离：
- 拼写检查和纠错
- DNA 序列比对（带加权操作）
- 模糊字符串匹配
- 杂乱文本数据的去重

### KL 散度（不是距离，但被当作距离使用）

KL 散度测量一个概率分布与另一个的差异。在第 09 课中已介绍，但它属于这里的讨论，因为尽管它不是距离，人们仍将其当作"距离"使用。

```
D_KL(P || Q) = sum(p(x) * log(p(x) / q(x)))
```

关键属性：KL 散度不是对称的。

```
D_KL(P || Q) != D_KL(Q || P)
```

这意味着它不满足距离度量的基本要求。它也不满足三角不等式。它是散度，不是距离。

前向 KL（D_KL(P || Q)）是"均值寻求"的：Q 试图覆盖 P 的所有模式。
反向 KL（D_KL(Q || P)）是"模式寻求"的：Q 专注于 P 的单一模式。

你看到 KL 散度的场合：
- VAE（ELBO 中的 KL 项将隐变量分布推向先验）
- 知识蒸馏（学生尝试匹配教师的分布）
- RLHF（KL 惩罚使微调模型接近基础模型）
- 策略梯度方法（约束策略更新）

### Wasserstein 距离（推土机距离）

Wasserstein 距离测量将一个概率分布转化为另一个所需的最小"工作量"。可以理解为：如果一个分布是一堆泥土，另一个是一个洞，你需要搬运多少泥土，搬运多远？

```
W(P, Q) = 对所有传输方案 gamma 的 E[d(x, y)] 的下确界
```

对于一维分布，它简化为累积分布函数之差的绝对值的积分：

```
W_1(P, Q) = integral |CDF_P(x) - CDF_Q(x)| dx
```

为什么 Wasserstein 很重要：
- 它是真正的度量（对称，满足三角不等式）
- 即使分布不重叠也能提供梯度（KL 散度趋向无穷）
- 这一性质使其成为 Wasserstein GAN（WGAN）的核心，解决了原始 GAN 的训练不稳定性

```
不重叠的分布：

P: [1, 0, 0, 0, 0]    Q: [0, 0, 0, 0, 1]

KL 散度：无穷大（零的对数）
Wasserstein：4（将所有质量移动 4 个箱位）

Wasserstein 提供有意义的梯度，KL 不行。
```

何时使用 Wasserstein：
- GAN 训练（WGAN、WGAN-GP）
- 比较可能不重叠的分布
- 最优传输问题
- 图像检索（比较颜色直方图）

### 为什么不同任务需要不同距离

| 任务 | 最佳距离 | 原因 |
|------|---------|------|
| 文本相似度 | 余弦 | 大小是噪声，方向是意义 |
| 图像像素比较 | L2 | 空间关系重要，特征尺度可比 |
| 稀疏高维特征 | L1 | 鲁棒，不放大稀有的大差异 |
| 集合重叠（标签、类别）| Jaccard | 数据本质上是集合值，不是向量 |
| 字符串匹配 | 编辑距离 | 操作对应人类编辑直觉 |
| 异常值检测 | 马氏距离 | 考虑特征相关性和尺度 |
| 比较分布 | KL 散度 | 测量用 Q 代替 P 损失的信息量 |
| GAN 训练 | Wasserstein | 即使分布不重叠也能提供梯度 |
| 嵌入（向量数据库）| 余弦或点积 | 嵌入被训练为在方向上编码意义 |
| 推荐 | 点积 | 大小可以编码流行度或置信度 |
| DNA 序列 | 加权编辑距离 | 替换代价因核苷酸对而异 |
| 制造质控 | L 无穷 | 任何维度的最坏情况偏差重要 |

### 与损失函数的联系

损失函数是应用于预测值与目标值的距离函数。

```
损失函数      使用的距离           行为
MSE           L2 平方             重惩罚大误差
MAE           L1                  平等惩罚所有误差
Huber 损失    大误差用 L1，       两全其美：对异常值鲁棒，
              小误差用 L2         零附近梯度平滑
交叉熵        KL 散度             测量分布不匹配
Hinge 损失    max(0, 边距 - d)    只惩罚低于边距的情况
三元组损失    L2（通常）          拉近正样本，推远负样本
对比损失      L2                  相似对靠近，不相似对超过边距
```

### 与正则化的联系

正则化在损失函数中添加权重的范数惩罚。

```
L1 正则化（Lasso）：  损失 + lambda * ||w||_1
  -> 稀疏权重。某些权重变为精确的零。
  -> 自动特征选择。
  -> 解在零处有角（不可微）。

L2 正则化（Ridge）：  损失 + lambda * ||w||_2^2
  -> 小权重。所有权重向零收缩。
  -> 没有特征选择（没有东西精确变为零）。
  -> 处处平滑。

弹性网络：             损失 + lambda_1 * ||w||_1 + lambda_2 * ||w||_2^2
  -> 结合了 L1 的稀疏性和 L2 的稳定性。
  -> 相关特征组一起保留或丢弃。
```

为什么 L1 产生稀疏性而 L2 不产生：想象 2D 权重空间中的约束区域。L1 是菱形，L2 是圆。损失函数的等值线（椭圆）最可能在角处接触菱形，那里某个权重为零。它们在光滑点接触圆，那里两个权重都非零。

### 最近邻搜索

每个距离函数都意味着一个最近邻搜索问题：给定一个查询点，在数据集中找到最近的点。

精确最近邻搜索在有 n 个点、d 维的数据集中每次查询是 O(n * d)。对于大数据集，这太慢了。

近似最近邻（ANN）算法以少量精度损失换取巨大速度提升：

```
算法            方法                          使用者
KD 树           轴对齐的空间分区               scikit-learn（低维）
球树            嵌套超球面                     scikit-learn（中维）
LSH             随机哈希投影                   近重复检测
HNSW            分层可导航小世界图             FAISS、Qdrant、Weaviate
IVF             带聚类搜索的倒排文件索引        FAISS（十亿规模）
乘积量化        压缩向量，在压缩空间搜索        FAISS（内存受限）
```

HNSW（分层可导航小世界）是现代向量数据库中的主流算法。它构建一个多层图，每个节点连接到其近似最近邻。搜索从顶层开始（稀疏，长跳跃），下降到底层（密集，短跳跃）。

## 应用

最常见的实际用途：在向量数据库中找到相似项目。

```python
import numpy as np

def cosine_similarity_matrix(X):
    norms = np.linalg.norm(X, axis=1, keepdims=True)
    norms = np.where(norms == 0, 1, norms)
    X_normalized = X / norms
    return X_normalized @ X_normalized.T

embeddings = np.random.randn(1000, 768)

sim_matrix = cosine_similarity_matrix(embeddings)

query_idx = 0
similarities = sim_matrix[query_idx]
top_k = np.argsort(similarities)[::-1][1:6]
print(f"与第 0 项最相似的前 5 项: {top_k}")
print(f"相似度: {similarities[top_k]}")
```

当你调用 `model.encode(text)` 然后搜索向量数据库时，幕后就发生着这些。嵌入模型将文本映射到向量。向量数据库计算查询向量与每个存储向量之间的余弦相似度（或点积），使用 ANN 算法避免检查所有向量。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| 范数 | "向量的大小" | 将向量映射到非负标量的函数，满足三角不等式、绝对齐次性，且仅对零向量为零 |
| L1 范数 | "曼哈顿距离" | 绝对分量值之和。在优化中产生稀疏性。对异常值鲁棒 |
| L2 范数 | "欧几里得距离" | 平方分量和的平方根。欧几里得空间中的直线距离 |
| Lp 范数 | "广义范数" | 绝对分量的 p 次方和的 p 次根。L1 和 L2 是特例 |
| L 无穷范数 | "最大范数"或"切比雪夫距离" | 最大绝对分量值。p 趋向无穷时 Lp 的极限 |
| 余弦相似度 | "向量间的角度" | 点积除以两个大小后的值。范围从 -1 到 +1。忽略向量长度 |
| 余弦距离 | "1 减余弦相似度" | 将余弦相似度转换为距离。范围从 0 到 2 |
| 点积 | "未归一化的余弦" | 逐元素乘积的和。等于余弦相似度乘以两个大小 |
| 马氏距离 | "考虑相关性的距离" | 使用数据协方差矩阵白化（去相关和归一化）后空间中的 L2 距离 |
| Jaccard 相似度 | "集合重叠" | 交集大小除以并集大小。用于集合，不用于向量 |
| 编辑距离 | "Levenshtein 距离" | 将一个字符串转换为另一个所需的最少插入、删除和替换次数 |
| KL 散度 | "分布之间的距离" | 不是真正的距离（不对称）。测量使用 Q 编码 P 的额外比特数 |
| Wasserstein 距离 | "推土机距离" | 将质量从一个分布传输到另一个的最小工作量。真正的度量 |
| 近似最近邻 | "ANN 搜索" | 比精确搜索快得多地找到近似最近点的算法（HNSW、LSH、IVF）|
| HNSW | "向量数据库算法" | 分层可导航小世界图。用于快速近似最近邻搜索的多层图 |
| L1 正则化 | "Lasso" | 在损失中加入权重的 L1 范数。将权重驱向零（稀疏性）|
| L2 正则化 | "Ridge"或"权重衰减" | 在损失中加入权重的 L2 范数平方。将权重向零收缩，不产生稀疏性 |
| 弹性网络 | "L1 + L2" | 结合 L1 和 L2 正则化。比单独使用任一种都能更好地处理相关特征组 |
