# Chameleon 与早期融合纯词元多模态模型

> 迄今为止我们看到的每个 VLM 都将图像和文本分开处理。视觉词元来自视觉编码器，流经投影器，然后在 LLM 内部与文本汇合。视觉词汇表和文本词汇表从不重叠。Chameleon（Meta，2024 年 5 月）提问：如果它们共享一个词汇表呢？训练一个 VQ-VAE，将图像转换为共享词汇表中的离散词元序列。每个多模态文档于是就变成了一个序列——文本词元和图像词元交错，使用单一的自回归损失。副作用：模型可以生成混合模态输出——在单次推理调用中交替输出文本和图像词元。本课解读早期融合理论，并端到端构建一个玩具版本。

**类型：** 构建
**编程语言：** Python（标准库，VQ-VAE 分词器 + 交错解码器）
**前置知识：** Phase 12 · 05、Phase 8（生成式 AI）
**预计时间：** 约 180 分钟

## 学习目标

- 解释为何共享词汇表 + 单一损失改变了模型的能力边界。
- 描述 VQ-VAE 如何将图像分词为与 Transformer 下一词元目标兼容的离散序列。
- 说出 Chameleon 的训练稳定性技巧：QK-Norm、dropout 放置和 LayerNorm 排序。
- 对比 Chameleon 与 BLIP-2 的 Q-Former 方案，描述各自适用的场景。

## 问题背景

基于适配器的 VLM（LLaVA、BLIP-2、Qwen-VL）将文本和图像视为两种不同的东西。文本词元经过 `embed(text_token)`；图像经过 `visual_encoder(image) → projector → ... 伪词元`。模型有两条在中途汇合的输入路径。

三个后果：

1. LLM 只能消费图像，不能生成图像，输出只有文本。
2. 混合模态文档（如文章中段落和图像交替出现）处理起来很尴尬——要么在模型外部解析多模态输入，要么链式生成。
3. 分布不匹配。视觉词元和文本词元存在于隐藏空间的不同区域，产生细微的对齐问题。

Chameleon 否定了这个前提：图像就是共享词汇表中的离散词元序列。在交错文档上训练模型，一个损失，一个自回归解码器，混合模态生成能力随之解锁。

## 核心概念

### VQ-VAE 作为图像分词器

分词器是向量量化变分自编码器，架构如下：

- 编码器：CNN + ViT，将图像映射到空间特征图，如维度 256 的 32×32 特征。
- 码本：K 个可学习向量（Chameleon 使用 8192），也是维度 256。
- 量化：对每个空间特征，通过 L2 距离查找最近的码本条目，用整数索引替换连续特征。
- 解码器：CNN，将量化特征还原为像素。

训练：VAE 重建损失 + 承诺损失 + 码本损失。码本索引构成图像的离散字母表。

对于 Chameleon：一张图像变成来自 8192 词汇表的 32×32 = 1024 个词元。与文本词元（来自 LLM 的 BPE 词汇表，如 32000）拼接。最终词汇表：40192。Transformer 看到一个序列，一个损失。

### 共享词汇表

Chameleon 的词汇表结合了文本词元、图像词元和模态分隔符。每个词元有一个 ID。输入嵌入层将每个 ID 映射到 D 维隐藏向量，输出投影将隐藏映射回词汇 logit，softmax 选择下一个词元，不管是哪种模态。

分隔符很重要：`<image>` 和 `</image>` 标签括住图像词元序列。生成时，如果模型输出 `<image>`，下游软件就知道接下来的 1024 个词元是 VQ 索引，需要发送到解码器进行像素渲染。

### 混合模态生成

推理就是在共享词汇表中进行下一词元预测。示例提示："画一只猫并描述它。"Chameleon 输出：

```
<image> 4821 1029 2891 ... (1024 个图像词元) </image>
这只猫是橙色的，坐在窗台上……
```

模型自主决定顺序——可能先输出图像再输出文本，或先文本再图像，或交错输出。相同的解码器，相同的损失。

与适配器 VLM（只能生成文本）相比，Chameleon 重新打开了关于模型输出模态的讨论。

### 训练稳定性——QK-Norm、dropout 和 LayerNorm 排序

早期融合训练在大规模下不稳定。Chameleon 论文记录了三个技巧：

- **QK-Norm**：在注意力机制内，在点积之前对查询和键投影应用 LayerNorm。防止深层网络中 logit 幅度爆炸。2024 年后多个大型模型都采用了这一技巧。
- **Dropout 放置**：每个残差加法之后都使用 dropout，而非仅在注意力和 MLP 之后。来自图像词元的梯度可能占主导时，需要更多正则化。
- **LayerNorm 排序**：在残差分支上使用前置 LN（标准），加上最后一个块跳跃连接上的额外 LN。稳定最后一层的梯度流。

没有这些技巧，340 亿参数的 Chameleon 训练在多个检查点处发散；有了这些技巧，训练收敛。训练配方与架构本身同等重要。

### 分词器的重建上限

VQ-VAE 是有损的。对 512×512 图像使用 8192 个码本条目和 1024 个词元，重建 PSNR 上限约为 26–28 dB。对可识别的图像生成已经足够，但明显不如连续空间扩散模型（Stable Diffusion 3 可达 32+ dB）。

分词器是瓶颈。更好的分词器（MAGVIT-v2、IBQ、SBER-MoVQGAN）可以提升上限。Emu3（12.12 课）仅通过更好的分词器就实现了 SDXL 级别的生成质量。

### Chameleon vs BLIP-2 / LLaVA

**Chameleon（早期融合，共享词汇表）：**
- 一个损失，一个解码器。
- 生成混合模态输出。
- 分词器是质量上限。
- 推理路径上每张生成图像都需要 VQ-VAE 解码器，成本较高。

**BLIP-2 / LLaVA（后期融合，独立编码器）：**
- 视觉输入，纯文本输出。
- 复用预训练 LLM。
- 理解任务没有分词器瓶颈。
- 单次前向传播，成本低。

按任务选择：如果需要图像生成，选 Chameleon 系列；如果只需要理解，适配器 VLM 更简单，能复用更多预训练计算量。

### Fuyu 和 AnyGPT

Fuyu（Adept，2023）是相关方案：完全跳过独立的视觉编码器，将原始图像块直接通过 LLM 的输入投影，就好像它们是词元一样，不需要分词器。比 Chameleon 更简单，但失去了共享词汇表输出生成能力。

AnyGPT（Zhan 等，2024）将 Chameleon 扩展到四种模态：文本、图像、语音、音乐。对每种模态都使用相同的 VQ-VAE 技巧，共享 Transformer，实现任意模态到任意模态的生成。更多内容见 12.16 课。

## 动手实践

`code/main.py` 构建了一个玩具端到端早期融合模型：

- 一个将 8×8 块映射到码本索引（K=16）的微型 VQ-VAE 风格量化器。
- 共享词汇表：（文本 id 0..31）+（图像 id 32..47）+（分隔符 48、49）。
- 在合成描述 + 图像词元序列上训练的玩具自回归解码器（二元组表）。
- 给定提示，输出交替文本 + 图像词元的采样循环。

代码故意将 Transformer 保持为极小规模（二元组），以便你可以端到端追踪信号流。

## 练习

1. Chameleon 使用 K=8192 个码本条目和每张 512×512 图像 1024 个词元。估算与 24 位 RGB 图像相比的压缩比。这是有损压缩吗？损失有多大？

2. 一张 4K 图像（3840×2160）在相同 VQ-VAE 密度下产生多少图像词元？Chameleon 风格的模型能在一次推理调用中生成一张 4K 图像吗？最先出问题的是上下文、分词器质量还是 KV 缓存？

3. 用纯 Python 实现 QK-Norm。给定 64 维的查询和键，展示 LayerNorm 前后的点积结果。为什么在深层网络中幅度控制很重要？

4. 阅读 Chameleon 第 2.3 节关于训练稳定性的内容。描述没有 QK-Norm 时在 340 亿参数规模下观察到的确切失败模式，"范数爆炸"的特征是什么？

5. 扩展玩具解码器，给定纯文本提示，输出混合模态回复。测量训练数据分布为 60% 文本优先 / 40% 图像优先时，模型多频繁地选择先输出图像还是先输出文本。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| 早期融合（Early fusion） | "统一词元" | 图像从第一步起就被转换为共享 Transformer 词汇表的离散词元 |
| VQ-VAE | "图像分词器" | CNN + ViT + 码本，将图像映射为 Transformer 可以预测的整数索引 |
| 共享词汇表（Shared vocabulary） | "一个词典" | 覆盖文本 + 图像 + 模态分隔符的单一词元 ID 空间 |
| QK-Norm | "注意力稳定器" | 在查询和键进行点积前应用 LayerNorm，防止范数爆炸 |
| 混合模态生成 | "文本 + 图像输出" | 在单次推理过程中自主生成交错文本和图像词元 |
| 码本大小（Codebook size） | "K 个条目" | VQ-VAE 可以量化的离散向量数量；在压缩率和保真度之间权衡 |
| 分词器上限（Tokenizer ceiling） | "重建极限" | 解码 VQ 词元可达到的最佳 PSNR；限制模型的图像质量 |
