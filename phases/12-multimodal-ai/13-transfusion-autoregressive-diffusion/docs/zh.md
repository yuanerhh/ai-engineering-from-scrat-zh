# Transfusion：一个 Transformer 中的自回归文本 + 扩散图像

> Chameleon 和 Emu3 将一切押注在离散词元上，这个方法可行，但量化瓶颈清晰可见——图像质量在连续空间扩散模型之下趋于平稳。Transfusion（Meta，Zhou 等，2024 年 8 月）做出了相反的赌注：保持图像连续化，完全去掉 VQ-VAE，用两个损失训练一个 Transformer。文本词元使用下一词元预测，图像块使用流匹配/扩散损失，两个目标优化相同的权重。Stable Diffusion 3 的底层架构（MMDiT）是其近亲。本课解读 Transfusion 论点，构建一个玩具双损失训练器，并追踪让一个 Transformer 完成两种任务的注意力掩码。

**类型：** 构建
**编程语言：** Python（标准库，MNIST 规模玩具双损失训练器）
**前置知识：** Phase 12 · 11（Chameleon）、Phase 8（生成式 AI）
**预计时间：** 约 180 分钟

## 学习目标

- 配置一个在同一骨干网络上对文本词元运行 NTP 损失、对图像块运行扩散 MSE 损失的 Transformer。
- 解释为何对图像块使用双向注意力、对文本词元使用因果注意力是正确的掩码选择。
- 在（计算量、质量、代码复杂度）三个维度对比 Transfusion 风格（连续图像、扩散损失）与 Chameleon 风格（离散图像、NTP）。
- 说出 MMDiT 的贡献：每个块使用模态特定的权重，在残差流中进行联合注意力。

## 问题背景

离散 vs 连续图像词元的争论比 LLM 还古老。连续表示（原始像素、VAE 潜变量）保留细节；离散词元（VQ 索引）符合 Transformer 的原生词汇表，但在量化步骤中损失细节。

Chameleon / Emu3 选择了离散：一个损失，一个架构，但图像保真度受限于分词器质量。

扩散模型选择了连续：出色的图像质量，但与 LLM 是独立的模型，需要复杂的噪声调度工程，且无法干净地与文本生成集成。

Transfusion 的问题是：能否两者兼得？保持图像连续化，仍然训练一个模型，在一个梯度步中缝合两个损失。

## 核心概念

### 双损失架构

单个仅解码器的 Transformer 处理包含以下内容的序列：

- 文本词元（离散，来自 BPE 词汇表）。
- 图像块（连续，16×16 像素块通过线性嵌入投影到隐藏维度——与 ViT 编码器的输入相同）。
- 标记连续块位置的 `<image>` 和 `</image>` 标签。

前向传播只运行一次，损失按词元类型选择两种头之一：

- 对文本词元：词汇 logit 头上的标准交叉熵。
- 对图像块：连续块上的扩散损失——预测添加到每个块的噪声。

梯度流经共享的 Transformer 主体，两个损失同时改善共享权重。

### 注意力掩码：因果文本 + 双向图像

文本词元必须是因果的——不能让文本词元聚合未来的文本，否则教师强迫会失效。而图像块代表一个快照，同一图像块内的块应该相互进行双向注意力。

掩码规则：

```
M[i, j] = 1 当：
  (i 是文本，j 是文本，且 j <= i)       # 文本的因果性
  OR (i 是图像，j 是图像，且 same_image_block(i, j))  # 图像块内的双向性
  OR (i 是文本，j 是图像，且 j < i_image_end)         # 文本聚合前面的图像
  OR (i 是图像，j 是文本，且 j < i_image_start)        # 图像聚合前面的文本
```

训练和推理时实现为块三角掩码。

### Transformer 内部的扩散损失

扩散损失是标准的：向图像块添加噪声，要求模型预测噪声（或等效地，预测干净块）。Transfusion 的版本使用流匹配——预测从噪声到干净数据的速度场。

训练过程：
1. 对每个图像块 x0，采样一个随机时间步 t。
2. 采样噪声 ε，计算 xt = (1-t) * x0 + t * ε（流匹配的线性插值）。
3. Transformer 预测 v_θ(xt, t)；损失 = MSE(v_θ(xt, t), ε - x0)。
4. 与同一序列的文本 NTP 损失一起反向传播。

推理时，生成过程为：
- 文本词元：标准自回归采样。
- 图像块：扩散采样循环（通常 10–30 步），以前面的文本词元为条件。

### MMDiT：Stable Diffusion 3 的变体

Stable Diffusion 3（Esser 等，2024 年 3 月）与 Transfusion 几乎同时发布了 MMDiT（多模态扩散 Transformer），两种架构互为姐妹。

MMDiT 的主要区别：

- **每个块使用模态特定的权重**：每个 Transformer 块对文本词元和图像块有独立的 Q、K、V 和 MLP 权重。注意力是联合的（跨模态）；其余部分是模态特定的。
- **整流流训练**：一种具有已知采样特性且比 DDPM 数学更简单的特定流匹配变体。
- **规模**：MMDiT 是 SD3 的骨干（20 亿和 80 亿参数变体）。Transfusion 论文扩展到 70 亿参数。

两种方法都指向同一核心思路：一个 Transformer 在文本上运行 NTP，在连续图像表示上运行扩散。

### 为何优于 Chameleon 风格

在图像生成上，连续扩散与离散 NTP 之间的质量差距是可以量化的。Transfusion 论文报告：

- 在 70 亿参数规模下，在 FID 上击败相同规模的 Chameleon 风格模型 3–5 分。
- 不需要分词器训练——图像编码器更简单（线性投影到隐藏维度，与 ViT 的输入层相同）。
- 推理时可以并行化图像块的去噪，这是自回归图像词元所不具备的。

缺点：Transfusion 是双损失模型，使训练动态更加复杂。损失权重需要调整，NTP 和扩散之间的调度不匹配可能导致某个损失头占主导。

### 下游发展

Janus-Pro（12.15 课）通过为理解和生成解耦视觉编码器（理解用 SigLIP，生成用 VQ），同时共享 Transformer 主体，精炼了 Transfusion 的思路。Show-o（12.14 课）将扩散换成了离散扩散（掩码预测）。统一生成家族在 Transfusion 之后迅速分支。

2026 年能够输出图像的生产 VLM——Gemini 3 Pro、GPT-5、Claude Opus 4.7 的图像生成路径——几乎可以肯定使用了这个家族的某种后代。具体细节是专有的。

## 动手实践

`code/main.py` 在一个微型类 MNIST 问题上构建了玩具 Transfusion：

- 文本描述是描述数字（0–9）的短整数序列。
- 图像是 4×4 的字节网格。
- 一对共享权重的线性投影层充当 Transformer 的替代；文本的 NTP 损失，带噪图像块的 MSE 损失。
- 训练循环交替运行两个损失，注意力掩码是显式的。
- 生成过程在一次前向传播中产生文本描述和一张 4×4 图像。

Transformer 是玩具级别的，但双损失管道、注意力掩码构建和推理循环才是真正的产出。

## 产出技能

本课产出 `outputs/skill-two-loss-trainer-designer.md`：给定一个新的多模态训练任务（文本 + 图像、文本 + 音频、文本 + 视频），设计双损失调度（损失权重、掩码形状、共享 vs 模态特定块），并标记实现风险。

## 练习

1. Transfusion 风格的模型训练了 70% 的文本词元和 30% 的图像块。图像扩散损失的幅度约是文本 NTP 损失的 10 倍。什么样的损失权重能平衡它们？

2. 为序列 `[T, T, <image>, P, P, P, P, </image>, T]` 实现块三角掩码，将每个条目标记为 0 或 1。

3. MMDiT 有模态特定的 QKV 权重，与 Transfusion 的完全共享 Transformer 相比，这增加了多少参数量开销？在 70 亿参数规模下，这值得吗？

4. 生成：给定一个文本提示，模型运行 50 步 NTP，然后遇到 `<image>`，再对 256 个块运行 20 步去噪。总共需要多少次前向传播？

5. 阅读 SD3 论文第 3 节，描述整流流以及它为何比 DDPM 在更少的推理步骤内收敛。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| 双损失训练 | "NTP + 扩散" | 单个 Transformer 在同一梯度步中同时优化文本词元的交叉熵和连续图像块的 MSE |
| 流匹配（Flow matching） | "整流流" | 预测从噪声到干净数据的速度场的扩散变体；比 DDPM 数学更简单 |
| MMDiT | "多模态 DiT" | Stable Diffusion 3 的架构：联合注意力，模态特定的 MLP 和归一化层 |
| 块三角掩码 | "因果文本 + 双向图像" | 对文本是因果的、对图像区域内是双向的注意力掩码 |
| 连续图像表示 | "无 VQ" | 图像块作为实值向量，而非整数码本索引 |
| 速度预测（Velocity prediction） | "v-参数化" | 网络输出是噪声和数据之间的速度场，而非噪声本身 |
