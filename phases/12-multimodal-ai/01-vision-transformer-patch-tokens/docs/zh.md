# 视觉 Transformer 与图像块-词元基本单元

> 在处理任何多模态任务之前，图像必须先被转换为 Transformer 可以处理的词元序列。2020 年的 ViT 论文用 16×16 像素的图像块、线性投影和位置嵌入解答了这个问题。五年后，2026 年所有前沿模型（Claude Opus 4.7 原生 2576px、Gemini 3.1 Pro、Qwen3.5-Omni）仍沿用这一方式——编码器从 ViT 演变为 DINOv2 再到 SigLIP 2，加入了寄存器词元，位置编码方案变为 2D-RoPE，但这一基本单元始终未变。本课从头到尾解读图像块-词元流水线，并用纯 Python 标准库实现，为 Phase 12 后续内容建立清晰的"视觉词元"心智模型。

**类型：** 学习
**编程语言：** Python（标准库，图像块分词器 + 几何计算器）
**前置知识：** Phase 7（Transformer）、Phase 4（计算机视觉）
**预计时间：** 约 120 分钟

## 学习目标

- 将 HxWx3 的图像转换为带有正确位置编码的图像块词元序列。
- 计算给定配置（图像块大小、分辨率、隐藏维度、层数）的 ViT 的序列长度、参数量和 FLOPs。
- 说出将 ViT 从 2020 年的研究带到 2026 年生产环境的三项升级：自监督预训练（DINO / MAE）、寄存器词元、原生分辨率打包。
- 针对下游任务在 CLS 池化、均值池化和寄存器词元之间做出合理选择。

## 问题背景

Transformer 处理的是向量序列。文本本来就是序列（字节或词元）。图像是具有三个颜色通道的二维像素网格——并不是序列。如果将每个像素展平，一张 224×224 的 RGB 图像会变成 150,528 个词元，这种长度下的自注意力根本无法运行（计算复杂度关于序列长度是二次方）。

2020 年之前的方案是在前端接一个 CNN 特征提取器：ResNet 输出 7×7 的 2048 维特征图，将这 49 个词元送入 Transformer。可行，但继承了 CNN 的归纳偏置（平移等变性、局部感受野），也丧失了 Transformer 的规模扩展优势。

Dosovitskiy 等人（2020）直接提问：如果跳过 CNN 会怎样？将图像切分为固定大小的图像块（如 16×16 像素），对每个块进行线性投影变成向量，添加位置嵌入，再将序列送入标准 Transformer。当时这是离经叛道之举——没有卷积的视觉。但在足够多的数据（JFT-300M，随后是 LAION）下，它在 ImageNet 上击败了 ResNet，且持续提升。

到 2026 年，ViT 基本单元已成为无可争议的基础。所有开放权重 VLM 的视觉塔都是其某种衍生（DINOv2、SigLIP 2、CLIP、EVA、InternViT）。问题不再是"是否使用图像块？"而是"图像块大小、分辨率策略、预训练目标、位置编码如何选择？"

## 核心概念

### 图像块即词元

给定形状为 `(H, W, 3)` 的图像 `x` 和图像块大小 `P`，将图像切割成 `(H/P) × (W/P)` 个不重叠的图像块。每个块是 `P × P × 3` 的像素立方体。将每个立方体展平为 `3P²` 维向量。应用共享线性投影矩阵 `W_E`（形状为 `(3P², D)`）将每个块映射到模型隐藏维度 `D`。

以 ViT-B/16 标准配置为例：
- 分辨率 224，图像块大小 16 → 网格 14×14 → 196 个块词元。
- 每个块有 `16 × 16 × 3 = 768` 个像素值，投影到 `D = 768`。
- 添加可学习的 `[CLS]` 词元 → 序列长度 197。

图像块投影在数学上与卷积核大小 `P`、步长 `P`、输出通道数 `D` 的 2D 卷积完全等价，生产代码也正是这样实现的——`nn.Conv2d(3, D, kernel_size=P, stride=P)`。"线性投影"是概念表述，卷积实现才是高效做法。

### 位置嵌入

图像块本身没有内在顺序——Transformer 将其视为无序集合。早期 ViT 添加了可学习的一维位置嵌入（每个位置一个 768 维向量，共 197 个）。可行，但与训练分辨率绑定：推理时若改变网格大小，需对位置表进行插值。

现代视觉骨干网络使用 2D-RoPE（Qwen2-VL 的 M-RoPE、SigLIP 2 的默认设置）或分解式 2D 位置编码。2D-RoPE 根据图像块的（行、列）索引旋转查询和键向量，模型从旋转角度中推断出相对二维位置，无需位置表，可在推理时处理任意网格大小。

### CLS 词元、池化输出与寄存器词元

图像级别的表示应该是什么？目前有三种选择并存：

1. `[CLS]` 词元：在图像块序列前添加一个可学习向量。所有 Transformer 块处理后，CLS 词元的隐藏状态即为图像表示。继承自 BERT，被原始 ViT、CLIP 采用。
2. 均值池化：对图像块词元输出隐藏状态取平均。被 SigLIP、DINOv2 以及大多数现代 VLM 采用。
3. 寄存器词元：Darcet 等人（2023）观察到，未设置显式"汇聚"词元的 ViT 会在预训练中产生高范数的"伪影"块，劫持自注意力。添加 4–16 个可学习的寄存器词元可以吸收这些负载，提升密集预测任务（分割、深度估计）的质量。DINOv2 和 SigLIP 2 均配备了寄存器词元。

选择对下游任务至关重要。CLS 适用于分类。对于将图像块词元送入 LLM 的 VLM，完全跳过池化——每个图像块直接成为 LLM 的输入词元，寄存器词元在移交前丢弃（它们是脚手架，而非内容）。

### 预训练：监督、对比、掩码、自蒸馏

2020 年的 ViT 在 JFT-300M 上进行监督分类预训练。很快被以下方法取代：

- CLIP（2021）：在 4 亿对图文数据上进行对比学习。见 12.02 课。
- MAE（2021，He 等）：遮盖 75% 的图像块，重建像素。自监督，仅需纯图像。
- DINO（2021）/ DINOv2（2023）：无标签、无描述的师生自蒸馏。2023 年的 DINOv2 ViT-g/14 是最强的纯视觉骨干，也是"密集特征"场景的默认选择。
- SigLIP / SigLIP 2（2023、2025）：使用 sigmoid 损失和 NaFlex（原生长宽比）的 CLIP。2026 年开放 VLM 中的主流视觉塔（Qwen、Idefics2、LLaVA-OneVision）。

预训练方式决定了骨干网络的适用场景：CLIP/SigLIP 适合图文语义匹配，DINOv2 适合密集视觉特征，MAE 适合用于下游微调的起点。

### 缩放规律

ViT 缩放（Zhai 等，2022）证明了 ViT 的质量遵循可预测的模型规模、数据规模和计算量规律。在固定计算量下：
- 更大的模型 + 更多的数据 → 更好的质量。
- 图像块大小是序列长度与保真度之间的杠杆。Patch 14（DINOv2/SigLIP SO400m 的典型选择）比 Patch 16 每张图像产生更多词元；对 OCR 和密集任务更好，速度更慢。
- 分辨率是另一个主要杠杆。从 224 到 384 再到 512 几乎总能提升质量，但 FLOPs 呈平方增长。

ViT-g/14（10 亿参数，Patch 14，分辨率 224 → 256 词元）和 SigLIP SO400m/14（4 亿参数，Patch 14）是 2026 年开放 VLM 的两个主力编码器。

### ViT 参数量计算

详细计算见 `code/main.py`。以 ViT-B/16 @ 224 为例：

```
图像块嵌入 = 3 * 16 * 16 * 768 + 768  ≈  591K
CLS + 位置  = 768 + 197 * 768          ≈  152K
单个块      = 4 * 768² (QKVO) + 2 * 4 * 768² (MLP) + 2 * 2*768 (LN)
             = 12 * 768² + 3K           ≈  7.1M
12 个块     = 85M
最终 LN    ≈ 1.5K
总计        ≈ 86M
```

在加载检查点之前，先用这种方式估算每个 ViT 的规模，骨干网络大小决定了任何下游 VLM 的显存下限。

### 2026 年生产配置

2026 年大多数开放 VLM 采用的编码器是原生分辨率（NaFlex）的 SigLIP 2 SO400m/14，具体参数：
- 4 亿参数。
- Patch 14，默认分辨率 384 → 每张图像 729 个图像块词元。
- 图像级任务使用均值池化；VQA 时全部 729 个块送入 LLM。
- 4 个寄存器词元，移交 LLM 前丢弃。
- 具有图像级缩放的 2D-RoPE，支持原生长宽比。

该配置中的每个决策都可以追溯到一篇可读的论文。

## 动手实践

`code/main.py` 是一个图像块分词器和几何计算器，接受（图像 H、W、图像块 P、隐藏维度 D、深度 L），输出：

- 分块后的网格形状和序列长度。
- 合成 8×8 像素玩具图像的词元序列（演示展平 + 投影过程）。
- 分解为图像块嵌入、位置嵌入、Transformer 块和输出头的参数量。
- 目标分辨率下每次前向传播的 FLOPs。
- ViT-B/16 @ 224、ViT-L/14 @ 336、DINOv2 ViT-g/14 @ 224、SigLIP SO400m/14 @ 384 的对比表格。

运行并对照公开数字验证参数量，调整图像块大小和分辨率感受词元数量的变化代价。

## 产出技能

本课产出 `outputs/skill-patch-geometry-reader.md`：给定 ViT 配置（图像块大小、分辨率、隐藏维度、深度），输出带有理由的词元数量、参数量和显存估算。在为 VLM 选择视觉骨干时使用这项技能，可以防止"词元数量爆炸导致 LLM 上下文溢出"的意外。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| 图像块（Patch） | "16×16 像素正方形" | 输入图像的固定大小非重叠区域；变成一个词元 |
| 图像块嵌入（Patch embedding） | "线性投影" | 将展平的块像素映射到 D 维向量的共享学习矩阵（或步长为 P 的 Conv2d） |
| CLS 词元 | "类别词元" | 预置的可学习向量，其最终隐藏状态代表整张图像；2026 年可选 |
| 寄存器词元（Register token） | "汇聚词元" | 额外的可学习词元，吸收 ViT 预训练中产生的高范数注意力伪影 |
| 位置嵌入（Position embedding） | "位置信息" | 使序列感知顺序的每位置向量或旋转；2D-RoPE 是现代默认选择 |
| 网格（Grid） | "图像块网格" | 给定分辨率和图像块大小的 (H/P) × (W/P) 二维块数组 |
| NaFlex | "原生灵活分辨率" | SigLIP 2 特性：单一模型无需重训练即可处理多种长宽比和分辨率 |
| 骨干网络（Backbone） | "视觉塔" | VLM 中预训练图像编码器，其图像块词元输出送入 LLM |
| 池化（Pooling） | "图像级摘要" | 将图像块词元转为单一向量的策略：CLS、均值、注意力池化或基于寄存器 |
| Patch 14 vs 16 | "更细 vs 更粗的网格" | Patch 14 每张图像产生更多词元，OCR 保真度更高，速度更慢；Patch 16 是经典默认值 |
