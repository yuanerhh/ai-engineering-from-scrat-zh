# LLaVA-OneVision：单图像、多图像、视频三合一模型

> 在 LLaVA-OneVision（Li 等，2024 年 8 月）出现之前，开放 VLM 领域各自为营：LLaVA-1.5 专注单图像，Mantis 和 VILA 面向多图像，Video-LLaVA 和 Video-LLaMA 处理视频。每个模型赢得各自的基准，却在其他场景上失败。LLaVA-OneVision 论证了：单一课程训练可以让一个模型在三种场景下都达到领先水平，而且涌现的跨场景迁移效果（单图像技能迁移到视频，多图像推理迁移到单图像）超越了专项模型之和。配方出奇地简单：在所有场景中保持恒定的视觉词元预算，加上一个明确的从单图像 → 多图像（OneVision）→ 视频的课程训练顺序。本课解读预算设计、课程安排和涌现行为。

**类型：** 构建
**编程语言：** Python（标准库，词元预算求解器 + 课程规划器）
**前置知识：** Phase 12 · 05（LLaVA）、Phase 12 · 06（任意分辨率）
**预计时间：** 约 180 分钟

## 学习目标

- 设计一个在单图像、多图像和视频输入中保持恒定的视觉词元预算。
- 制定一个课程训练顺序，将技能从单图像迁移到视频，同时不发生灾难性遗忘。
- 解释为何在参数量相同时，课程设计正确的单一模型优于专项模型。
- 说出 LLaVA-OneVision 报告的三种涌现能力：多摄像头推理、标记点提示、iPhone 截图智能体。

## 问题背景

图像、多图像和视频对模型的要求各不相同。

单图像需要高分辨率词元（AnyRes，约 2880 个视觉词元）以捕捉 OCR 和细节，每样本预算：一张图像，2880 词元。

多图像需要中等分辨率下的多张图像（每张约 576 词元），以便跨图像推理能放入上下文。每样本预算：4–8 张图像，每张 576 词元，共 2300–4600 词元。

视频需要大量低分辨率帧（池化后每帧约 196 词元）以捕捉时序动态。每样本预算：8–32 帧，每帧 196 词元，共 1600–6200 词元。

如果训练独立模型，就只能选择一种预算。如果训练单一模型，预算就需要跨场景合理扩展，同时不撑爆上下文。

OneVision 之前，默认答案是"专注一个场景，忽略其他"。Video-LLaVA 在图像模型上额外训练阶段来增加视频支持；LLaVA-NeXT 通过分块拼接增加了多图像支持；没有一个能干净地处理三种场景。

## 核心概念

### OneVision 词元预算

LLaVA-OneVision 选择了每样本约 3000–4000 词元的统一视觉词元预算，在不同场景下以不同方式分配：

- 单图像：AnyRes-9（3×3 格 + 缩略图），每格在 384 分辨率下有 729 个块，激进的 2×2 双线性池化后每格 182 词元。总计：9 × 182 + 182 = 1820 词元。或 AnyRes-4 每格 729 词元 = 2916 + 729。
- 多图像：每张图像在中等分辨率（384，无分块拼接）下 729 词元，不做池化。6 张图像的预算 → 4374 词元。
- 视频：32 帧，384 分辨率，激进的 3×3 双线性池化后每帧 81 词元。总计：32 × 81 = 2592 词元。

这种分配保持总词元数大致恒定。LLM 看到的批次永远不会超出上下文。不同场景下编码器产生不同的几何形状，但 LLM 消费的是相同的预算。

### 三阶段课程

LLaVA-OneVision 分三个阶段训练：

1. **单图像 SFT（SI 阶段）**：所有数据都是单图像加文本，在高分辨率 AnyRes 输入上训练，培养感知、OCR 和细粒度理解能力。使用 LLaVA-NeXT 数据加 OneVision 特有的单图像数据。
2. **OneVision SFT（OV 阶段）**：混合单图像 + 多图像 + 视频（均匀采样帧），在统一词元预算上训练，使模型能够处理异构批次形状。不重置权重，从 SI 阶段继续训练。
3. **任务迁移（TT 阶段）**：继续使用目标任务混合，根据产品需求通常加重多图像或视频比例。可选的部署前微调。

关键：课程顺序至关重要。先训视频或先训多图像，都会导致比先训单图像更差的图像性能，即使使用相同的数据量。论文明确消融了这一点。

### 课程为何有效

单图像训练构建感知基础。块词元携带细粒度视觉特征，LLM 学会将其与文本整合。多图像和视频引入了结构性挑战（哪张是哪张图像，先发生了什么），这些挑战在没有强感知基础的情况下很难学习。

如果从头开始混合训练所有场景，模型会欠拟合感知（每批单图像数据有限）并过拟合结构（大量多图像/视频数据）。结果：一个擅长跨图像推理模式但视觉理解较浅的模型。

课程排序使你从 SI 阶段获得强感知能力，再从 OV 阶段获得组合/时序推理能力，两者都不丢失。

### 涌现的跨场景技能

LLaVA-OneVision 论文报告了三种涌现能力：

1. **多摄像头推理**：分别在多图像和视频上训练；推理时要求对多摄像头驾驶场景进行推理。模型正确整合了各视角，尽管训练时从未见过这种确切格式。
2. **标记点提示**：用户在图像中用数字标记物体，模型推理"标记 3 相对于标记 7 在做什么"。既没有在标记数据上训练，也没有在标注数据上训练；通过空间定位 + 多图像参考的组合习得。
3. **iPhone 截图智能体**：用户提供 iPhone 屏幕截图并要求规划下一步点击操作。在 UI 截图、用户工作流视频和多图像前后对比上训练，泛化到智能体使用场景。

这些都不是直接训练的任务，而是从课程的组合结构中涌现出来的。

### 视觉词元池化

词元预算要求进行池化。OneVision 在二维块网格上使用双线性插值：24×24 = 576 个块变为 12×12 = 144（2 倍因子）或 8×8 = 64（3 倍因子）。池化在块网格空间中进行，而非词元空间，以保持局部性。

每个场景的池化因子选择本身是一个超参数。较少池化 = 更多词元 = 更丰富的表示；更多池化 = 更少词元 = 可以容纳更多帧/图像。

### LLaVA-OneVision-1.5

2025 年的后续版本（LLaVA-OneVision-1.5，arXiv 2509.23661）在训练数据、模型权重和代码上都实现了"完全开放"。在某些基准上缩小了与专有模型的差距，使该配方得到普及。相同的课程，更多的数据，更好的基础 LLM，无架构变化。

### 与 Qwen2.5-VL 的对比

Qwen2.5-VL（12.09 课）做出了不同的选择：使用 M-RoPE 和动态 FPS，而非固定池化。其预算随输入扩展——一分钟的视频比五秒钟的视频使用更多词元。LLaVA-OneVision 固定预算，调整池化比例。两种方法都有效，它们在可配置性和可预测性之间做出了权衡。

## 动手实践

`code/main.py` 是一个 OneVision 风格 VLM 的课程和预算规划器。给定每样本词元预算和目标场景混合（如单图像 40%、多图像 30%、视频 30%），它会：

- 为每个场景分配分辨率、池化因子和帧数。
- 验证每个场景都在共享预算范围内。
- 报告预期词元数量、LLM FLOPs 以及哪些场景词元不足。
- 打印分阶段训练计划。

在规划 OneVision 微调或检查 VLM 部署的每请求成本时使用它。

## 产出技能

本课产出 `outputs/skill-onevision-budget-planner.md`：给定目标任务分布和每样本预算，输出 AnyRes 因子、每帧池化比例、视频帧数和课程阶段权重。在训练或微调统一场景 VLM 时使用。

## 练习

1. 你的产品支持 80% 单图像、10% 多图像（2–4 张）、10% 视频（8–16 帧）。设计词元预算，节省下来的预算（不必重度支持多图像）应该放到哪里？

2. 阅读 LLaVA-OneVision 第 4.3 节（涌现能力）。提出一种课程可能解锁但论文没有报告的第四种涌现技能。

3. 调换课程顺序——先训多图像，再训单图像，再训视频。预测哪些基准会下降以及原因。

4. 论文报告了仅用每样本 8 帧训练的视频基准。这能泛化到推理时的 30 秒视频吗？词元预算和时序推理哪个先出问题？

5. 将 24×24 块双线性池化到 12×12 是每维 2 倍的缩减。用标准库 Python 实现池化，验证每个 2×2 块的均值与双线性输出一致。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| OneVision 场景 | "单图像、多图像或视频" | 统一 VLM 处理的三种输入形状之一；预算在三者之间保持恒定 |
| 词元预算 | "每样本多少词元" | LLM 每个训练/推理样本看到的视觉词元总数，通常为 3000–4000 |
| 课程（Curriculum） | "训练顺序" | 为实现涌现迁移而选择的阶段顺序（单图像 → 多图像 → 视频） |
| 双线性池化 | "词元压缩" | 对块网格（二维）应用双线性插值以减少词元数量，同时保持局部性 |
| 涌现技能 | "没训过却会了" | 推理时出现的能力，没有匹配的训练数据，源于课程的组合结构 |
| AnyRes-k | "k 格方案" | k 个固定分辨率子格加一个缩略图，典型的 k ∈ {4, 9} |
| 任务迁移 | "跨场景泛化" | 在单图像上习得的技能应用到视频（反之亦然），通过共享骨干实现 |
