# InternVL3：原生多模态预训练

> InternVL3 之前的每个开放 VLM 都遵循相同的三步配方：取一个在数万亿文本词元上训练的文本 LLM，接上视觉编码器，然后微调连接部分。这个方法可行，但会产生对齐债务——文本 LLM 的全部预训练预算都花在了纯文本上，并不能原生理解视觉词元。事后添加视觉时，LLM 必须重新学习如何将视觉输入与其文本推理联系起来，同时还不能遗忘已有的文本能力。InternVL3（Zhu 等，2025 年 4 月）拒绝了事后方案：一次预训练，文本和多模态数据从第一步起就交错进行。结果是在 780 亿参数规模上以开放模型匹敌 Gemini 2.5 Pro 的 MMMU-Pro 得分。本课解读原生预训练的理由，以及这样做后发生了哪些变化。

**类型：** 学习
**编程语言：** Python（标准库，训练语料混合器）
**前置知识：** Phase 12 · 05、Phase 12 · 07（配方）
**预计时间：** 约 120 分钟

## 学习目标

- 解释为何事后 VLM 训练会积累对齐债务，引用三个可量化的症状（灾难性遗忘、答案漂移、视觉-文本不一致性）。
- 描述 InternVL3 的原生预训练语料混合方案，解释为何文本:交错:描述的比例至关重要。
- 对比 V2PE（可变视觉位置编码）与 Qwen2-VL 的 M-RoPE。
- 说出视觉分辨率路由器（ViR）和解耦视觉-语言（DvD）部署优化方案。

## 问题背景

事后 VLM 训练是默认方案。LLaVA、BLIP-2、Qwen-VL、Idefics——所有这些模型都取一个已经预训练的 LLM（Llama、Vicuna、Qwen、Mistral）并添加视觉能力。训练阶段通常如下：

1. 冻结 LLM + 冻结视觉编码器 + 可训练投影器，在描述对上训练以对齐嵌入。
2. 解冻 LLM，在指令数据上训练（LLaVA-Instruct、ShareGPT4V）。
3. 可选的任务特定微调。

对齐债务有三个症状：

- **灾难性遗忘**：事后 VLM 忘记了纯文本技能。GSM8K 分数下降 5–10 分，Hellaswag 下降，纯文本智能体退化。
- **答案漂移**：对同一视觉问题的细微措辞不同导致不同答案。视觉编码器与 LLM 的连接比 LLM 自身词元的绑定更弱。
- **视觉-文本不一致性**：VLM 能正确描述图像，却随后给出与自身描述相矛盾的答案。视觉词元不像文本词元那样参与 LLM 的内部一致性检查。

这些症状有充分记录。MM1.5 第 4 节进行了量化，LLaVA-OneVision 的消融实验也有所暗示。原生预训练是解答。

## 核心概念

### 原生多模态预训练

InternVL3 在一个从第一步起就原生多模态的语料上从零开始训练。混合方案如下：

- 40% 纯文本数据（FineWeb、Proof-Pile-2 等）
- 35% 交错图文数据（OBELICS、MMC4 风格）
- 20% 图文配对描述数据
- 5% 视频-文本数据

视觉词元、文本词元和跨模态交互从第一个梯度步开始就参与同一个损失。没有对齐预训练，没有投影器冻结阶段，没有需要从灾难性遗忘中恢复的过程。

基础模型的训练是单阶段的，随后进行指令微调，但基础模型从一开始就将视觉词元视为一等公民。

### V2PE（可变视觉位置编码）

Qwen2-VL 使用固定轴分配的 M-RoPE。InternVL3 引入了 V2PE：每种模态类型（文本、图像、视频）的位置编码各不相同，且有可学习的缩放比例。实践中：

- 文本词元获得一维位置（文本索引）。
- 图像块获得二维位置（行、列）。
- 视频帧获得三维位置（时间、行、列）。

三者共享相同的 RoPE 频率基础，但每段隐藏维度的分配是可学习参数，而非固定分割。在预训练期间可以自由权衡时间与空间频率分辨率。

V2PE 的消融声称：在相同计算量下，视频基准比 M-RoPE 高 1–2 分。不是革命性的改进，但更简洁。

### 视觉分辨率路由器（ViR）

部署优化。并非所有图像都需要全分辨率编码。一张包含低细节单个物体的照片，以 1280px 原生分辨率编码时浪费词元。ViR 是一个小型分类器，在编码之前预测回答该问题所需的最低分辨率。

路由有三个档次：低分辨率（256 词元）、中等（576）、高（2048+）。在生产流量中，60% 的查询使用低分辨率或中等分辨率就已足够。净效果：在质量相等的情况下，吞吐量提升 2–3 倍。

### 解耦视觉-语言部署（DvD）

在服务大型 VLM 时，视觉编码器每张图像只运行一次，而 LLM 对每个输出词元都要自回归运行。两个组件有不同的瓶颈（视觉 = 卷积 + 注意力的 GPU 内存带宽；LLM = KV 缓存）。DvD 将它们部署在不同 GPU 上，通过流式传输连接。

对于 8B + 4 亿编码器模型，DvD 与同机部署相比，每节点吞吐量大约翻倍。

### 单阶段 vs 多阶段质量

InternVL3 的主要基准声明：780 亿参数规模，MMMU-Pro 与 Gemini 2.5 Pro 持平；380 亿参数，与 GPT-4o 持平；80 亿参数，领跑开放 8B 排行榜。所有这些都基于单阶段预训练 + 指令微调的配方。

对齐债务假设是可量化的：InternVL3-8B 每获得一分视觉基准增益所损失的文本基准分数（MMLU、GSM8K）少于 Qwen2.5-VL-7B。模型更擅长泛化，因为训练是整体的，而非分两步进行。

### InternVL3.5 与 InternVL-U

InternVL3.5（2025 年 8 月）扩展了该配方：相同的原生预训练方法，更多数据，更多参数。MMMU 改进是增量性的。

InternVL-U（2026 年）增加了统一生成——在同一骨干网络上通过 MMDiT 头支持图像输出。"U"代表"理解 + 生成"，追求 Transfusion 风格的统一模型（12.13 课）。相同的原生预训练骨干同时支持理解和生成头。

### 原生预训练的权衡

原生预训练并非没有代价：

- **计算成本**：从零训练一个新 VLM 的成本与训练一个文本 LLM 相同——数百万 GPU 小时。事后适应复用了现有 LLM 权重，节省了大部分成本。
- **数据**：大规模的交错图文语料库很稀缺。OBELICS 有 1.41 亿文档，MMC4 有 5.71 亿，而纯文本数据有 15 万亿词元。多模态预训练数据稀缺是个硬性约束。
- **基础 LLM 复用**：原生预训练放弃了之后更换新 LLM 的选项。事后方式允许你通过只重训适配器来将 Llama-3.1 换成 Llama-4。

InternVL3 的赌注：对齐债务比复用损失更严重。基准测试支持这一声明。生产成本的高门槛使未来的实验室难以廉价复制。事后 VLM 将继续存在，因为对大多数项目来说它们仍然更便宜。

## 动手实践

`code/main.py` 是一个训练语料混合器和 ViR 路由器模拟器，功能包括：

- 接受目标语料混合（%文本、%交错、%描述、%视频）并计算每种模态的预期步数。
- 在一批查询（分布：50% 低细节、30% 中等、20% 高细节）上模拟 ViR 路由，报告平均词元数。
- 根据编码器 vs LLM FLOPs 给出 DvD 吞吐量估算。
- 打印事后 vs 原生预训练在参数、计算、数据和预期对齐债务症状上的对比。

## 产出技能

本课产出 `outputs/skill-native-vs-posthoc-auditor.md`：给定提议的 VLM 训练计划，审核是选择原生预训练还是事后训练，标记对齐债务风险，并推荐语料混合方案。在规划新开放 VLM 项目且需要选择训练策略时使用。

## 练习

1. 估算 InternVL3-8B（原生预训练）和 LLaVA-OneVision-7B（事后训练）之间的计算差距，GPU 小时之比大约是多少？差距由什么解释？

2. InternVL3 报告 40% 文本 / 35% 交错 / 20% 描述 / 5% 视频。如果目标任务以视频为主，提出新的比例，并说明为何基础模型仍需要大量文本和描述数据。

3. 阅读 MM1.5 第 4 节关于遗忘的内容。说出事后训练造成最大回退的基准。回退损失了多少分？

4. ViR 将 60% 的流量路由到低分辨率编码。哪类查询会被错误路由（本应使用高分辨率却使用了低分辨率）？提出三种路由器失效模式。

5. DvD 将视觉和 LLM 分到不同 GPU。在什么流量模式下，DvD 会降低而非提升吞吐量？

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| 原生多模态预训练 | "从零一起训" | 文本 + 图像 + 视频词元从第 1 步就参与损失，而非事后接入 |
| 对齐债务（Alignment debt） | "事后惩罚" | 将视觉接入冻结 LLM 所带来的可量化的文本技能退化和答案一致性下降 |
| V2PE | "可变视觉位置编码" | 每种模态可学习的位置编码分配；InternVL3 对 M-RoPE 的改进 |
| ViR | "分辨率路由器" | 在编码前按查询选择所需最低分辨率的小型分类器，节省推理词元 |
| DvD | "解耦部署" | 视觉编码器在一块 GPU 上，LLM 在另一块，流式传输；大型 VLM 吞吐量翻倍 |
| InternVL-U | "统一理解 + 生成" | 2026 年的后续版本，在原生预训练骨干上增加图像生成头 |
| 交错语料（Interleaved corpus） | "OBELICS / MMC4" | 按自然阅读顺序包含文本和图像的文档；原生预训练的原材料 |
