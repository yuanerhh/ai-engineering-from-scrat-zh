# Show-o 与离散扩散统一模型

> Transfusion 混合了连续表示和离散表示。Show-o（Xie 等，2024 年 8 月）走向了另一条路：文本词元使用因果下一词元预测，图像词元使用 MaskGIT 精神下的掩码离散扩散，两者都在一个带混合注意力掩码的 Transformer 中运行。结果在一个骨干网络、每种模态一个分词器、一种损失形式（将下一词元预测扩展到掩码预测）上统一了 VQA、文到图生成、图像修复和混合模态生成。本课解读 Show-o 的设计——为何掩码离散扩散是一种并行、少步骤的图像生成器——并与 Transfusion 和 Emu3 做对比。

**类型：** 学习
**编程语言：** Python（标准库，掩码离散扩散采样器）
**前置知识：** Phase 12 · 13（Transfusion）
**预计时间：** 约 120 分钟

## 学习目标

- 解释掩码离散扩散：均匀掩码词元然后要求 Transformer 恢复它们的调度。
- 在速度和质量上对比并行图像解码（Show-o、MaskGIT）与自回归图像解码（Chameleon、Emu3）。
- 说出 Show-o 在一个检查点中处理的三种任务：T2I（文到图）、VQA、图像修复。
- 选择一种掩码调度（余弦、线性、截断），并分析其对样本质量的影响。

## 问题背景

Transfusion 的双损失训练有效，但动态更复杂——连续扩散损失与离散 NTP 损失的数值尺度不同。平衡损失权重是一项超参数搜索，架构有效但复杂。

Show-o 的答案：保持两种模态都是离散的（与 Chameleon 相同），但通过掩码离散扩散并行生成图像，而非顺序生成。训练目标变为单一的掩码词元预测，自然地推广了下一词元预测。

## 核心概念

### 掩码离散扩散（MaskGIT）

Chang 等（2022）的原始 MaskGIT 技巧优雅简洁：从一张完全掩码的图像（每个词元都是特殊的 `<MASK>` ID）开始；在每一步，并行预测所有掩码词元，保留置信度最高的前 K 个预测，将其余重新掩码；约 8–16 次迭代后，所有词元都被填充完毕。每步解除多少词元掩码的调度需要调优——余弦调度效果良好。

训练很简单：从 [0, 1] 均匀采样一个掩码比例，将其应用到图像的 VQ 词元上，训练 Transformer 恢复掩码的词元。正是 BERT 为文本做的事情，扩展到了图像生成。

### Show-o：一个 Transformer，混合掩码

Show-o 将 MaskGIT 放入因果语言模型 Transformer 中，注意力掩码为：

- 文本词元：因果（标准 LLM）。
- 图像词元：图像块内完全双向（掩码词元预测时可以看到所有其他图像词元）。
- 文到图：文本聚合前面的图像，图像聚合前面的文本。

训练在以下任务间交替：
1. 文本序列上的标准 NTP。
2. T2I 样本：文本 → 掩码图像词元上的掩码词元预测损失。
3. VQA 样本：图像 → 掩码文本词元（实际上就是 NTP）。

统一损失是对 `<MASK>` 词元的交叉熵，覆盖了文本 NTP（只有最后一个词元被"掩码"）和图像掩码扩散（随机子集被掩码）。

### 并行采样

Show-o 在约 16 步内生成图像，而非约 1000 步（逐词元自回归）或约 20 步（扩散）。每步并行预测所有掩码词元，提交置信度最高的前 K 个，然后重复。

对比：
- Chameleon / Emu3（逐词元自回归）：N_tokens 次前向传播，图像通常 1024–4096 次。
- Transfusion（连续扩散）：约 20 步，每步一次完整 Transformer 传播。
- Show-o（掩码离散扩散）：约 16 步，每步一次完整 Transformer 传播。

Show-o 比相同规模的 Chameleon 更快，每步成本低于 Transfusion（离散词汇 logit vs 连续 MSE 损失），步数与 Transfusion 大致相当。

### 一个检查点处理四种任务

Show-o 在推理时支持四种任务，通过提示格式选择：

- 文本生成：标准自回归文本输出。
- VQA：图像输入，文本输出。
- T2I：文本输入，通过掩码离散扩散输出图像。
- 图像修复：带有部分词元被掩码的图像，填充缺失部分。

图像修复能力来自掩码预测训练：掩码 VQ 词元网格的某个区域，提供其余部分加文本提示，预测被掩码的词元。

### 掩码调度

每步解除多少词元掩码的调度决定了生成质量。Show-o 推荐余弦调度：

```
mask_ratio(t) = cos(pi * t / (2 * T))   # t = 0..T
```

步骤 0 时，所有词元被掩码（比例 1.0）；步骤 T 时，无词元被掩码。余弦调度将概率质量集中在预测最具信息量的中等掩码比例上，线性调度也有效，但更快达到瓶颈。

### Show-o2

Show-o2（2025 年后续版本，arXiv 2506.15564）扩展了 Show-o：更大的 LLM 基础、更好的分词器、改进的掩码调度，架构模式相同。

### Show-o 在 2026 年分类法中的位置

- 离散词元 + NTP：Chameleon、Emu3。简单但推理慢。
- 离散词元 + 掩码扩散：Show-o、MaskGIT、LlamaGen、Muse。并行采样，仍受分词器的有损性限制。
- 连续 + 扩散：Transfusion、MMDiT、DiT。最高质量，训练更复杂。
- VLM 中的连续 + 流匹配：JanusFlow、InternVL-U。最新方向。

按任务选择：当你需要在一个开放模型中以合理速度实现 T2I + 修复 + VQA 时，选 Show-o；当质量是最重要的且你可以承受双损失管道复杂性时，选 Transfusion。

## 动手实践

`code/main.py` 模拟了 Show-o 采样过程：

- 包含 16 个 VQ 词元的玩具网格。
- 一个根据提示和当前未掩码词元预测 logit 的模拟"Transformer"。
- 在余弦调度下 8 步并行掩码采样。
- 打印中间状态（掩码模式的演变）和最终词元。

运行它，观察掩码逐步消散的过程。

## 练习

1. 掩码离散扩散在约 16 步内完成采样。为什么不是 1 步？如果在第 0 步解除所有词元的掩码会发生什么？

2. 掩码扩散免费提供了图像修复能力。提出一个（真实或假设的）产品用例，说明 Show-o 的修复能力优于专项模型的场景。

3. 余弦调度 vs 线性调度：追踪 T=8 时每步解除掩码的词元数量，哪种更均衡？

4. 一张 512×512 的 Show-o 图像包含 1024 个词元。词汇表 K=16384，模型输出 1024 × log₂(16384) = 14336 位（约 1.75 KiB）数据。Stable Diffusion 输出 512×512×24 位 = 6291456 位（约 768 KiB）的原始像素。压缩比是多少？这带来了什么质量？

5. 阅读 LlamaGen（arXiv:2406.06525）。LlamaGen 的类别条件自回归图像模型与 Show-o 的掩码方法有何不同？

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| 掩码离散扩散 | "MaskGIT 风格" | 训练预测被掩码的词元；推理时迭代解除最置信预测的掩码 |
| 余弦调度 | "解除掩码调度" | 推理步骤中掩码比例的衰减曲线；将置信度增长集中在中等比例 |
| 并行解码 | "同时处理所有词元" | 每步在一次前向传播中预测完整的掩码词元序列，然后提交前 K 个 |
| 混合注意力 | "因果 + 双向" | 对文本词元是因果的，对图像块内是双向的掩码 |
| 图像修复（Inpainting） | "填充生成" | 以带部分掩码词元的图像为条件，预测缺失的词元；训练目标自然赠送的能力 |
| 提交率（Commitment rate） | "每步前 K 个" | 每次迭代中被宣告"完成"的词元数量；控制推理速度与质量的权衡 |
