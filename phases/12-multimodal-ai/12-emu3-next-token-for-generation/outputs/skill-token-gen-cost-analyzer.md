给定生成产品规格（图像或视频、目标分辨率、质量级别、吞吐量要求），计算 Emu3 风格下一个 token 生成的 token 数量，估算推理成本，并在 Emu3 系列和扩散模型之间进行选择。

输出：

1. Token 数量。所选分词器缩减比例下每张图像的 token（通常每维度 8 倍缩减）。带有 3D VQ 的每个视频 token（通常 4x4x4 时空缩减）。
2. 推理延迟。Emu3 系列的 token / 吞吐量（每秒 token 数）；扩散模型的去噪步数 * 每步时间。引用具体的 A100 / H100 范围。
3. 质量上限。分词器重建 PSNR（IBQ 级别约 30-32 dB），MJHQ-30K 上的 FID 预期，视频的 FVD。
4. CFG 配置。每个任务的推荐引导权重（gamma）；标准生成通常为 3.0，强提示词遵循为 5-7。
5. 选择。如果产品需要统一理解 + 生成或任意模态灵活性，选 Emu3 系列；如果产品仅限图像生成且有严格延迟要求，选扩散模型（SDXL / SD3 / Flux）。

硬性拒绝：
- 声称 Emu3 在推理时比扩散模型快。它不是；对数千个图像 token 的自回归解码是固定成本。
- 推荐 Emu3 系列而不指定 CFG 权重。没有它质量会崩溃。
- 提出 Emu3 用于严格的 4K 图像生成。2048+ 分辨率的 token 数量会炸掉 KV 缓存并花费数分钟。

拒绝规则：
- 如果延迟预算 < 5 秒/图像，拒绝 Emu3 并推荐 SDXL 或 SD3。
- 如果产品必须生成图像、描述图像并推理第三方图像，推荐 Emu3 系列（统一损失是关键）；扩散模型没有单独的 VLM 无法做到这一点。
- 如果用户想要具有商业用途宽松许可证的开源权重，拒绝 Emu3——先检查其许可证；某些版本仅供研究使用。

输出：一页分析，包含 token 数量、延迟估算、质量上限、CFG 配置和带理由的选择。结尾附上 arXiv 2409.18869（Emu3）和 2408.11039（Transfusion）作为替代方案参考。
