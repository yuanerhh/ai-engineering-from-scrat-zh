# 从 CLIP 到 BLIP-2：Q-Former 作为模态桥接器

> CLIP 能对齐图像与文字，但无法生成描述、回答问题或进行对话。BLIP-2（Salesforce，2023）用一个小型可训练桥接器解决了这个问题：32 个可学习查询向量通过交叉注意力机制聚合冻结 ViT 的特征，然后直接插入冻结 LLM 的输入流。1.88 亿参数的桥接器将一个 110 亿参数的 LLM 与 ViT-g/14 连接起来。2026 年之前所有基于适配器的 VLM——MiniGPT-4、InstructBLIP、LLaVA 的各种变体——都是其后代。本课解读 Q-Former 的架构，解释其两阶段训练，并构建一个玩具版本，将视觉词元送入冻结的文本解码器。

**类型：** 构建
**编程语言：** Python（标准库，交叉注意力 + 可学习查询演示）
**前置知识：** Phase 12 · 02（CLIP）、Phase 7（Transformer）
**预计时间：** 约 180 分钟

## 学习目标

- 解释为何在冻结视觉编码器和冻结 LLM 之间放置可训练瓶颈，在成本和稳定性上优于端到端微调。
- 实现一个交叉注意力块，其中固定数量的可学习查询聚合外部图像特征。
- 逐步解读 BLIP-2 的两阶段预训练：表示学习（ITC + ITM + ITG）然后生成式学习（带冻结解码器的语言模型损失）。
- 对比 Q-Former 与 LLaVA 所用的简单 MLP 投影器，说明各自在何种情况下更优。

## 问题背景

你有一个冻结的 ViT，每张图像产生 256 个维度为 1408 的图像块词元；你有一个冻结的 7B LLM，期望维度为 4096 的词元嵌入。最直接的桥接方式——一个从 1408 到 4096 的线性层——可行，但将全部 256 个块词元送入 LLM 上下文，每张图像要占用 256 个额外词元。一个 32 张图像的批次，光视觉模态就消耗了 8192 个词元。

BLIP-2 的问题是：能否将 256 词元的图像表示压缩到更少的词元（比如 32 个），同时保留足够信息让 LLM 进行描述生成、问答和图像推理？还能在不触动冻结骨干网络的情况下训练这个桥接器，将训练成本控制在桥接器参数范围内？

答案：Q-Former。32 个可学习的"查询"向量对 ViT 的块词元做交叉注意力，生成 LLM 消费的 32 词元视觉摘要，总参数 1.88 亿。在接触 LLM 之前，先用对比、匹配和生成三个目标进行训练。

## 核心概念

### 可学习查询

Q-Former 的核心技巧：不让 LLM 的文本词元直接聚合图像块，而是引入一组 32 个可学习查询向量 `Q`，让*它们*聚合图像块。这些查询是模型的参数——在训练中习得，对每张图像使用同一组 32 个查询。

交叉注意力后，每个查询都持有图像的压缩摘要——"描述主要物体"、"描述背景"、"数数物体"等。查询并不按字面意义专门化到语义标签上；它们学习任何能使下游损失下降的编码。

### 架构

Q-Former 是一个小型 Transformer（12 层，约 1 亿参数），有两条路径：

1. 查询路径：32 个查询向量依次经过自注意力（相互之间）、对冻结 ViT 块词元的交叉注意力，然后经过 FFN。
2. 文本路径：类 BERT 的文本编码器与查询路径共享自注意力和 FFN 权重。文本路径禁用交叉注意力。

训练时两条路径同时运行。查询和文本通过共享的自注意力相互作用，这意味着查询可以在需要时（ITM、ITG）基于文本进行条件化。推理时用于 VLM 移交，只有查询路径运行，输出 32 个视觉词元。

### 两阶段训练

BLIP-2 分两个阶段预训练：

**阶段 1：表示学习（不涉及 LLM）**，三个损失：
- ITC（图文对比）：CLIP 风格的对比，在池化查询词元与文本 CLS 词元之间进行。
- ITM（图文匹配）：二元分类器——这个图文配对是否匹配？进行硬负样本挖掘。
- ITG（图像引导文本生成）：文本上的因果语言模型头，以查询为条件。强迫查询编码可供文本生成解码的内容。

只有 Q-Former 训练，ViT 冻结，不涉及 LLM。

**阶段 2：生成式学习**。接入冻结的 LLM（OPT-2.7B 或 Flan-T5-XL 等）。通过一个小型线性层将 32 个查询输出投影到 LLM 的嵌入维度。将其前置到文本提示前。仅训练线性投影和 Q-Former，在拼接的提示 + 图像 + 描述序列上使用语言模型损失。

阶段 2 结束后，Q-Former + 投影层即为完整的视觉适配器。推理流程：图像 → ViT → Q-Former → 线性投影 → 前置到文本 → 冻结 LLM 输出结果。

### 参数经济

BLIP-2 配置：ViT-g/14（11 亿，冻结）+ OPT-6.7B（67 亿，冻结）+ Q-Former（1.88 亿，训练）= 总计约 80 亿，训练参数仅 1.88 亿。Q-Former 约占完整栈参数的 2.4%，训练成本与之相符：几块 A100 跑数天，而非端到端训练的数周。

质量：BLIP-2 在零样本 VQA 上与 Flamingo-80B 持平甚至超越，而规模小 50 倍。桥接器有效。

### InstructBLIP 与指令感知 Q-Former

InstructBLIP（2023）在 Q-Former 中额外引入了指令文本本身。交叉注意力时，查询同时可以访问图像块和指令，从而针对每条指令专门化（"数数车辆"、"描述情绪"），而不是学习单一固定摘要。在 held-out 任务上的基准表现有所提升。

### MiniGPT-4 与仅投影器的方案

MiniGPT-4 保留了 Q-Former，但仅训练输出线性投影，其余全部冻结。成本低，但代价是质量——使用的查询是 BLIP-2 的，而非自己训练的。适合快速迭代，但不是最优架构。

### LLaVA 为何选择更简单的方案

LLaVA（2023，12.05 课）用一个简单的 2 层 MLP 替换了 Q-Former，将每个 ViT 块词元投影到 LLM 空间——24×24 网格对应 576 个词元，全部送入 LLM。压缩率更低，但让 LLM 可以在原始块上进行注意力。当时这颇具争议；到 2023 年底，随着视觉指令数据（LLaVA-Instruct-150k）证明 MLP 可以训练到保留足够信号，这一方案成为主流。权衡在于：LLaVA 的上下文填充得更快，但天然适合多图像和视频场景。

到 2026 年，领域分化：Q-Former 在词元预算受限时（长视频、多图像）仍存活；MLP 投影器在单词元质量优先时占主导。

### 门控交叉注意力：前身 Flamingo

Flamingo（12.04 课）早于 BLIP-2，也使用了相同的交叉注意力思路，但在冻结 LLM 的每一层都插入，而非作为单一桥接器。BLIP-2 表明，仅在输入层压缩也同样有效。Gemini 和 Idefics 结合了两种方式：交错输入词元加可选的门控交叉注意力，用于上下文少样本学习。

### 2026 年的后代

- Q-Former：BLIP-2、InstructBLIP、MiniGPT-4，以及大多数视频-语言模型（出于词元预算考虑）。
- Perceiver resampler：Flamingo 的变体（12.04 课）；Idefics 系列、Eagle、OmniMAE。
- MLP 投影器：LLaVA、LLaVA-NeXT、LLaVA-OneVision、Cambrian-1。
- 注意力池化：VILA、PaliGemma。

四种方案都有效，决定性问题是词元预算受限还是单词元质量优先。

## 动手实践

`code/main.py` 构建了标准库风格的 Q-Former 交叉注意力：

1. 模拟 256 个图像块词元（维度 128）。
2. 实例化 32 个可学习查询（维度 128）。
3. 运行缩放点积交叉注意力（Q 来自查询，K/V 来自块）。
4. 通过线性层投影到 LLM 维度（512）。
5. 输出 32 个 LLM 就绪的视觉词元。

所有数学运算用纯 Python（向量的嵌套循环）实现。形状上是玩具级别但完全正确，注意力权重矩阵被打印出来，可以观察每个查询从哪些块获取信息。

2. BLIP-2 阶段 1 的 Q-Former 同时运行三个损失：ITC、ITM、ITG。用伪代码写出每个损失的前向函数签名，哪一个需要文本编码器路径激活？

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| Q-Former | "查询 Transformer" | 具有 32 个可学习查询向量的小型 Transformer，通过交叉注意力聚合冻结 ViT 特征 |
| 可学习查询（Learnable queries） | "视觉软提示" | 作为交叉注意力查询端的固定参数集；每个模型学习一次，对所有输入共享 |
| 交叉注意力（Cross-attention） | "Q 来自这里，K/V 来自那里" | 查询、键、值来自不同来源的注意力；查询从 ViT 块获取信息的方式 |
| ITC | "图文对比" | 应用于 Q-Former 池化查询 vs 文本 CLS 的 CLIP 风格损失 |
| ITM | "图文匹配" | 在硬负样本挖掘对上进行的二元分类器；强迫查询区分细粒度不匹配 |
| ITG | "图像引导文本生成" | 文本以查询为条件的因果语言模型损失；强迫查询编码可供文本解码的内容 |
| 两阶段预训练 | "表示然后生成" | 阶段 1 单独训练 Q-Former（ITC/ITM/ITG）；阶段 2 接入冻结 LLM 并训练投影 + Q-Former |
| 冻结骨干（Frozen backbone） | "不微调" | 视觉编码器和 LLM 权重固定；只有桥接器训练 |
| 投影头（Projection head） | "线性到 LLM 维度" | 将 Q-Former 输出映射到 LLM 嵌入维度的最终线性层 |
| Perceiver resampler | "Flamingo 的版本" | 类似的可学习查询交叉注意力，被 Flamingo 在每一层使用而非作为单一桥接器 |
