# LLaVA 与视觉指令微调

> LLaVA（2023 年 4 月）是地球上被复制次数最多的多模态架构。它用一个 2 层 MLP 替换了 BLIP-2 的 Q-Former，用简单的词元拼接替换了 Flamingo 的门控交叉注意力，并在 GPT-4 从纯文本描述生成的 158k 条视觉指令对话上训练。2023 年至 2026 年间，任何构建 VLM 的从业者都构建了某种 LLaVA 变体。LLaVA-1.5 加入了 AnyRes；LLaVA-NeXT 提升了分辨率；LLaVA-OneVision 在一个配方中统一了单图像、多图像和视频。本课解读这个配方，实现投影器，并解释"更简单为何胜出"。

**类型：** 构建
**编程语言：** Python（标准库，投影器 + 指令模板构建器）
**前置知识：** Phase 12 · 02（CLIP）、Phase 11（LLM 工程——指令微调）
**预计时间：** 约 180 分钟

## 学习目标

- 构建一个 2 层 MLP 投影器，将 ViT 块嵌入（维度 1024）映射到 LLM 的嵌入维度（维度 4096）。
- 解读 LLaVA 两阶段配方：（1）在 558k 描述对上对齐投影器，（2）在 158k 条 GPT-4 生成对话上进行视觉指令微调。
- 构建包含图像词元占位符、系统提示和用户/助手轮次的 LLaVA 格式提示。
- 解释为何社区从 Q-Former 转向 MLP，尽管 Q-Former 在词元预算上更有优势。

## 问题背景

BLIP-2 的 Q-Former（12.03 课）将一张图像压缩为 32 个词元，干净、高效、基准表现良好。但它有两个问题。

第一，Q-Former 是可训练的，但其损失并非最终任务。阶段 1 训练 ITC+ITM+ITG，阶段 2 训练语言模型损失。查询学到的是某种中间表示，LLM 再从中解码。瓶颈会造成信息损失。

第二，Q-Former 需要 1.88 亿参数，在 2023 年 LLaVA 的规模下，你必须针对目标 LLM 协同设计它。换一个 LLM，就要重新训练 Q-Former；换一个视觉编码器，也要重新训练。每种组合都是独立的研发项目。

LLaVA 的答案简单得令人尴尬：取 ViT 的 576 个块词元，每个通过一个 2 层 MLP（`1024 → 4096 → 4096`），然后把全部 576 个词元送入 LLM 的输入序列。没有瓶颈，没有针对奇怪目标的阶段 1 预训练，只需在直接的语言模型损失上训练 MLP。

数据从哪里来？LLaVA 的第二个洞见：用 GPT-4（纯文本）生成指令数据。将 COCO 图像的文字描述（5 条人工描述 + 边界框列表）输入 GPT-4，让它生成对话、描述和复杂推理问题，获得 158k 条指令-回答对，完全无需人工标注。

结果：一个在 8 块 A100 上训练一天的 VLM，在 MMMU 上击败了 Flamingo，并发布了社区可以扩展的开放检查点。到 2023 年底，已衍生出 50 多个分支。

## 核心概念

### 架构

LLaVA-1.5 @ 13B：
- 视觉编码器：CLIP ViT-L/14 @ 336（阶段 1 冻结，阶段 2 可选解冻）。
- 投影器：带 GELU 激活的 2 层 MLP，`1024 → 4096 → 4096`。
- LLM：Vicuna-13B（后来换成 Llama-3.1-8B）。

图像 + 文本提示的前向传播：

```
img -> ViT -> 576 个维度 1024 的块
块 -> MLP -> 576 个维度 4096 的词元
提示：系统提示 + "<image>" 占位符 + 用户问题
将 <image> 词元替换为 576 个投影词元
将完整序列送入 LLM
解码回复
```

图像占用 LLM 上下文的 576 个词元。在 2048 上下文中，文本剩余 1472 个词元；在 32k 上下文中，则几乎可以忽略不计。

### 阶段 1：投影器对齐

冻结 ViT，冻结 LLM，只训练 2 层 MLP。数据集：558k 图文配对（LAION-CC-SBU）。损失：以投影图像词元为条件，对描述进行语言建模。

单个 epoch，批量大小 128，几小时即可完成。投影器学会将 ViT 空间映射到 LLM 空间，无任务特定监督。

### 阶段 2：视觉指令微调

解冻投影器（仍可训练），解冻 LLM（通常全量，有时用 LoRA），在 158k 条视觉指令对话上训练。

指令数据是关键。Liu 等人的生成方法：
1. 取一张 COCO 图像。
2. 提取文字描述（5 条人工描述 + 边界框列表）。
3. 用三种提示模板发送给 GPT-4：
   - 对话：让用户和助手就这张图像进行一问一答。
   - 详细描述：给出图像的丰富详细描述。
   - 复杂推理：提出需要对图像进行推理的问题，然后回答。
4. 将 GPT-4 的输出解析为（指令，回复）对。

这整个过程完全不接触图像本身——只用文字描述。GPT-4 产生合理的图像内容，虽有噪声，但有效：158k 条对话足以解锁对话能力。

### 社区为何竞相复制

- 无需调整阶段 1 专用损失，全程语言模型损失。
- 投影器几小时即可训练，而非数天。
- LLM 可以更换（LLaVA-Llama2、LLaVA-Mistral、LLaVA-Llama3），只需重新训练投影器。
- 视觉指令数据流水线使用 GPT-4，针对新领域的重新生成成本低廉。

### LLaVA-1.5 与 LLaVA-NeXT

LLaVA-1.5（2023 年 10 月）新增：
- 学术任务数据（VQA、OKVQA、RefCOCO）混入指令微调。
- 更好的系统提示。
- 2048 → 32k 上下文。

LLaVA-NeXT（2024 年 1 月）新增：
- AnyRes：将高分辨率图像分割成 2×2 或 1×3 个 336×336 的裁剪块，加一个全局低分辨率缩略图。每个裁剪块变成 576 个词元；每张图像共约 2880 个视觉词元，OCR 和图表任务性能大幅提升。
- 更好的指令数据混合（含高质量 GPT-4V 描述的 ShareGPT4V）。
- 更强的基础 LLM（Mistral-7B、Yi-34B）。

### LLaVA-OneVision

12.08 课深入介绍 OneVision。简述：同样的投影器，但使用课程训练，在一个模型中覆盖单图像、多图像和视频，共享视觉词元预算。

### 与 Q-Former 的对比

| | Q-Former（BLIP-2） | MLP（LLaVA） |
|---|---|---|
| 每张图像的视觉词元 | 32 | 576（基础）或 2880（AnyRes） |
| 可训练参数 | 1.88 亿 + LM | 4000 万 + LM |
| 阶段 1 损失 | ITC+ITM+ITG | 仅 LM |
| LLM 更换 | 需要重训练 | 最小重训练即可更换 |
| 多图像 | 较为复杂 | 自然（拼接） |
| 视频 | 较为复杂 | 自然（逐帧拼接） |
| 词元预算 | 小 | 大 |

MLP 在简洁性和词元灵活性上获胜；Q-Former 在词元预算上获胜。到 2023 年底，词元预算不再是制约因素（LLM 上下文增长到 32k-128k+），简洁性占据主导。

### 提示格式

```
A chat between a curious human and an artificial intelligence assistant. The assistant gives helpful, detailed, and polite answers to the human's questions. USER: <image> Describe this image in detail. ASSISTANT: The image shows ...
```

`<image>` 是占位词元，在分词前被替换为 576 个视觉词元（AnyRes 时为 2880 个）。分词器看到一个比训练时稍长的序列，但 LLM 能处理这个新输入，因为阶段 1 已经训练了它。

### 参数经济

LLaVA-1.5-7B 分解：
- CLIP ViT-L/14 @ 336：3.03 亿（阶段 1 冻结，阶段 2 通常解冻）。
- 投影器（2 层线性）：约 2200 万可训练。
- Llama-7B：70 亿。
- 总计：73 亿参数。阶段 2 可训练：全量 70 亿 + 2200 万投影器。

阶段 2 训练成本：在 8×A100 上约 20 小时。这是关键数字——一天、一个节点、可复现。这就是 LLaVA 迅速传播的原因。

## 动手实践

`code/main.py` 实现了：

1. 纯 Python 实现的 2 层 MLP 投影器（玩具规模：维度 16 → 32 → 32）。
2. 提示构建流水线：系统提示 + 将 `<image>` 替换为 N 个投影词元 + 用户轮次 + 助手生成占位符。
3. 576 词元视觉块在 LLM 上下文中的可视化（占 2k / 32k / 128k 上下文的百分比）。

## 产出技能

本课产出 `outputs/skill-llava-vibes-eval.md`：给定 LLaVA 系列检查点，运行一个 10 提示的感性评估套件（3 条描述生成、3 条 VQA、2 条推理、2 条拒绝），返回人类可读的评分卡。这不是基准测试，而是确认投影器和 LLM 连接良好的冒烟测试。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| 投影器（Projector） | "MLP 桥接" | 带 GELU 的 2 层 MLP，将 ViT 维度映射到 LLM 维度 |
| 图像词元（Image token） | "<image> 占位符" | 推理前被 N 个投影视觉词元替换的提示标记 |
| 视觉指令微调 | "LLaVA 阶段 2" | 在 GPT-4 生成的（图像、指令、回复）三元组上训练 |
| 阶段 1 对齐 | "投影器预训练" | 冻结 ViT 和 LLM，在描述上用语言模型损失训练投影器 |
| AnyRes | "多裁剪块拼接" | 将高分辨率图像分割成裁剪块网格并拼接每个裁剪块的视觉词元 |
| LLaVA-Instruct | "GPT-4 生成" | 从 COCO 描述 + GPT-4 合成的 158k 条指令-回复对 |
| 视觉编码器冻结 | "骨干锁定" | CLIP 权重在阶段 1 不更新，有时在阶段 2 也不更新 |
| ShareGPT4V | "更好的描述" | GPT-4V 生成的 100 万条密集描述，用于更高质量的对齐 |
| VQA | "视觉问答" | 回答关于图像的开放性问题的任务 |
| Prismatic VLMs | "设计空间论文" | Karamcheti 2024 年系统测试投影器和数据选择的消融实验 |
