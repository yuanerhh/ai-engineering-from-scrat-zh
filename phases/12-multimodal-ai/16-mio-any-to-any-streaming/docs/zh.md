# MIO 与任意到任意流式多模态模型

> GPT-4o 提供了大多数开放模型无法复制的产品体验：一个能实时听声音、看视频、开口说话的智能体。2024 年底，开放生态的答案是 MIO（Wang 等，2024 年 9 月）。MIO 对文本、图像、语音和音乐分别进行词元化，在交错序列上训练一个因果 Transformer，实现任意模态到任意模态的生成。AnyGPT（Zhan 等，2024 年 2 月）是概念验证，MIO 是规模化实现，Unified-IO 2（Allen AI，2023 年 12 月）是具备视觉 + 动作接地的近亲。本课解读任意到任意范式——四个分词器、一个 Transformer、流式友好的解码。

**类型：** 学习
**编程语言：** Python（标准库，四模态词元分配器 + 流式解码循环）
**前置知识：** Phase 12 · 11（Chameleon）、Phase 6（语音与音频）
**预计时间：** 约 120 分钟

## 学习目标

- 设计一个容纳文本、图像、语音和音乐词元且无冲突的共享词汇表。
- 在压缩率和重建质量的权衡上对比 SEED-Tokenizer（图像）和 SpeechTokenizer 残差-VQ（语音）。
- 解释构建任意到任意生成能力的四阶段课程训练。
- 列出三种开放任意到任意方案及其主要权衡：MIO、AnyGPT、Unified-IO 2。

## 问题背景

统一多模态模型声称容易、规模化实现困难。2024 年之前大多数"任意到任意"系统都是流水线式的：视觉模型 → 文本表示 → 语音模型 → 音频。每次跳转都会丢失信息、增加延迟，并使训练变得更复杂。GPT-4o 的演示视频展示了亚秒级响应的单模型替代方案；开放系统落后了数月。

工程挑战包括：

- 每种模态都需要一个分词器，压缩率要足够低损以支持重建，词元速率要在 Transformer 可消费的范围内。
- 单一词汇表必须为文本（32k+）、图像（16k+）、语音（4k+）、音乐（8k+）分配空间，最少 4 万个条目。
- 训练数据必须覆盖每种输入-输出组合（文本→图像、图像→语音、语音→图像等），或模型必须具备组合泛化能力。
- 推理必须足够快地流式输出词元，以满足对话延迟要求（首字节音频时延 <500ms）。

## 核心概念

### 四种模态的四个分词器

MIO 的分词器栈：

- 文本：标准 BPE，词汇量约 32000。
- 图像：SEED-Tokenizer（2023）——带离散码本的量化 VAE，4096 个条目，每张图像 32×32 个词元。
- 语音：SpeechTokenizer 残差-VQ（2023）——将 16kHz 波形编码为 8 个层次码本；第一层是粗粒度内容，后续层添加韵律和说话人身份信息。
- 音乐：类似的残差-VQ（Meta MusicGen / Encodec 系列），4-8 个码本。

每种模态产生整数词元。词元在共享词汇表中获得不相交的 ID 范围：

```
文本:   0..31999
图像:  32000..36095  (4096 个图像词元)
语音:  36096..40191  (4096 个语音基础词元，加上残差层)
音乐:  40192..48383  (8192 个音乐词元)
分隔符:48384..48390  (<image>、<speech>、<music>、</...> 等)
```

总计：约 48k 词汇表。输入嵌入层和输出投影覆盖全部条目。

### 流式解码

语音生成使用残差-VQ。Transformer 预测基础（第 0 层）语音词元；并行解码的残差量化器预测后续层。每个第 0 层词元在 16kHz 下约对应 50ms 的音频。

流式模式如下：

1. 用户对麦克风说话；实时音频分词器每 50ms 生成一批语音词元。
2. MIO 在词元到达时即时消费（提示预填充 + 增量前向传播）。
3. 输出词元在生成时流式输出；并行语音解码器以约 50-150ms 的延迟将其转换为音频采样点。
4. 首字节音频时延：MIO 论文中约 300-500ms，接近 GPT-4o 的约 250ms。

Mini-Omni（arXiv:2408.16725）、GLM-4-Voice（arXiv:2412.02612）和 Moshi（arXiv:2410.00037）是互补的流式语音-LLM 设计。Moshi 在单张 GPU 上实现了 160ms 往返延迟。

### 四阶段课程训练

MIO 的训练课程：

1. 阶段 1——对齐。大规模模态对语料：文本-图像、文本-语音、文本-音乐。每对使用各自的词汇表段。训练共享词汇表。
2. 阶段 2——交错。多模态交错文档（包含图像和视频的博客、附有文字稿的播客等）。训练跨模态上下文理解。
3. 阶段 3——语音增强。额外的音频数据提升语音质量，同时不损失文本能力。
4. 阶段 4——SFT。跨模态指令微调：VQA、图像描述、旁白、语音到语音对话。

跳过任何阶段都会损害特定能力：跳过阶段 2 模型失去跨模态上下文理解；跳过阶段 3 语音质量变差。

### 视觉思维链

MIO 引入了视觉思维链：模型生成中间图像词元作为推理步骤。对于"猫在爬树吗？"这类问题，模型会：

1. 生成 `<image>` 词元，渲染场景（来自输入图像或草图）。
2. 生成文本分析草图。
3. 生成最终答案。

渲染的中间图像充当草稿纸。空间推理任务的基准测试有所提升。这一思路与文本推理的思维链如出一辙。

### 任意到任意的竞品

- AnyGPT（arXiv:2402.12226）：4 种模态（文本、图像、语音、音乐），设计类似。
- Unified-IO 2（arXiv:2312.17172）：增加了视觉动作输出、深度图、法线图，任务更多样化，规模较小。
- NExT-GPT（arXiv:2309.05519）：LLM + 模态专用扩散解码器，非单模型方案。
- CoDi（arXiv:2305.11846）：可组合扩散，通过共享潜变量实现任意到任意。

MIO 是最接近纯词元任意到任意的方案，AnyGPT 是其概念先驱。

### 延迟预算

对于对话产品，每个组件的延迟都至关重要：

- 麦克风到音频词元：约 50ms。
- 预填充（音频词元 + 历史）：8B 模型约 100ms。
- 第一个输出词元：约 50ms。
- 并行残差-VQ + 语音解码器：约 100-150ms。

首字节音频时延（TTFAB）最低约 300ms。GPT-4o 声称约 250ms，Moshi 声称 160ms，MIO/AnyGPT 按公开基准测试约在 400-600ms 范围。

### 任意到任意仍然困难的原因

即使到 2026 年，开放任意到任意模型在两个维度上仍落后于闭源模型：

- 语音质量：残差-VQ 分词器是有损的，对话语音听起来比 ElevenLabs 级别更机械。
- 跨模态推理：要求模型"唱出你所看到的内容"仍比纯视觉任务失败率高。

这些都是开放研究问题。Qwen3-Omni（第 12.20 课）是 2025 年最先进的开放尝试。

## 动手实践

`code/main.py` 实现以下功能：

- 定义四模态词汇表分配并打印。
- 将一组多模态输入（文本、图像、音频片段、音乐）通过分词器路由器进行路由。
- 模拟文本到语音响应的流式解码，附带延迟计数。
- 根据编码器、预填充和解码器延迟计算预期首字节音频时延。

## 练习

1. 你的产品接受语音输入并返回语音输出。端到端延迟预算目标是多少？列出各个耗时组件。

2. SpeechTokenizer 残差-VQ 使用 8 个码本。提出为什么并行解码残差层是必要的（而非顺序解码），它带来了多少延迟节省。

3. 你的词汇表有 32k 文本 + 4k 图像 + 4k 语音。添加 8k 音乐和约 10 个分隔符。在隐藏维度 4096 下，嵌入矩阵的参数量是多少？

4. 视觉思维链生成了一张中间图像。哪类问题受益？哪类问题因额外词元而受损？

5. 阅读 Moshi（arXiv:2410.00037）。描述其"内心独白"技术，并与 MIO 的视觉思维链进行对比。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| 任意到任意（Any-to-any） | "多模态输入输出" | 一个模型可以接受并生成文本、图像、语音和音乐的任意方向 |
| 残差-VQ（Residual-VQ） | "语音分词器栈" | 多码本词元化，每一层添加信息；基础层是内容，后续层是韵律 |
| SEED-Tokenizer | "图像编码" | 带 4096 条目码本的离散图像分词器，MIO 所使用 |
| 视觉思维链 | "视觉草稿纸" | 模型在给出最终答案前生成中间图像作为推理步骤 |
| 首字节音频时延（TTFAB） | "TTFAB" | 从用户语音到第一段音频输出的延迟；对话体验要求 <500ms |
| 四阶段课程训练 | "训练配方" | 对齐 → 交错 → 语音增强 → SFT，按此顺序进行 |
