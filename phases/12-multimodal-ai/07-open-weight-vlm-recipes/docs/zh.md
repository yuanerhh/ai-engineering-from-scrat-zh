# 开放权重 VLM 配方：真正重要的是什么

> 2024–2026 年的开放权重 VLM 文献是一片消融表格的海洋。Apple 的 MM1 测试了 13 种图像编码器、连接器和数据混合的组合；Allen AI 的 Molmo 证明了详细人工描述优于 GPT-4V 蒸馏；Cambrian-1 进行了 20 多个编码器比较；Idefics2 明确了五轴设计空间；Prismatic VLMs 在统一基准上对比了 27 种训练配方。在所有这些噪声中，有一小组结果在所有论文中保持一致：图像编码器比连接器架构更重要，数据混合比两者都重要，详细人工描述优于蒸馏合成数据。本课为你解读这些表格，省去你自己阅读的时间。

**类型：** 学习 + 实验
**编程语言：** Python（标准库，消融表格解析器 + 配方选择器）
**前置知识：** Phase 12 · 05（LLaVA 基线）
**预计时间：** 约 180 分钟

## 学习目标

- 说出 VLM 设计空间的五个轴：图像编码器、连接器、LLM、数据混合、分辨率策略。
- 阅读 MM1 / Idefics2 / Cambrian-1 的消融表格，预测哪个参数影响哪个基准。
- 给定计算预算和任务组合，为新 VLM 选择配方（编码器、连接器、数据、分辨率）。
- 解释为何在相同词元数量下，详细人工描述优于 GPT-4V 蒸馏。

## 问题背景

存在数百个开放权重 VLM，"良好"与"最先进"之间的差距大多不在架构上，而在数据、分辨率策略和编码器选择上。知道在模型表现不佳时首先调哪个参数，可以帮你避免耗费 500 万 GPU 小时的错误。

2023 年的浪潮（LLaVA-1.5、InstructBLIP、MiniGPT-4）使用描述对预训练 + LLaVA-Instruct-150k，良好的基线，MMMU 约达 35%。

2024 年的浪潮（MM1、Idefics2、Molmo、Cambrian-1、Prismatic VLMs）进行了详尽消融，结果令人惊讶且实用。

## 核心概念

### 五轴设计空间

Idefics2（Laurençon 等，2024）明确了这五个轴：

1. 图像编码器：CLIP ViT-L/14、SigLIP SO400m/14、DINOv2 ViT-g/14、InternViT-6B。编码器在块大小、分辨率和预训练目标上各有不同。
2. 连接器：MLP（2–4 层）、Q-Former（32 查询 + 交叉注意力）、Perceiver Resampler（64 查询）、C-Abstractor（卷积 + 双线性池化）。
3. 语言模型：Llama-3 8B/70B、Mistral 7B、Phi-3、Gemma-2、Qwen2.5。LLM 规模是主要参数成本。
4. 训练数据：描述对（CC3M、LAION）、交错数据（OBELICS、MMC4）、指令数据（LLaVA-Instruct、ShareGPT4V、PixMo、Cauldron）。
5. 分辨率策略：固定 224/336/448、AnyRes、原生动态分辨率；训练中逐步提升或保持不变。

每个生产 VLM 都在每个轴上做出选择。MMMU 分数的大部分方差由第 1、4、5 轴解释——而非连接器的选择。

### 第一轴：编码器 > 连接器

MM1 第 3.2 节显示：从 CLIP ViT-L/14 换成 SigLIP SO400m/14，MMMU 提升 3 分以上；将连接器从 MLP 换成 Perceiver Resampler，提升不到 1 分。Idefics2 也得到了相同结论：SigLIP > CLIP，在相同词元数量下 Q-Former ≈ MLP ≈ Perceiver。

Cambrian-1 的"视觉编码器大比拼"（Tong 等，2024）在视觉中心基准（CV-Bench）上对 20 多个编码器进行了评测。排行榜顶部是 DINOv2 和 SigLIP 的组合；CLIP 处于中游；ImageBind 和 ViT-MAE 较低。从 CLIP ViT-L 到 DINOv2 ViT-g/14 在 CV-Bench 上的差距约为 5–7 分。

2026 年开放 VLM 的默认编码器是 SigLIP 2 SO400m/14（用于语义 + 密集特征），在需要分割/定位时有时与 DINOv2 ViT-g/14 特征拼接（Cambrian 的"空间视觉聚合器"正是这样做的）。

### 第二轴：连接器设计收益不大

MM1、Idefics2、Prismatic 和 MM-Interleaved 都得出了相同结论：在固定视觉词元数量下，连接器架构几乎无关紧要。在相同词元预算下，对均值池化块的 2 层 MLP 与 32 查询的 Q-Former 相差不超过 1 分。

真正重要的是词元数量。更多视觉词元 = LLM 更多计算 = 更好性能（达到一定程度后边际递减）。每张图像 64 个词元对 OCR 来说太少；576–1024 词元是大多数开放 VLM 的最优区间；2048 以上仅对文档和图表有帮助。

Q-Former vs MLP 是成本问题，而非质量问题：Q-Former 将词元数量上限固定在 32–64，与图像分辨率无关；MLP 输出所有块词元。对于高分辨率输入，Q-Former 节省 LLM 上下文；对于低分辨率，差异可以忽略。

### 第三轴：LLM 规模决定上限

所有 VLM 论文中，将 LLM 从 7B 翻倍到 13B 可靠地在 MMMU 上增加 2–4 分。到 70B 时，大多数基准趋于饱和。VLM 的多模态推理上限就是 LLM 的文本推理上限——视觉编码器只能提供输入，无法替代推理。

这就是为什么 Qwen2.5-VL-72B 和 Claude Opus 4.7 在 MMMU-Pro 和 ScreenSpot-Pro 上遥遥领先：语言大脑足够强大。7B VLM 无法通过巧妙的连接器设计替代 70B VLM。

### 第四轴：数据——详细人工描述优于蒸馏

Molmo + PixMo（Deitke 等，2024）是 2024 年每个人都应该阅读的结果。Allen AI 让人工标注员用 1–3 分钟的密集语音转文字方式描述图像，产生了 71.2 万张密集标注图像，训练数据中完全没有 GPT-4V 蒸馏。

Molmo-72B 在 11/11 个基准上击败了 Llama-3.2-90B-Vision。差距不在架构——在于描述质量。详细人工描述每张图像包含的信息量是短网络描述的 5–10 倍，且在 GPT-4V 蒸馏会产生幻觉的地方保持事实准确。

ShareGPT4V（Chen 等，2023）和 Cauldron（Idefics2）采用了相同的思路，混合了人工 + GPT-4V 描述。趋势明确：在 2026 年的前沿，描述密度 > 描述数量 > 蒸馏便利性。

### 第五轴：分辨率及其策略

Idefics2 的消融：384 → 448 提升 1–2 分；448 → 980 结合图像分割（AnyRes）在 OCR 基准上再提升 3–5 分。固定分辨率训练在中等精度处达到瓶颈；分辨率渐进提升（从 224 开始，结束于 448 或原生分辨率）训练更快且最终精度更高。

Cambrian-1 研究了分辨率与词元的权衡：在固定计算量下，可以用更低分辨率换取更多词元，或用更少词元换取更高分辨率。对于 OCR，更高分辨率获胜；对于一般场景理解，更低分辨率更多词元获胜。

2026 年的生产配方：阶段 1 在 384 固定分辨率训练，阶段 2 使用动态分辨率，OCR 密集任务最高可到 1280。

### Prismatic 受控对比

Prismatic VLMs（Karamcheti 等，2024）控制了所有轴：同一个 13B LLM、同一份指令数据、同一个评估方案——每次只变化一个轴。结论：

- 每张图像的视觉词元数量解释了约 60% 的方差。
- 编码器选择解释了约 20%。
- 连接器架构解释了约 5%。
- 其余（数据混合、调度、学习率）占剩余约 15%。

这是一个粗略的分解，但对于"我应该首先消融什么"这个问题，是文献中最清晰的回答。

### 2026 年的选择指南

基于上述证据，2026 年新 VLM 项目的默认开放配方：

- 编码器：SigLIP 2 SO400m/14，使用 NaFlex 原生分辨率；若需要分割/定位，与 DINOv2 ViT-g/14 特征拼接。
- 连接器：块词元上的 2 层 MLP。除非受词元预算限制，否则跳过 Q-Former。
- LLM：Qwen2.5 / Llama-3.1 / Gemma 2，成本优先选 7B，质量优先选 70B，根据目标延迟选择。
- 数据：PixMo + ShareGPT4V + Cauldron，加上任务特定指令数据。
- 分辨率：动态（长边最小 256，最大 1280 像素）。
- 训练策略：阶段 1 对齐（仅训练投影器），阶段 2 全量微调，阶段 3 任务特定微调。

以上每个默认值都能追溯到本课末尾引用的论文中某个经过测量的消融实验。

## 动手实践

`code/main.py` 是一个消融表格解析器和配方选择器，编码了 MM1 和 Idefics2 的消融表格（精简版），支持查询：

- "给定预算 X 和任务 Y，哪个配方最优？"
- "在 7B Llama 上将 SigLIP 换成 CLIP，预期 MMMU 变化多少？"
- "以 80% 置信度，我应该首先消融哪个轴？"

输出是带有预期基准变化量和"优先消融"建议的排名配方列表。

## 产出技能

本课产出 `outputs/skill-vlm-recipe-picker.md`：给定目标任务组合、计算预算和延迟目标，输出完整配方（编码器、连接器、LLM、数据混合、分辨率策略），并引用支撑每个选择的消融实验。防止工程师每次启动新 VLM 项目时重新发明 Idefics2 消融表格。

## 练习

1. 阅读 MM1 第 3.2 节。对于固定 2B LLM 和 5000 万图像的预算，哪个编码器获胜？如果换成 13B LLM，答案会改变吗？为什么？

2. Cambrian-1 发现在视觉中心基准上 DINOv2 + SigLIP 的拼接优于单独使用其中任一，但在 MMMU 上没有额外收益。预测哪些基准会提升，哪些会持平。

3. 你的目标是在 2B LLM 上构建一个移动端 UI 智能体。选择编码器、连接器、分辨率和数据混合。用具体的消融表格条目为每个选择提供理由。

4. Molmo 发布了 4B 和 72B 两个版本，4B 与封闭 7B VLM 相当；72B 在 11/11 个基准上击败了 Llama-3.2-90B-Vision。这对 LLM 规模饱和假设说明了什么？

5. 设计一个消融表格，将数据混合质量与编码器质量在 7B VLM 上隔离开来。最少需要多少次训练运行？提出四组轴设置。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| 消融（Ablation） | "调一个参数" | 训练多个在设计空间中恰好有一个轴不同的版本，保持其他所有条件不变 |
| 连接器（Connector） | "桥接器"/"投影器" | 将视觉编码器输出映射到 LLM 词元空间的可训练模块（MLP、Q-Former、Perceiver） |
| 详细人工描述 | "密集描述" | 多句人工书写的描述（通常 80–300 词元），比网络 alt 文字丰富得多 |
| 蒸馏（Distillation） | "GPT-4V 描述" | 由更强的专有 VLM 生成的训练数据；方便但容易继承幻觉 |
| AnyRes / 动态分辨率 | "高分辨率路径" | 通过分块拼接或 M-RoPE 将大于编码器原生分辨率的图像送入的策略 |
| 分辨率渐进提升 | "课程学习" | 从低分辨率开始并逐步提升的训练策略，加快对齐学习速度 |
| 视觉中心基准 | "CV-Bench / BLINK" | 侧重细粒度视觉感知而非语言密集推理的评估 |
| PixMo | "Molmo 的数据" | Allen AI 的 71.2 万张密集标注图像数据集；人工语音转录为密集描述 |
