# ColPali 与视觉原生文档 RAG

> 传统 RAG 将 PDF 解析为文本、分块、嵌入块、存储向量。每个步骤都会丢失信号：OCR 丢弃图表数据，分块打断表格行，文本嵌入忽略图形。ColPali（Faysse 等，2024 年 7 月）提出了一个更简单的问题：为什么要提取文本呢？直接通过 PaliGemma 嵌入页面图像，用 ColBERT 风格的延迟交互进行检索，保留文档携带的所有布局、图形、字体和格式信号。已发布的基准测试：在视觉丰富的文档上，端到端精度比文本 RAG 高 20-40%。ColQwen2、ColSmol 和 VisRAG 扩展了这一模式。本课解读视觉原生 RAG 的论点，并构建一个小型类 ColPali 索引器。

**类型：** 构建
**编程语言：** Python（标准库，多向量索引器 + MaxSim 评分器）
**前置知识：** Phase 11（LLM 工程——RAG 基础）、Phase 12 · 05（LLaVA）
**预计时间：** 约 180 分钟

## 学习目标

- 解释双编码器检索（每个文档一个向量）与延迟交互检索（每个文档多个向量）的区别。
- 描述 ColBERT 的 MaxSim 操作，以及 ColPali 如何将其从文本词元推广到图像块。
- 构建一个小型类 ColPali 索引器：页面 → 图像块嵌入 → 对查询词嵌入进行 MaxSim → 返回前 k 页。
- 在发票/财务报告使用场景上对比 ColPali + Qwen2.5-VL 生成器与文本 RAG + GPT-4。

## 问题背景

在 PDF 上进行文本 RAG 会丢弃文档中大部分内容。财务报告的第三季度收入增长通常在图表中；医疗报告的发现在注释图像中；法律合同的签名栏是布局事实，不是文本事实。

文本 RAG 流水线：

1. PDF → 通过 OCR / pdftotext 转换为文本。
2. 文本 → 300-500 词元的块。
3. 块 → 双编码器嵌入（一个向量）。
4. 用户查询 → 嵌入 → 余弦相似度 → 前 k 个块。
5. 块 + 查询 → LLM。

五个有损步骤。图表未被捕获，表格跨块被打断，多列布局被展平，图形注释消失。

ColPali 的解决方案：跳过 OCR，直接嵌入页面图像。使用 ColBERT 风格的延迟交互进行检索，使模型在查询时能够关注细粒度图像块。

## 核心概念

### ColBERT（2020）

ColBERT（Khattab & Zaharia，arXiv:2004.12832）是一种文本检索方法。它不是每个文档一个向量，而是每个词元一个向量。查询时：

- 查询词元获得各自的嵌入（N_q 个向量）。
- 文档词元获得嵌入（N_d 个向量，通常已缓存）。
- 得分 = 对查询词元求和，对文档词元取最大余弦相似度：Σ_i max_j cos(q_i, d_j)。

这就是 MaxSim 操作。每个查询词元"选择"其最佳匹配的文档词元，最终得分是这些最大值的总和。

优点：召回率高，能处理词级语义。缺点：每个文档需要 N_d 个向量，存储成本高。

### ColPali

ColPali（Faysse 等，arXiv:2407.01449）将 ColBERT 模式应用于图像。

- 每页由 PaliGemma（ViT + 语言）编码为图像块嵌入：每页 N_p 个向量。
- 每个用户查询（文本）编码为查询词嵌入：N_q 个向量。
- 得分 = Σ_i max_j cos(q_i, p_j)，即查询文本词元和页面图像块的 MaxSim。
- 按总分检索前 k 页。

文档摄取时：用 PaliGemma 嵌入每一页，存储所有图像块嵌入。查询时：嵌入查询词元，对所有已索引页面嵌入计算 MaxSim，返回前 k 页。

优点：在视觉丰富的文档上端到端比文本 RAG 高 20-40%。每个块向量捕获局部布局和内容。

缺点：每页 N_p 个块 × 4 字节浮点 × D 维向量 = 存储快速增长。通过 PQ/OPQ 量化可以缓解。

### ColQwen2 与 ColSmol

ColQwen2（illuin-tech，2024-2025）将 PaliGemma 替换为 Qwen2-VL，基础编码器更好，检索效果更好。

ColSmol 是用于本地/边缘部署的小规模变体，约 1B 参数的 ColSmol 检索器可在消费级 GPU 上运行。

### VisRAG

VisRAG（Yu 等，arXiv:2410.10594）是一种不同的变体：不使用图像块的 MaxSim，而是通过 VLM 将每页池化为单个向量，再用双编码器检索。索引更快、存储更小，但召回率较弱。

质量与成本的权衡：需要质量时用 ColPali，需要规模时用 VisRAG。

### M3DocRAG

M3DocRAG（Cho 等，arXiv:2411.04952）将多模态检索扩展到多页多文档推理。跨文档检索页面，为 VLM 组合多页上下文。

### ViDoRe——基准测试

ColPali 的配套基准。视觉文档检索评估（Visual Document Retrieval Evaluation），任务包括财务报告、科学论文、行政文件、医疗记录、手册。指标：nDCG@5。

ColPali-v1 在 ViDoRe 上得分约 80% nDCG@5；同一文档上的文本 RAG 得分约 50-60%。

### 端到端 RAG 流水线

视觉原生 RAG 的完整流程：

1. 摄取：PDF → 页面图像 → PaliGemma 编码 → 存储所有图像块嵌入。
2. 查询：用户文本 → 查询词嵌入 → 对所有已索引页面计算 MaxSim → 前 k 页。
3. 生成：前 k 页图像 + 查询 → VLM（Qwen2.5-VL 或 Claude）→ 答案。

全程无 OCR。图形、图表、字体、布局全部流入答案。

### 存储计算

50 页财务报告，每页 729 个图像块，128 维嵌入：

- ColPali：50 × 729 × 128 × 4 字节 ≈ 18MB 原始数据，PQ 压缩后约 4MB。
- 文本 RAG：50 个块 × 768 维 × 4 字节 ≈ 150kB。

ColPali 每个文档约多 30 倍存储。规模化时，OPQ/PQ 可将倍数降低到约 5-10 倍，通常可以接受。

### 文本 RAG 仍然胜出的场景

- 无布局信号的纯文本文档（维基百科文章、聊天记录）——文本 RAG 更简单且存储更经济。
- 存储成本占主导的数百万页归档。
- 需要在检索旁边提供可提取 OCR 文本的严格合规要求。

2026 年的其他所有场景——财务报告、科学论文、法律合同、医疗记录、UX 文档——视觉原生 RAG 胜出。

## 动手实践

`code/main.py` 实现以下功能：

- 玩具图像块编码器：将"页面"（特征向量的小型网格）映射到图像块嵌入数组。
- MaxSim 评分器：计算查询词嵌入集合与页面图像块集合之间的 ColBERT 风格得分。
- 索引 5 个玩具页面，运行 3 个查询，返回前 k 个带得分的结果。

## 产出技能

本课产出 `outputs/skill-vision-rag-designer.md`：给定文档 RAG 项目，在 ColPali / ColQwen2 / VisRAG / 文本 RAG 之间做出选择，并计算存储规模。

## 练习

1. 200 页年报，每页 729 个图像块，128 维嵌入，4 字节浮点。计算原始存储量和 PQ 压缩（8倍）后的存储量。

2. MaxSim 是 Σ_i max_j cos(q_i, p_j)。与简单均值相似度相比，这个求和捕获了什么额外信息？

3. ColPali 将页面索引为图像块集合。如果改为在词级索引（如 ColBERT），会有什么变化？权衡是什么？

4. 为 100 万页语料库设计端到端流水线，查询延迟预算为 500ms。选择 ColQwen2 / VisRAG 并说明理由。

5. 阅读 M3DocRAG（arXiv:2411.04952）。描述多页注意力模式，以及它与单页 ColPali 检索的区别。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| 延迟交互（Late interaction） | "ColBERT 风格" | 使用每词元或每块嵌入 + MaxSim 进行检索，而非单一文档向量 |
| MaxSim | "块最大值" | 对每个查询词元，选择相似度最高的文档词元；对查询求和 |
| 双编码器（Bi-encoder） | "单向量" | 每个文档一个向量；更快但精度较低 |
| 多向量（Multi-vector） | "每文档多个向量" | 每个文档/页面存储 N_p 个向量；存储成本增加但召回率提高 |
| 块嵌入（Patch embedding） | "页面特征" | VLM 编码器为每个图像块生成的向量，每页缓存 |
| ViDoRe | "视觉文档基准" | ColPali 的视觉文档检索基准套件 |
| PQ 量化（PQ quantization） | "乘积量化" | 在保持向量相似度的同时缩减约 8 倍存储的压缩方法 |
