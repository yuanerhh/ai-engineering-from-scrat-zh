# 多模态智能体与计算机使用（综合实战）

> 2026 年的前沿产品是一个能够读取截图、点击按钮、导航 Web UI、填写表单并端到端完成工作流的多模态智能体。SeeClick 和 CogAgent（2024）验证了 GUI 定位这一基础能力，Ferret-UI 扩展到了移动端，ChartAgent 引入了图表的视觉工具调用。VisualWebArena 和 AgentVista（2026）是前沿模型追逐的基准——即便是 Gemini 3 Pro 和 Claude Opus 4.7，在 AgentVista 的困难任务上得分也只有约 30%。本课综合实战汇聚了 Phase 12 的每一条线索：感知（高分辨率 VLM）、推理（带工具调用的 LLM）、接地（坐标输出）、长程记忆和评估。

**类型：** 综合实战
**编程语言：** Python（标准库，动作 schema + 智能体循环骨架）
**前置知识：** Phase 12 · 05（LLaVA）、Phase 12 · 09（Qwen-VL JSON）、Phase 14（智能体工程）
**预计时间：** 约 240 分钟

## 学习目标

- 设计多模态智能体循环：感知 → 推理 → 行动 → 观察 → 重复。
- 构建 VLM 可以以 JSON 格式输出的 GUI 定位输出 schema（点击坐标、输入文本、滚动、拖拽）。
- 对比纯截图智能体、辅助功能树智能体和混合智能体。
- 在一小部分 VisualWebArena 切片上建立多模态智能体基准测试评估。

## 问题背景

订票网站工作流："帮我找一张 4 月 15 日去东京的机票，靠过道座位，800 美元以内，帮我预订。"

多模态智能体需要：

1. 截取浏览器截图。
2. 将截图 + URL + 目标解析为计划。
3. 输出结构化动作：点击（x,y 坐标）、在元素 E 处输入"东京"、向下滚动、选择（单选按钮）。
4. 将动作应用到浏览器。
5. 观察新状态（下一张截图）。
6. 重复，直到任务完成。

每个步骤都是一次多模态 VLM 调用。VLM 的输出必须是可解析的 JSON。错误会在各步骤中累积，因此恢复能力至关重要。

## 核心概念

### GUI 定位——基础能力

GUI 定位是：给定截图和自然语言指令，输出要点击的 (x, y) 坐标（或其他动作）。

SeeClick（arXiv:2401.10935）是第一个大规模开放成果：在合成 + 真实 GUI 数据上微调 VLM，将坐标输出为普通文本词元。

CogAgent（arXiv:2312.08914）为密集 UI 添加了 1120×1120 高分辨率编码，在 Web 导航上得分约 84%。

Ferret-UI（arXiv:2404.05719）专注于移动 UI，与 iOS 辅助功能数据集成。

输出格式通常是 JSON：

```json
{"action": "click", "x": 384, "y": 220, "element_desc": "搜索按钮"}
```

`element_desc` 有助于恢复：如果坐标在截图之间有偏移，语义提示可以让系统重新定位。

### 动作 Schema

典型动作 schema 包含 6-10 种动作类型：

- `click`：(x, y)
- `type`：(text, x?, y?)
- `scroll`：(direction, amount)
- `drag`：(x0, y0, x1, y1)
- `select`：(option_index)
- `hover`：(x, y)
- `navigate`：(url)
- `wait`：(ms)
- `done`：(success, explanation)

智能体每步输出一个动作。浏览器包装器执行该动作并返回新状态。

### 纯截图 vs 辅助功能树

两种输入模式：

- 纯截图：完整图像，无结构信息。最通用，适用于任何应用。
- 辅助功能树：结构化 DOM / iOS 辅助功能信息。对定位更可靠，在树可用时效果最佳。
- 混合：两者兼用，树作为原子动作的可靠定位器，截图提供语义上下文。

生产智能体在可能时使用混合模式。浏览器自动化（Selenium + 辅助功能）始终有树；桌面应用有时有。

### 长程记忆

20 步工作流会生成 20 张截图，VLM 的上下文很快被填满。三种压缩策略：

- 摘要链：每 5 步之后，总结已发生的事情，丢弃旧截图。
- 跳帧：保留第一张、最后一张和每第 3 张截图。
- 工具记录日志：执行动作，保留已完成操作的文字日志，不再回看旧截图。

Claude 的 computer-use API 使用日志模式，更简单、更可靠。

### 视觉工具调用

ChartAgent（arXiv:2510.04514）为图表理解引入了视觉工具调用：裁剪、缩放、OCR、调用外部检测。智能体可以输出"将区域 (100, 200, 300, 400) 裁剪后调用 OCR"作为工具调用，工具返回文字，VLM 继续推理。

这个模式可以推广：集合标记提示、区域注释和外部检测工具都符合"输出工具调用、接收结构化响应"的同一 schema。

### 2026 年基准测试

- ScreenSpot-Pro：约 1k 个 Web 截图上的 GUI 定位。开放 SOTA Qwen2.5-VL-72B 约 85%，前沿约 90%。
- VisualWebArena：端到端 Web 任务（购物、论坛、分类广告）。开放 SOTA 约 20%，Gemini 3 Pro 约 27%。
- AgentVista（arXiv:2602.23166）：2026 年最难的基准，12 个领域的真实工作流。前沿模型得分 27-40%，开放模型 10-20%。
- WebArena / WebShop：较旧的基准，已被前沿模型饱和。

### 仍然困难的原因

智能体性能瓶颈：

1. 细粒度视觉定位。"点击那个小 X"在移动分辨率下经常失败。
2. 长程规划。10 步动作后，智能体会偏离目标。
3. 错误恢复。当点击失败（点错了按钮）时，检测 + 恢复的训练数据很少。
4. 跨页面上下文。在标签页之间跳转或处理长表单时会丢失状态。

研究方向：记忆架构、显式重新规划、多模态验证（截图匹配以确认动作成功）。

### 综合实战任务

综合实战任务：构建一个计算机使用智能体，能够：

1. 读取订票网站模拟页面的 HTML + 截图。
2. 规划多步序列：搜索 → 选择 → 填写表单 → 提交。
3. 输出符合动作 schema 的 JSON 动作。
4. 在固定的 10 任务切片上进行评估。

本课提供了易于扩展为真实浏览器的脚手架代码。

## 动手实践

`code/main.py` 是综合实战脚手架：

- 动作 schema JSON 定义（10 种动作）。
- 作为字典的模拟浏览器状态。
- 智能体循环骨架：接收状态、输出动作、应用、循环。
- 10 任务迷你基准（合成页面）测量端到端成功率。
- 动作失败时的错误恢复钩子。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| GUI 定位（GUI grounding） | "点击坐标" | 模型在截图上为指令目标输出 (x,y) 坐标 |
| 动作 Schema | "工具定义" | 有效动作（点击、输入、滚动、拖拽）的 JSON 描述 |
| 辅助功能树（Accessibility tree） | "结构化 DOM" | 来自浏览器/iOS API 的机器可读 UI 层次结构 |
| 混合智能体（Hybrid agent） | "截图 + 树" | 同时使用图像和结构信息；比单独使用任一种更可靠 |
| 视觉工具调用（Visual tool use） | "缩放/裁剪/检测" | 智能体在执行计划中途调用外部视觉工具（OCR、检测） |
| 摘要链（Summary-chain） | "记忆压缩" | 定期用文字摘要替换长截图历史 |
| VisualWebArena | "端到端 Web 基准" | 2024 年端到端 Web 任务基准 |
| AgentVista | "2026 年困难基准" | 12 个领域的真实工作流；即便 Gemini 3 Pro 得分也只有约 30% |
