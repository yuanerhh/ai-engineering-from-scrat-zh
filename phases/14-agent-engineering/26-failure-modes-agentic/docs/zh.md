# 失败模式：智能体为何会出错

> MASFT（伯克利，2025）将 14 种多智能体失败模式归类为 3 个类别。微软的分类法记录了现有 AI 失败如何在智能体设置中放大。行业实地数据聚焦于五种反复出现的模式：幻觉操作、范围蔓延、级联错误、上下文丢失、工具误用。

**类型：** 学习 + 构建
**编程语言：** Python（标准库）
**前置知识：** Phase 14 · 05（Self-Refine 与 CRITIC）、Phase 14 · 24（可观测性）
**预计时间：** 约 60 分钟

## 学习目标

- 列出 MASFT 的三个失败类别以及每个类别中至少四种具体模式。
- 解释为什么智能体失败会放大现有的 AI 失败模式（偏见、幻觉）。
- 描述行业反复出现的五种模式及其缓解措施。
- 实现一个标准库检测器，用失败模式标签标注智能体追踪。

## 问题背景

团队发布的智能体在 90% 的追踪上运行良好。10% 的失败不是随机噪声——它们落入少数几个反复出现的类别。一旦你能命名它们，就可以监控并修复它们。

## 核心概念

### MASFT（伯克利，arXiv:2503.13657）

多智能体系统失败分类法。14 种失败模式聚类为 3 个类别。标注者间 Cohen's Kappa 为 0.88——这些类别是可靠可区分的。

核心主张：失败是多智能体系统中的根本性设计缺陷，而非可通过更好基础模型修复的 LLM 限制。

### 微软智能体 AI 系统失败模式分类法

- 现有 AI 失败（偏见、幻觉、数据泄露）在智能体设置中放大。
- 新失败由自主性产生：大规模意外操作、工具误用、任务漂移。
- 这份白皮书是智能体产品的风险登记册。

### 智能体 AI 中的故障特征（arXiv:2603.06847）

- 失败源于编排、内部状态演化和环境交互。
- 不仅仅是"糟糕的代码"或"糟糕的模型输出"。

### LLM 智能体幻觉调查（arXiv:2509.18970）

两种主要表现形式：

1. **指令遵循偏差**——智能体不遵循系统提示词。
2. **长程上下文误用**——智能体遗忘或错误应用早期轮次的上下文。

子意图错误：遗漏（遗漏步骤）、冗余（重复步骤）、无序（步骤顺序混乱）。

### 行业反复出现的五种模式

Arize、Galileo、NimbleBrain 2024-2026 实地分析收敛于：

1. **幻觉操作。** 智能体调用不存在的工具或伪造参数。
2. **范围蔓延。** 智能体将任务扩展到用户要求之外（创建额外的 PR，发送额外的邮件）。
3. **级联错误。** 一次错误调用触发下游影响。一个幻觉 SKU 触发四次 API 调用——一个多系统事件。
4. **上下文丢失。** 长时程任务忘记早期轮次的约束。
5. **工具误用。** 用错误的参数调用正确的工具，或完全使用错误的工具。

级联是最危险的。智能体无法区分"我失败了"和"任务不可能完成"，经常在 400 错误上幻觉出成功消息以关闭循环。

### 缓解措施：每步设置门控

在推理链的每个步骤设置自动验证门控，对照环境状态检查事实依据。具体而言：

- 逐步安全分类器（第 21 课）。
- 工具调用参数验证（第 06 课）。
- 对照已知事实交叉检查检索到的内容（第 05 课，CRITIC）。
- 通过重新探查状态来检测成功幻觉（文件真的被创建了吗？）。

### 失败监控在哪里出错

- **只标记崩溃。** 大多数智能体失败会产生看起来有效的输出。需要内容级检查。
- **没有基线。** 漂移检测需要最后已知良好的状态；没有它你无法说"这正在变得更糟"。
- **过度告警。** 每次失败都产生一个页面告警。聚类并限速。

## 动手实践

`code/main.py` 实现了一个标准库失败模式标注器：

- 涵盖五种模式的合成追踪数据集。
- 每种模式的检测函数（工具调用、输出、重复操作的签名模式）。
- 一个标注器，对每条追踪进行标注并报告模式分布。

运行：

```
python3 code/main.py
```

输出：每条追踪的标签 + 聚合分布，廉价再现了 Phoenix 的追踪聚类所呈现的内容。

## 使用建议

- **Phoenix** 用于生产漂移聚类（第 24 课）。
- **Langfuse** 用于会话重放 + 注释。
- **自定义** 用于可观测性平台无法检测的领域特定签名。

## 产出技能

`outputs/skill-failure-detector.md` 生成针对你领域定制的失败模式检测器，接入追踪存储。

## 练习

1. 添加"成功幻觉"检测器：智能体返回成功但目标状态未改变。
2. 标注你构建的产品中的 100 条真实追踪。哪种模式占主导？修复它的成本是多少？
3. 实现"级联半径"指标：给定第 N 步的失败，它影响了多少个下游步骤？
4. 阅读 MASFT 的 14 种失败模式。选择三种适用于你产品的。编写检测器。
5. 将一个检测器接入 CI 作业：如果 >=5% 的追踪标注了某种模式，则构建失败。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| MASFT | "多智能体失败分类法" | 伯克利 14 模式分类 |
| 级联错误 | "涟漪失败" | 一个早期错误传播到 N 个步骤 |
| 上下文丢失 | "忘记了约束" | 长时程轮次丢弃早期轮次的事实 |
| 工具误用 | "工具/参数错误" | 有效调用，错误的调用方式 |
| 成功幻觉 | "伪造完成" | 智能体在 400 错误上声称成功；状态未改变 |
| 范围蔓延 | "过度扩展" | 智能体做的超出了被要求的 |
| 指令遵循偏差 | "不服从" | 忽略系统提示词或用户约束 |
| 子意图错误 | "计划缺陷" | 计划执行中的遗漏、冗余、无序 |
