# 评估驱动的智能体开发

> Anthropic 的指导原则："从简单提示词开始，通过全面评估优化它们，只在需要时才添加多步骤智能体系统。"评估不是最后一步。它是驱动 Phase 14 中每个其他选择的外层循环。

**类型：** 学习 + 构建
**编程语言：** Python（标准库）
**前置知识：** Phase 14 全部内容
**预计时间：** 约 60 分钟

## 学习目标

- 列出三个评估层——静态基准、自定义离线、在线生产——以及各自的用途。
- 解释评估器-优化器紧密循环。
- 描述 2026 年最佳实践：评估与代码共存，在 CI 中运行，门控 PR。
- 将 Phase 14 的每节课连接到它生成的评估案例。

## 问题背景

智能体通过演示。它们在生产中以演示无法预测的方式失败。基准回答"这个模型总体上有能力吗？"而不是"这个智能体是否为我的产品发布正确的补丁？"答案：在三个层次上持续运行评估，每个防护栏和学习规则都映射到一个评估案例。

## 核心概念

### 三个评估层

1. **静态基准**——代码方面的 SWE-bench Verified（第 19 课）、浏览/桌面方面的 WebArena/OSWorld（第 20 课）、通用方面的 GAIA（第 19 课）、工具使用方面的 BFCL V4（第 06 课）。用于跨模型比较和回归门控。污染是真实存在的：SWE-bench+ 发现了 32.67% 的解决方案泄露。始终报告 Verified / +-审计的分数。

2. **自定义离线评估**——你的产品形态：
   - LLM 作为判断器（Langfuse、Phoenix、Opik——第 24 课）。
   - 基于执行（运行补丁，检查测试）。
   - 基于轨迹（与黄金轨迹比较动作序列；OSWorld-Human 显示顶级智能体比黄金多 1.4-2.7 倍步骤）。

3. **在线评估**——生产环境：
   - 会话重放（Langfuse）。
   - 防护栏触发告警（第 16 课，第 21 课）。
   - 每步成本/延迟跟踪（第 23 课 OTel span）。

### 评估器-优化器（Anthropic）

紧密循环：

1. 提案者生成输出。
2. 评估器判断。
3. 精炼，直到评估器通过。

这是 Self-Refine（第 05 课）的泛化。你关心的任何智能体流程都可以用评估器-优化器包装以提高可靠性。

### 2026 年最佳实践

- 评估与代码共存。
- 在每个 PR 的 CI 中运行。
- 根据评估分数门控合并（例如"与主分支相比无超过 5% 的回归"）。
- 每个防护栏映射到一个评估案例。
- 每个学习规则（Reflexion、pro-workflow 学习规则）映射到一个失败案例。

### 评估驱动开发在哪里失败

- **没有基线。** 没有最后已知良好状态的评估是无法解读的。存储基线。
- **没有依据的 LLM 判断器。** 判断器也会产生幻觉。CRITIC 模式（第 05 课）——判断器依据外部工具。
- **对评估过度拟合。** 为评估优化会偏离生产实用性。轮换案例。
- **不稳定的评估。** 不确定性案例导致误报。固定种子，快照状态。

## 动手实践

`code/main.py` 是一个标准库评估运行框架：

- 按类别（基准、自定义、在线）的案例注册表。
- 一个被测试的脚本化智能体。
- 评估器-优化器循环：提案、判断、精炼，直到通过或达到最大轮数。
- CI 门控：聚合通过率 + 与基线的回归对比。

运行：

```
python3 code/main.py
```

输出：每个案例的通过/失败，回归标志，CI 门控判决。

## 使用建议

- 在与智能体代码相同的仓库中编写评估案例。
- 通过 CI 在每个 PR 上运行它们。
- 在回归时使构建失败。
- 随时间跟踪通过率。
- 将每次生产失败与新案例关联。

## 产出技能

`outputs/skill-eval-suite.md` 为智能体产品构建一个带 CI 门控和回归跟踪的三层评估套件。

## 练习

1. 取你的一个生产失败。编写一个能复现它的评估案例。你的智能体现在能通过吗？
2. 为你的领域构建一个带三个维度（事实性、语气、范围）的 LLM 判断器评分标准。对 50 个会话打分。
3. 将评估套件接入 CI。在 >=5% 回归时使构建失败。
4. 添加轨迹效率指标：智能体与黄金轨迹相比走了多少步？
5. 将 Phase 14 的每节课映射到你套件中的一个评估案例。有遗漏吗？那就是需要填补的差距。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| 静态基准 | "现成评估" | SWE-bench、GAIA、AgentBench、WebArena、OSWorld |
| 自定义离线评估 | "领域评估" | 针对你产品形态的 LLM 判断器/执行/轨迹 |
| 在线评估 | "生产评估" | 会话重放、防护栏告警、成本/延迟跟踪 |
| 评估器-优化器 | "提案-判断-精炼" | 迭代直到判断器通过 |
| CI 门控 | "合并阻断器" | 评估回归时使构建失败 |
| 基线 | "最后已知良好" | 用于检测回归的参考分数 |
| 轨迹效率 | "步骤数与黄金对比" | 智能体步骤数除以人类专家最小步骤数 |
