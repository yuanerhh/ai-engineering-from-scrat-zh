# 思维树与 LATS：深思熟虑的搜索

> 单条思维链轨迹没有回溯的空间。ToT（Yao 等人，2023）将推理转变为在每个节点进行自我评估的树。LATS（Zhou 等人，2024）在蒙特卡洛树搜索下统一了 ToT、ReAct 和 Reflexion。24 点游戏从 4%（CoT）提升到 74%（ToT）；LATS 在 HumanEval 上达到 92.7% pass@1。

**类型：** 构建
**编程语言：** Python（标准库）
**前置知识：** Phase 14 · 01（智能体循环）、Phase 14 · 03（Reflexion）
**预计时间：** 约 75 分钟

## 学习目标

- 将推理框架化为搜索：节点是"思考"，边是"扩展"，价值是"有多有希望"。
- 用标准库实现 ToT 风格的 BFS 树搜索，带自我评估评分。
- 扩展为带 Select（选择）/ Expand（扩展）/ Simulate（模拟）/ Backpropagate（反向传播）的玩具 LATS MCTS 循环。
- 决定何时搜索值得付出 token 倍增代价（24 点游戏、代码生成），何时单条轨迹足够（简单问答）。

## 问题背景

思维链是线性的。如果第一步错了，后续每一步都在一个错误的前提上工作。在 24 点游戏（用四个数字通过 + − × ÷ 得到 24）上，GPT-4 CoT 达到 4% 精度。模型在早期选择了错误的子表达式，无法恢复。

推理需要的是提出多个候选项、评估它们、选择有希望的，并在遇到死路时回溯的能力。这就是搜索。思维树（Tree of Thoughts）和 LATS 是两种标准表述。

## 核心概念

### 思维树（Yao 等人，NeurIPS 2023）

每个节点是一个连贯的中间步骤（"一个思考"）。每个节点可以扩展到 K 个子思考。LLM 用评分提示词对每个节点进行自我评估。搜索探索树——BFS（广度优先）、DFS（深度优先）或 beam（束搜索）。

```
                     (根节点: "从 4 6 4 1 得到 24")
                    /               |            \
           ("6 - 4 = 2")    ("4 + 1 = 5")    ("4 * 6 = 24")  <- 分数: 高
              /   \              |                  |
          ...    ...          ...                完成
```

自我评估是关键所在。论文展示了三种变体：`sure / likely / impossible` 分类、`1..10` 数值分数，以及候选项之间的投票。三种都在 24 点游戏上大幅超过 CoT（GPT-4 下 4% -> 74%）。

### LATS（Zhou 等人，ICML 2024）

LATS 在蒙特卡洛树搜索（MCTS）下统一了 ToT、ReAct 和 Reflexion。LLM 扮演三个角色：

- **策略（Policy）**：提出候选的下一步行动（ReAct 风格）。
- **价值函数（Value function）**：对部分轨迹评分（ToT 风格的自我评估）。
- **自我反思器（Self-reflector）**：在失败时写一段自然语言反思（Reflexion 风格），并用它为未来的 rollout（展开）重新播种。

环境反馈（观察结果）混入价值函数，使搜索被真实工具结果而非仅模型意见所引导。论文时的结果：GPT-4 在 HumanEval pass@1 上达到 92.7%（当时最先进），GPT-3.5 在 WebShop 上达到平均 75.9（接近基于梯度的微调）。

### MCTS 最简要点

每次迭代的四个阶段：

1. **Select（选择）**——使用 UCT（树的置信上界）从根节点走到叶节点。
2. **Expand（扩展）**——通过策略生成 K 个子节点。
3. **Simulate（模拟）**——从子节点使用策略展开，用价值函数（或环境奖励）对叶节点评分。
4. **Backpropagate（反向传播）**——向路径上方更新访问次数和价值估计。

UCT 公式：`Q(s, a) + c * sqrt(ln N(s) / N(s, a))`。第一项是利用，第二项是探索。按任务调整 `c`。

### 成本现实

搜索会使 token 爆炸。ToT 在 24 点游戏上使用比 CoT 多 100-1000 倍的 token。LATS 类似。这不是免费的；将搜索保留用于：

- 单条轨迹明显不足的任务（24 点游戏、复杂代码）。
- 正确性比实时性更重要的任务。
- 有廉价、可靠价值函数的任务（代码的单元测试、数学的明确目标）。

如果你的任务有单一正确答案和嘈杂的评估器，搜索往往会使事情变得更糟——它会找到一个"得分好"的错误答案。

### 2026 年的定位

大多数生产智能体不运行 LATS。它们运行带工具驱动验证的 ReAct（CRITIC，第 05 课）。搜索出现在专门的细分领域：

- 以测试作为价值函数的编码智能体（HumanEval 风格）。
- 探索多条查询路径的深度研究智能体。
- LangGraph 子图中规划密集的工作流。

AlphaEvolve（第 11 课）是 2025 年的极端案例：对代码进行进化搜索，机器可检验的适应度，前沿突破（56 年来首次 4x4 矩阵乘法改进）。

## 动手实践

`code/main.py` 实现了：

- 一个在程式化的"选择算术运算"任务上的小型 ToT BFS。
- 同一任务上的玩具 LATS MCTS 循环（Select / Expand / Simulate / Backpropagate），带 UCT 选择。
- 一个组合符号分数和自我评估分数的价值函数。

运行：

```
python3 code/main.py
```

轨迹显示 ToT 用 BFS 每节点扩展三个候选项，对比 LATS 通过 MCTS 收敛到最佳 rollout。两者的 token 计数均会打印出来。

## 使用建议

LangGraph 将 ToT 风格的探索作为子图模式提供；LangChain 团队 2024 年 5 月关于 LATS 的博客文章是参考教程。LlamaIndex 提供了一个 `TreeOfThoughts` 智能体。对于 2026 年的大多数生产智能体，这个模式存在于 `if task_complexity > threshold: use_search()` 门之后——参见第 05 课中的评估器-优化器模式。

## 练习

1. 用 UCT c=0.1 和 c=2.0 运行玩具 LATS。轨迹有什么变化？
2. 将价值函数替换为更嘈杂的评分器（添加随机抖动）。MCTS 仍然能找到最佳叶节点吗？它能容忍的最低信噪比是多少？
3. 实现 beam search ToT（每层保留 top-k）并与 BFS 比较。在紧张的 token 预算下哪个更好？
4. 阅读 LATS 5.1 节。复现 HumanEval 轨迹计数：达到报告的 pass@1 需要多少次 rollout？
5. 阅读 LATS 论文关于"LATS 帮助较少时"的讨论。写一段将任务形态映射到搜索策略的决策规则。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| 思维树（Tree of Thoughts） | "分支思维链" | Yao 等人——带自我评估的思考节点树 |
| LATS | "LLM 的 MCTS" | Zhou 等人——在 MCTS 下统一 ToT + ReAct + Reflexion |
| UCT | "置信上界" | 平衡利用（Q）和探索（ln N / n）的选择公式 |
| 价值函数 | "这个状态有多好" | 提示词驱动的 LLM 分数或环境奖励；输入到反向传播 |
| 策略 | "行动提议者" | ReAct 风格的生成器；输出候选的下一步思考/行动 |
| Rollout | "模拟轨迹" | 从节点使用策略走到叶节点，用价值评分 |
| 反向传播 | "更新祖先节点" | 将叶节点的奖励向上推，更新访问次数和 Q 值 |
| 搜索成本 | "token 爆炸" | 24 点游戏上比 CoT 多 100-1000 倍；采用前先预算 |
