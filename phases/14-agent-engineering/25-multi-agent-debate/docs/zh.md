# 多智能体辩论与协作

> Du 等人（ICML 2024，"思维社会"）让 N 个模型实例独立提出答案，然后在 R 轮中迭代地相互批评以收敛。改善了事实准确性、规则遵循和推理。稀疏拓扑在 token 成本上优于全网格。

**类型：** 学习 + 构建
**编程语言：** Python（标准库）
**前置知识：** Phase 14 · 12（工作流模式）、Phase 14 · 05（Self-Refine 与 CRITIC）
**预计时间：** 约 60 分钟

## 学习目标

- 解释辩论协议：N 个提案者，R 轮，收敛到共同答案。
- 描述辩论为何能改善事实准确性、规则遵循和推理。
- 解释稀疏拓扑：并非每个辩手都需要看到其他所有人。
- 用脚本化 LLM 实现标准库辩论，包含全网格和稀疏变体；测量 token 成本与准确率。

## 问题背景

Self-Refine（第 05 课）是一个模型批评自己——有群体思维的风险。CRITIC（第 05 课）通过外部工具来依据批评——并非总是可用。辩论引入了第三种模式：多个实例、交叉批评、通过分歧收敛。

## 核心概念

### 思维社会（Du 等人，ICML 2024）

- N 个模型实例独立对同一问题提出答案。
- 在 R 轮中，每个模型读取其他人的提案并批评它们。
- 模型根据批评更新自己的答案。
- 经过 R 轮后，返回收敛答案。

由于成本考量，原始实验使用 N=3、R=2。在难题（MMLU、GSM8K、国际象棋走法有效性、传记生成）上，准确率随更多智能体和更多轮次而提高。

跨模型组合优于单模型辩论：ChatGPT + Bard 组合 > 任意单独一个。

### 稀疏拓扑

"用稀疏通信拓扑改进多智能体辩论"（arXiv:2406.11776，2024-2025）表明全网格辩论并不总是最优的。稀疏拓扑（星形、环形、轮毂辐射型）可以在更低 token 成本下匹配准确率。每个辩手只看到部分同伴。

含义：

- 全网格 N=5，R=3 = 5 × 3 = 15 个提案，每个读取 4 个同伴 = 60 次批评操作。
- 星形 N=5，R=3（一个中心 + 4 个辐射）= 15 个提案，辐射只读中心 = 12 次批评操作。

### 辩论何时有帮助

- **事实准确性。** N 个独立提案，交叉检查减少幻觉。
- **规则遵循。** 国际象棋走法有效性——一个模型遗漏规则，其他模型发现。
- **开放式推理。** 多种框架缩小到正确答案。

### 辩论何时有害

- **延迟敏感的用户体验。** N × R 串行轮次是你可能无法承受的延迟。
- **成本敏感的规模。** 每个问题 N × R 个 token。
- **简单事实查询。** 一次查询比五次辩论更便宜。

### 2026 年的实际应用

- **Anthropic 编排器-工作者**（第 12 课）——带合成步骤的辩论的一种变体。
- **LangGraph 监督者**（第 13 课）——中心路由器 + 专家智能体可以将辩论实现为一个节点。
- **OpenAI Agents SDK**（第 16 课）——智能体来回切换以进行迭代批评。
- **多智能体评估**——配对辩论 + 评估器-优化器以获取评估信号。

### 这个模式在哪里出错

- **收敛崩溃。** 所有智能体都收敛到第一个错误答案。通过强制分歧轮次来缓解。
- **中心故障。** 在星形拓扑中，一个坏的中心会污染所有人。轮换或使用多个中心。
- **提示词同质化。** 所有智能体使用相同的提示词；它们产生相同的答案。使用多样化的提示词和/或模型。

## 动手实践

`code/main.py` 实现了标准库辩论：

- `Debater` 类（带每辩手意见漂移的脚本化 LLM）。
- `FullMeshDebate` 和 `SparseDebate` 运行器。
- 三个问题：一个事实性的，一个基于规则的，一个推理性的。
- 指标：收敛答案、收敛所需轮数、总批评操作数。

运行：

```
python3 code/main.py
```

输出：每协议的准确率和成本；稀疏拓扑在较低成本下在 3 个问题中的 2 个上匹配全网格。

## 使用建议

- **Anthropic 编排器-工作者** 用于简单的 2-3 工作者辩论。
- **LangGraph** 用于带检查点的有状态多轮辩论。
- **自定义** 用于研究或专门的正确性保证。

## 产出技能

`outputs/skill-debate.md` 搭建一个带可配置拓扑、N、R 和收敛规则的多智能体辩论框架。

## 练习

1. 实现"强制分歧"规则：在第 1 轮中，每个辩手必须产生不同的提案。测量对收敛速度的影响。
2. 添加置信度加权聚合：辩手返回（答案，置信度）；聚合器按置信度加权。这有帮助吗？
3. 将一个"智能体"替换为具有不同观点的不同脚本化 LLM。异质性能提高准确率吗？
4. 测量全网格与稀疏拓扑在你的 3 个问题上的 token 成本。绘制成本与准确率关系图。
5. 阅读"思维社会"论文。将你的玩具移植到 N=5，R=3。什么出了问题？什么变得更好了？

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| 辩论 | "多智能体批评" | N 个提案者，R 轮交叉批评，收敛 |
| 全网格 | "所有人读所有人" | 每个辩手每轮读取每个同伴 |
| 稀疏拓扑 | "有限同伴视图" | 辩手只读取部分同伴 |
| 轮毂辐射型 | "星形拓扑" | 一个中心辩手，N-1 个辐射只读中心 |
| 收敛 | "达成共识" | 辩手收敛到共同答案 |
| 思维社会 | "Du 等人辩论论文" | ICML 2024 多智能体辩论方法 |
