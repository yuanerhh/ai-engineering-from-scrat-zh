给定一个任务类别（重复性智能体运行类型——例如"重构函数"、"关闭支持工单"），维护反思的情节记忆缓冲区。每条反思记录一种失败模式及其自然语言纠正洞察。该缓冲区将被前置到同类任务的下一次试验中。

输出内容：

1. **反思捕获**。当一次试验以低于阈值的评估分数结束时，生成一条格式为"我因为 Y 未能完成 X；下次应该 Z。"的单行反思。丢弃外部失败（网络错误、上游 500 错误）的反思，除非它们可重现。
2. **TTL 与去重**。反思默认在 N 次试验后过期（建议 10 次）。完全重复的反思合并。近似重复（小型嵌入模型余弦相似度 > 0.9，或共享子字符串 >= 80%）只保留最新的一条。
3. **范围策略**。三种范围：task-class（按任务名称）、user（同一用户跨任务）、agent（跨所有用户）。默认为 task-class。仅当反思涉及用户特定偏好时才升级到 user 范围；永远不自动升级到 agent 范围。
4. **压缩**。当缓冲区超过预算时，执行离线压缩：聚类近似重复，汇总，合并。压缩在热路径之外运行——不要延迟主智能体的响应。
5. **Prompt 集成**。生成一个标题为"从过往试验中学到的内容"的单独块，包含项目符号列表。Prompt 中最多显示 6 条；溢出内容放入汇总条目（"……以及 4 条关于超时的旧反思"）。

硬性拒绝：

- 将反思写为"下次要更加小心"。这不具有可操作性。用强制要求具体"下次操作"的 Prompt 重新运行反思器。
- 根据挂钟时间而非试验次数使反思过期。TTL 应以试验为范围，而非以时间为范围，以支持离线可重放的运行。
- 存储包含密钥（API 密钥、Token、PII）的反思。在提交到缓冲区前，用具体的"包含密钥"类别错误拒绝。

拒绝规则：

- 若未附加评估器，拒绝并建议使用第 05 课（Self-Refine/CRITIC）——反思需要信号，而非凭感觉。
- 若任务类别为一次性任务（永不重复），拒绝；情节记忆对从不重复的任务毫无作用。

输出：结构化缓冲区文件（包含反思对象的 JSON：trial id、task class、scope、text、created_at、ttl_remaining）、下次试验的 Prompt 块，以及列出即将过期条目的"过期反思"报告。

结尾给出"下一步阅读"指引：若缓冲区持续达到上限，指向第 06 课（上下文压缩）；若需将压缩移出热路径，指向第 08 课（Letta 离线计算）。
