# 注意力变体选择器

帮助开发者为新 Transformer 选择并论证注意力拓扑，或为正在扩展到更长上下文的现有模型提供建议。

## 需要收集的输入信息

1. **目标上下文长度** — 训练时和推理时（通常不同——许多模型训练时使用 16K，推理时扩展）。
2. **检索需求**（1-5 分）：1 = 纯对话，5 = 大海捞针 / RAG / 含长仓库上下文的代码。
3. **推理内存预算** — 每请求 KV 缓存容忍度（每 token 每层字节数是正确的单位）。
4. **训练成本容忍度** — 从头训练 SWA 成本低；将差分注意力改装到预训练模型成本高。
5. **硬件目标** — Hopper+ 支持完整 FlashAttention-3，Ada 支持 FA2，更老的 GPU 有遮掩限制。

## 决策规则

- **上下文 ≤ 16K 且检索 ≤ 3**：使用 FlashAttention 全注意力。不要过早优化。
- **上下文 16-128K 且检索 ≤ 3**：混合 SWA + 全局，比例 5:1，窗口 1024（Gemma 3 形状）。在保持检索能力的同时压缩 KV。
- **上下文 > 128K**：纯 SWA 加每 4-6 层一个全局层，加位置插值 / YaRN 缩放（第 4 课）。
- **检索 = 5 且训练预算允许**：仅在最顶部 4 层考虑差分注意力（KV 翻倍减半，获得大部分噪声消除收益）。
- **发布公开 API**：优先使用稳定模式（全注意力、SWA、Gemma-3 混合）。除非有内核工程师，否则跳过原生稀疏注意力 / 差分注意力。
- **无法修改基础模型**：SWA 可在推理时通过遮掩改装；差分注意力和稀疏注意力不能。

## 始终标记

- 低于 70 亿参数的纯 SWA 模型通常在推理基准上有明显退化。建议不使用。
- 窗口大小 < 512 几乎从不正确。选更大值或使用其他拓扑。
- 差分注意力论文的报告基于小模型（3-70 亿）。截至 2026 年初，大规模扩展证据有限。
- 每种变体都与 RoPE / YaRN 缩放（第 4 课）交互。明确说明位置方案。

## 输出格式

返回：

1. **推荐方案** — 单一命名拓扑（例如"Gemma-3 混合，W=1024，5:1 SWA:全局"）。
2. **理由** — 将每个输入映射到上述决策规则。
3. **KV 缓存估算** — 在目标上下文下，每 token 每层字节数，以及批次大小为 1 时的 GB 数。
4. **迁移路径** — 如果基础模型已训练，如何进行改装。
5. **已知风险** — 哪些基准 / 工作负载可能退化。
