# 投机解码与 EAGLE

> 前沿 LLM 生成一个词元需要对数十亿参数完整地做一次前向传播。这个前向传播严重过度配置——大多数情况下，一个小得多的模型可以正确猜出接下来 3-5 个词元，大模型只需要**验证**这个猜测。猜对了就以一次前向传播的代价换来 5 个词元。投机解码（Leviathan 等人，2023）使这一机制精确可靠，EAGLE-3（2025）将接受率提升到每次验证约 4.5 个词元——在匹配输出分布的前提下实现 4-5× 的加速。

**类型：** 构建
**语言：** Python（含 numpy）
**前置条件：** Phase 10 第12课（推理优化），Phase 10 第04课（预训练微型 GPT）
**时间：** 约 75 分钟

## 问题所在

在 H100 上对 70B 级模型的解码吞吐量通常为每秒 40-80 个词元。每个词元需要完整地从 HBM 读取所有模型权重并做一次前向传播。你无法在不改变输出的情况下缩小模型，也无法在内存限制之外增加批大小。除非能让模型每次前向传播输出多个词元，否则就此陷入僵局。

自回归生成看起来天生是串行的：`x_{t+1} = sample(p(· | x_{1:t}))`。但存在并发的机会——如果有一个廉价的预测器能说"接下来 4 个词元可能是 [a, b, c, d]"，你就可以在**大模型的单次前向传播**中验证所有 5 个位置，并接受最长的匹配前缀。

Leviathan、Kalai、Matias（2023，《通过投机解码实现 Transformer 快速推理》）通过一个聪明的接受/拒绝规则使这一机制精确化，且保留了目标模型的采样分布——输出分布完全相同，速度提升 2-4×。

## 概念讲解

### 双模型设置

- **目标模型** `M_p`：你真正想要从中采样的大型、慢速、高质量模型，分布为 `p(x)`
- **草稿模型** `M_q`：小型、快速、质量较低的模型，分布为 `q(x)`，比目标模型小 5-30 倍

每步操作：

1. 草稿模型自回归地提出 `K` 个词元：`x_1, x_2, ..., x_K ~ q`
2. 目标模型对所有 `K+1` 个位置并行运行**一次**前向传播，为每个提议词元产生 `p(x_k)`
3. 从左到右用改进的拒绝采样规则（见下）逐一接受/拒绝，接受最长的匹配前缀
4. 若某个词元被拒绝，从修正分布中采样替换词元并停止；否则从 `p(· | x_1...x_K)` 采样一个奖励词元

若草稿与目标完全匹配，每次目标前向传播得到 K+1 个词元；若草稿在位置 1 就错了，只得到 1 个词元。

### 精确性规则

投机解码在分布上**可证明等价于从 p 采样**。拒绝规则如下：

```
对每个草稿词元 x_t：
    r ~ Uniform(0, 1)
    若 r < p(x_t) / q(x_t)：
        接受 x_t
    否则：
        从残差分布采样替换：(p - q)+ / ||(p - q)+||_1
        停止
```

其中 `(p - q)+` 表示逐点差的正数部分。当草稿与目标一致（`p ≈ q`）时，接受率接近 1；当它们不同时，残差分布的构造保证整体样本仍然精确服从 `p`。

**贪心情形：** 对于温度=0 的采样，只需检查 `argmax(p) == x_t`。若是则接受，否则输出 `argmax(p)` 并停止。

### 预期加速比

若草稿模型的词元级接受率为 `α`，每次目标前向传播产生的预期词元数为：

```
E[tokens] = (1 - α^{K+1}) / (1 - α)        # K = 草稿长度，α ∈ [0, 1]
```

当 `α = 0.8, K = 4` 时：`(1 - 0.8^5)/(1 - 0.8) = 3.36` 个词元每次前向。单次目标前向大约需要 `cost_q * K + cost_p`（K 次草稿步骤加一次目标验证）。若 `cost_p >> cost_q * K`，吞吐量加速比约为 `3.36×`。

唯一真正的参数是 `α`，它完全取决于草稿与目标的对齐程度。好的草稿是一切。

### 训练草稿模型：蒸馏

随机小型模型是糟糕的草稿。标准方案是从目标模型蒸馏：

1. 选择小型架构（对于 70B 目标约 10 亿，对于 7B 目标约 5 亿）
2. 对大量文本语料运行目标模型，存储其下一词元分布
3. 用 KL 散度对目标分布（而非真实词元）训练草稿模型

结果：在代码上 `α` 通常为 0.6-0.8，在自然语言对话上为 0.7-0.85，生产中加速 2-3×。

### EAGLE：树形起草 + 特征复用

Li、Wei、Zhang、Zhang（2024，《EAGLE：投机采样需要重新思考特征不确定性》）发现标准投机解码的两个低效之处：

1. 草稿做 K 次串行步骤，每次都是完整栈。但草稿可以复用目标最近验证时的特征（隐藏状态）——目标已经计算了丰富的表示，草稿却要从头重新推导。
2. 草稿输出线性链。若草稿能输出一棵候选树（每个节点多个猜测），目标的单次前向传播就可以用树形注意力掩码并行验证多条候选路径，并选择最长的被接受分支。

EAGLE-1 的改变：
- 草稿输入 = 位置 t 处目标的最后一层隐藏状态，而非原始词元
- 草稿架构 = 1 个 Transformer 解码器层（而非独立的小型模型）
- 输出 = 每个深度 K = 4-8 个候选的树，深度 4-6

EAGLE-2（2024）增加了动态树拓扑：在草稿不确定的地方树变宽，在确定的地方保持窄，在不增加验证代价的情况下提升 `α_effective`。

EAGLE-3（Li 等人，2025，《EAGLE-3：通过训练时测试扩展大语言模型的推理加速》）去掉了固定的顶层特征依赖，并用新的"测试时模拟"损失训练草稿——在匹配目标测试时分布的输出上训练草稿，而非教师强制的训练分布。接受率从 EAGLE-2 的 0.75 提升到 0.82，每次验证的平均词元数从 3.0 提升到 4.5。

### 树形注意力验证

当草稿输出一棵树时，目标模型使用**树形注意力掩码**在一次前向传播中验证它——这是一个编码树拓扑而非纯线性的因果掩码，每个词元只关注树中的祖先节点。验证传播仍然是一次前向传播、一次矩阵乘法，拓扑掩码只需要额外几个 KV 条目。

```
        root
       /    \
      a      b
     / \    / \
    c  d   e   f
```

若 `a, b` 是竞争的第一词元候选，`c, d, e, f` 是第二词元候选，所有六个位置在一次前向传播中被验证。输出是任意被接受路径上的最长前缀。

### 何时胜出，何时不胜

**胜出：**
- 具有可预测文本的对话/补全（代码、常见英文、结构化输出）——`α` 较高
- 解码期间 GPU 计算未被充分利用的场景（内存受限阶段）——树形起草利用可用的 FLOPs

**失败/无收益：**
- 高随机性输出（高温度的创意写作）——`α` 下降到接近 `1/|vocab|`
- 非常高并发的批处理服务——批处理已经用满 FLOPs，树形验证几乎没有空间
- 目标模型非常小，草稿不会小多少

生产场景通常报告：对话 2-3× 的墙上时间加速，代码生成 3-5×，创意写作接近零。

## 动手实践

`code/main.py` 包含：

- 参考实现 `speculative_decode(target, draft, prompt, K, temperature)`，实现精确拒绝规则并验证其保留目标分布（经验 KL < 0.01 vs 朴素目标采样）
- 构建深度为 K、带 top-p 分支的树的 EAGLE 风格树形起草器
- 生成正确因果模式的树形注意力掩码构建器
- 在小型 LM 上运行两者的接受率测试框架（将一个 GPT-2-small 蒸馏为 GPT-2-medium 目标的草稿）

```python
def speculative_step(p_target, q_draft, K, temperature=1.0):
    """投机解码的一轮。返回被接受词元的列表。"""
    # 1. 草稿 K 个词元
    draft_tokens = []
    q_probs = []
    state = draft_state_init()
    for _ in range(K):
        probs = softmax(q_draft(state) / temperature)
        t = np.random.choice(len(probs), p=probs)
        draft_tokens.append(t)
        q_probs.append(probs[t])
        state = draft_step(state, t)

    # 2. 目标在每个草稿位置 + 1 个额外位置并行计算 p
    p_probs_all = target_forward_batched(p_target, draft_tokens, temperature)

    # 3. 从左到右接受/拒绝
    accepted = []
    for k, tok in enumerate(draft_tokens):
        r = np.random.uniform()
        if r < p_probs_all[k][tok] / q_probs[k]:
            accepted.append(tok)
        else:
            residual = np.maximum(p_probs_all[k] - q_probs[k], 0)
            residual /= residual.sum()
            accepted.append(np.random.choice(len(residual), p=residual))
            return accepted
    # 4. 所有 K 个被接受 → 从目标采样奖励词元
    accepted.append(np.random.choice(len(p_probs_all[-1]), p=p_probs_all[-1]))
    return accepted
```

## 实际运用

- **vLLM** 和 **SGLang** 提供一流的投机解码支持。参数：`--speculative_model`、`--num_speculative_tokens`。通过 `--spec_decoding_algorithm eagle` 参数支持 EAGLE-2/3。
- **NVIDIA TensorRT-LLM** 原生支持 Medusa 和 EAGLE 树形解码。
- **参考草稿模型**：`Qwen/Qwen3-0.6B-spec`（用于 Qwen3-32B 的草稿），`meta-llama/Llama-3.2-1B-Instruct-spec`（用于 70B 的草稿）。
- **Medusa 头**（Cai 等人，2024，《Medusa：带多解码头的简单 LLM 推理加速框架》）：在目标模型上添加 K 个并行预测头，而非单独的草稿模型。部署更简单，但接受率略低于 EAGLE。

## 练习

1. 实现精确拒绝规则并经验性地验证它。通过 `speculative_decode` 和朴素目标采样各运行 1 万个样本，计算两个输出分布之间的全变差距离，应 < 0.01。

2. 计算加速比公式。给定固定的 `α` 和 `K`，绘制每次目标前向传播的预期词元数。找出 α ∈ {0.5, 0.7, 0.9} 的最优 K。

3. 训练一个微型草稿。取一个 1.24 亿 GPT-2 目标，在 1 亿词元上用 KL 损失蒸馏一个 3000 万 GPT-2 草稿。在留出文本上测量 `α`，预期 0.6-0.7。

4. 实现 EAGLE 风格的树形起草。草稿在每个深度输出 top-3 分支而非链。构建树形注意力掩码，验证目标接受任意被接受路径上的最长分支。

5. 测量失败模式。在温度=1.5（高随机性）下运行投机解码，证明 α 崩溃，算法由于草稿开销比朴素解码更慢。

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|-----------|---------|
| 目标模型 | "大模型" | 你想从中采样的慢速、高质量模型（p 分布）|
| 草稿模型 | "投机者" | 小型、快速的预测器（q 分布）；比目标小 5-30 倍 |
| K / 草稿长度 | "前瞻长度" | 每次验证传播中投机的词元数 |
| α / 接受率 | "命中率" | 草稿提议被接受的逐词元概率 |
| 精确拒绝规则 | "接受测试" | r < p/q 的比较，保留目标的分布 |
| 残差分布 | "修正的 p-q" | `(p - q)+ / ||(p - q)+||_1`，拒绝时从中采样的分布 |
| 树形起草 | "分支投机" | 草稿输出候选树，用树形结构注意力掩码在一次传播中验证 |
| 树形注意力掩码 | "拓扑掩码" | 编码树拓扑的因果掩码，使每个节点只关注其祖先 |
| Medusa 头 | "并行头" | 目标模型上的 K 个额外预测头；无需独立草稿模型 |
| EAGLE 特征复用 | "隐藏状态草稿" | 草稿输入是目标的最后一层隐藏状态而非原始词元，缩小草稿 |
| 测试时模拟损失 | "EAGLE-3 训练" | 在匹配目标测试时分布的输出上训练草稿，而非教师强制 |
