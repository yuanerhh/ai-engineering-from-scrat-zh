# 差分注意力（V2）

> Softmax 注意力会将少量概率分散到每个不匹配的词元上。在 10 万个词元时，这种噪声会积累并淹没信号。差分 Transformer（Ye 等人，ICLR 2025）通过将注意力计算为两个 softmax 的差来解决这个问题，从而消去共享的噪声底。DIFF V2（微软，2026 年 1 月）是生产栈的重写版本：解码延迟与基线 Transformer 持平，不需要自定义内核，兼容 FlashAttention。本课从 V1 到 V2 端到端讲解，并提供一个可在标准库 Python 中运行的差分操作玩具实现。

**类型：** 构建
**语言：** Python（标准库）
**前置条件：** Phase 7 · 第02课（自注意力），Phase 7 · 第15课（注意力变体），Phase 10 · 第14课（架构详解）
**时间：** 约 60 分钟

## 学习目标

- 精确阐述 softmax 注意力为何存在噪声底，以及为何噪声随上下文长度增长
- 推导差分注意力公式，并解释为何相减能消去共享噪声分量同时保留信号
- 梳理 V1 到 V2 的差异：什么变快了、什么变简单了、什么变更稳定了，以及每项改变对于生产预训练为何是必要的
- 用纯 Python 从头实现差分注意力，并在合成信号+噪声查询上经验性地验证噪声消除性质

## 问题所在

标准 softmax 注意力有一个数学性质，在规模化时会变成操作上的麻烦。对于查询 `q`，注意力权重为 `softmax(qK^T / sqrt(d))`。Softmax 永远无法产生精确的零——每个不匹配的词元都会获得一些正质量。这部分残差质量就是噪声，并且随上下文长度增长。在 12.8 万个词元时，即使每个不匹配词元只获得 0.001% 的概率，127,999 个词元加起来约贡献总量的 12%。模型必须学会绕开一个随上下文增长的噪声底。

经验上这表现为注意力头干扰：长上下文 RAG 中的幻觉引用、10 万词元检索任务上的"迷失在中间"失败，以及超过 3.2 万词元的 needle-in-haystack 基准上微妙的精度下降。差分 Transformer 论文（arXiv:2410.05258，ICLR 2025）测量了这一差距：DIFF Transformer 相比同等规模的基线达到更低的困惑度、更高的长上下文精度和更少的幻觉。

DIFF V1 有三个问题使其无法进入前沿预训练管线：值缓存在每个解码步骤需要加载两次，需要破坏 FlashAttention 兼容性的自定义 CUDA 内核，以及逐头 RMSNorm 在 70B 以上规模的长期训练中导致不稳定。DIFF V2（微软 unilm 博客，2026 年 1 月 20 日）修复了所有三个问题。本课将讲解两个版本，构建差分操作符，并在玩具查询上对噪声消除进行基准测试。

## 概念讲解

### Softmax 的噪声底

对于查询 `q` 和键 `K = [k_1, ..., k_N]`，注意力权重为：

```
w_i = exp(q . k_i / sqrt(d)) / sum_j exp(q . k_j / sqrt(d))
```

没有任何 `w_i` 是零。如果 `k_i` 与 `q` 完全无关，分数 `q . k_i` 不为 0——它在零附近以方差 `||q||^2 / d` 波动。经过 softmax 归一化后，每个无关词元仍然对加权和贡献 `O(1/N)`。所有无关词元的总贡献为 `O((N-1)/N) = O(1)`——这不是一个小量。

模型真正想要的是类似 hard top-k 的东西：匹配词元上有高权重，其他地方接近零。Softmax 太平滑了，无法直接做到这点。

### 差分的思路

将每个头的 Q 和 K 投影分成两部分：Q = (Q_1, Q_2)，K = (K_1, K_2)，计算两个注意力图：

```
A_1 = softmax(Q_1 K_1^T / sqrt(d))
A_2 = softmax(Q_2 K_2^T / sqrt(d))
```

输出：

```
DiffAttn = (A_1 - lambda * A_2) V
```

相减消去了两个图共享的任何噪声分布。如果两个图都在 12.7 万个无关词元上有大致均匀的权重（在随机初始化时会如此），这些就抵消了。信号——在少数真正相关词元上的峰值权重——只会在两个图以相同幅度出现时才抵消，而一旦模型训练完成就不会这样。

`lambda` 是每个头的可学习标量，参数化为 `lambda = exp(lambda_q1 · lambda_k1) - exp(lambda_q2 · lambda_k2) + lambda_init`，可以为负。`lambda_init` 默认为小正数，如 0.8。

### 为何这与头戴式噪声消除相似

想象两个嘈杂的麦克风录制同一个声音。两者都拾取说话者加上相关的背景噪声。相减之后共享噪声消失，声音保留下来，因为两个信号在相位或幅度上有足够的差异，阻止了完全抵消。逐头的 `lambda` 正好学习这个平衡。

### V1 vs V2：差异对比

V1 保持参数量与基线 Transformer 相同。为了每个头获得两个查询，头维度减半。这损失了头的表达能力，更痛苦的是——每头的值缓存减半。解码时每步需要加载值缓存两次（每个 softmax 分支一次）。结果：尽管参数量相同，解码比基线慢。

V2 将查询头数翻倍，保持 KV 头数不变（从上投影借参数）。头维度与基线相同。相减后，额外的维度被投影回去以匹配基线 Transformer 的 O_W 投影。三件事同时发生：

1. 解码速度与基线持平（KV 缓存只加载一次）
2. FlashAttention 原封不动地运行（不需要自定义内核）
3. 解码时的算术强度提升（每字节从 HBM 加载获得更多计算）

V2 还去掉了 V1 用于稳定相减的逐头 RMSNorm。在 70B 级别的预训练规模下，该 RMSNorm 会在后期训练中造成不稳定。V2 用更简单的初始化方案替代它，在无需额外模块的情况下保持训练稳定性。

### 何时应用

| 工作负载 | 收益 |
|---------|-----|
| 长上下文 RAG（64k+）| 更干净的注意力图，更少的幻觉引用 |
| Needle-in-haystack 基准 | 超过 32k 时精度显著提升 |
| 多文档问答 | 减少跨文档干扰 |
| 8k 上下文的代码补全 | 边际收益，不值得改变架构 |
| 短对话（< 4k）| 与基线实际上没有区别 |

价值随上下文长度增长。在 4k 词元时，噪声底小到标准注意力足以应付；在 128k 时，噪声底正在伤害你。

### 与 2026 年其他旋钮的兼容性

| 特性 | 与 DIFF V2 兼容？ |
|-----|----------------|
| GQA | 是（V2 增加 Q 头，不增加 KV 头）|
| MLA（DeepSeek）| 原则上是，但没有发表论文结合两者 |
| MoE | 是（注意力独立于 MLP 块）|
| RoPE | 是（不变）|
| YaRN / 长上下文扩展 | 是（正是 DIFF 最有帮助的地方）|
| FlashAttention | V2 中是（V1 中否）|
| 投机解码 | 是（注意力改变对投机解码循环不可见）|

## 动手实践

`code/main.py` 用纯 Python 实现差分注意力。一个具有已知信号+噪声结构的玩具查询让你直接测量噪声消除比率。

### 第 1 步：标准 softmax 注意力

标准库矩阵操作：列表的列表、手动矩阵乘法、带数值稳定性减法（减去最大值）的 softmax。

```python
def softmax(row):
    m = max(row)
    exps = [math.exp(x - m) for x in row]
    s = sum(exps)
    return [e / s for e in exps]
```

### 第 2 步：将 Q、K 分成两半

V1 风格：头维度减半。V2 风格：保持头维度，头数翻倍。玩具实现出于教学清晰度采用 V1 风格——数学完全相同，只是记账方式不同。

### 第 3 步：两个 softmax 分支 + 相减

```python
A1 = [softmax([dot(q1, k) / scale for k in K1]) for q1 in Q1]
A2 = [softmax([dot(q2, k) / scale for k in K2]) for q2 in Q2]
diff_weights = [[a1 - lam * a2 for a1, a2 in zip(r1, r2)] for r1, r2 in zip(A1, A2)]
out = [[sum(w * v[j] for w, v in zip(row, V)) for j in range(d_v)] for row in diff_weights]
```

注意：输出权重可以为负，这没问题——值缓存仍然能处理有符号的贡献，后续的 V 投影会吸收符号。

### 第 4 步：噪声消除测量

构建一个长度为 1024 的合成序列，将信号词元放在已知位置，其余用噪声填充。计算（a）标准 softmax 注意力在信号位置的权重和（b）差分注意力权重。测量两者的信号噪声比。DIFF 注意力可靠地产生更高的信噪比，根据两个分支的训练差异程度提升 3x-10x。

### 第 5 步：V1 vs V2 参数统计

给定配置（hidden=4096，heads=32，d_head=128），打印：

- 基线 Transformer：Q、K、V 各大小 `hidden * hidden`，MLP 为 4 * hidden
- DIFF V1：Q、K 各大小 `hidden * hidden`，V 大小 `hidden * hidden`（不变），头维度内部减半。添加逐头 `lambda` 参数（O(heads * d_head)）
- DIFF V2：Q 大小 `2 * hidden * hidden`，K 大小 `hidden * hidden`，V 大小 `hidden * hidden`。相减后额外维度在 O_W 投影前被投影回去。添加相同的 `lambda` 参数

玩具程序测量 V2 的额外参数代价（每个注意力块约多 `hidden * hidden`）并打印出来。

## 实际运用

截至 2026 年 4 月，DIFF V2 尚未在所有生产推理服务器中发布，但 vLLM 和 SGLang 的集成正在进行中。目前该模式出现在：

- 微软内部面向长上下文的生产模型
- 以 256k+ 上下文为目标的若干开放模型训练运行中的研究复现
- 在交替层上结合 DIFF 注意力与滑动窗口注意力的混合架构

2026 年何时适合使用：

- 从头训练以 64k+ 有效上下文为目标的新模型——从一开始就加入差分注意力；事后重训代价高昂
- 在"迷失在中间"失败主导评估的长上下文模型上进行微调——Q 投影上的 LoRA 可以近似 DIFF 结构

何时不适合：

- 你在服务一个具有稳定长上下文性能的预训练密集模型——重训代价很少能从已有权重中收回
- 你的上下文始终低于 16k——噪声底可以忽略不计

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|-----------|---------|
| 差分注意力 | "两个 softmax 相减" | 将 Q、K 分成两半，计算两个 softmax 图，从第一个中减去第二个（乘以 lambda），然后乘以 V |
| 噪声底 | "Softmax 的非零尾部" | Softmax 对每个无关词元分配的 O(1/N) 权重，在长上下文中累计为 O(1) |
| lambda | "相减缩放因子" | 每头可学习标量，参数化为 `exp(lq1·lk1) - exp(lq2·lk2) + lambda_init`；可以为负 |
| DIFF V1 | "ICLR 2025 版本" | 原始差分 Transformer；头维度减半以保持参数量，需要自定义内核，解码更慢 |
| DIFF V2 | "2026 年 1 月修复版" | Q 头翻倍，KV 头不变；解码速度与基线持平，兼容 FlashAttention |
| 逐头 RMSNorm | "V1 稳定器" | V1 在差分后应用的额外归一化；V2 为防止后期训练不稳定而将其移除 |
| 信噪比 | "多少注意力被浪费了" | 真实信号位置上的权重与无关位置上平均权重之比 |
| 迷失在中间 | "长上下文失败模式" | 长上下文中间文档检索精度下降的经验现象——DIFF 注意力可以减少这一问题 |
| 算术强度 | "每字节加载的 FLOPs" | V2 通过每次 KV 加载翻倍查询来在解码时提升的比率；对内存受限的解码很重要 |
