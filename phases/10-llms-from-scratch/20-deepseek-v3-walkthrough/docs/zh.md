# DeepSeek-V3 架构详解

> Phase 10 · 第14课列举了每个开放模型调整的六个架构旋钮。DeepSeek-V3（2024 年 12 月，共 6710 亿参数，激活 370 亿）调整了全部六个旋钮，并额外增加了四个：多头潜在注意力、无辅助损失的负载均衡、多词元预测和 DualPipe 训练。本课从头到尾阅读 DeepSeek-V3 的架构，并从发布的配置中推导出每个参数量。学完之后，你能解释为何 671B/37B 的比例是正确的押注，以及为何 MLA + MoE 组合比单独使用任何一个在前沿更优。

## 学习目标

- 阐述四个 DeepSeek 特有创新（MLA、MTP、无辅助损失路由、DualPipe），并指出每个针对的是架构/训练栈的哪个部分

## 问题所在

DeepSeek-V3 是第一个架构上与 Llama 家族有实质性差异的前沿开放模型。Llama 3 405B 是"调整了六个旋钮的 GPT-2"。DeepSeek-V3 是调整了全部六个旋钮再加四个的 GPT-2。阅读 Llama 3 配置是阅读 DeepSeek 配置的热身，但深层结构——注意力块的形状、路由逻辑、训练时目标——差异足够大，需要单独的详解。

学习它的回报：DeepSeek-V3 的开放权重发布改变了开放模型"前沿能力"的含义。这一架构是 2026 年许多训练运行正在复制的蓝图，理解它是任何涉及前沿 LLM 训练或推理角色的基本门槛。

## 概念讲解

### 不变的核心，再次强调

DeepSeek-V3 仍然是自回归的，仍然堆叠解码器块，每个块仍然有注意力加 MLP 加两个 RMSNorm，仍然在 MLP 中使用 SwiGLU，仍然使用 RoPE，使用 Pre-norm 和权重绑定嵌入——与每个 Llama 或 Mistral 相同的基线。

### 转折点：MLA 取代 GQA

从第14课你已知道 GQA 通过在 Q 头组间共享 K 和 V 来缩小 KV 缓存。多头潜在注意力（MLA）走得更远：K 和 V 被压缩为共享的低秩潜在表示（`kv_lora_rank`），然后在计算时逐头解压。KV 缓存只存储潜在表示——通常是每个词元每层 512 个浮点数，而非 8 × 128 = 1024 个浮点数。

在 128k 上下文下，DeepSeek-V3 使用 MLA（每个词元每层一个共享潜在 `c^{KV}`，K 和 V 均通过可被后续矩阵乘吸收的上投影从该潜在导出）：

```
kv_cache = num_layers * kv_lora_rank * max_seq_len * bytes_per_element
         = 61 * 512 * 131072 * 2
         = 7.6 GB
```

假设的 GQA 基线（Llama 3 70B 形状，8 个 KV 头，头维度 128）需要：

```
kv_cache = 2 * 61 * 8 * 128 * 131072 * 2
         = 30.5 GB
```

在 128k 上下文下，MLA 比 Llama-3-70B 风格的 GQA 缓存小 4 倍。

权衡：MLA 在每次注意力计算时增加了逐头解压步骤，但额外计算相比节省的带宽微乎其微，对长上下文推理净收益明显。

### 路由：无辅助损失的负载均衡

MoE 路由器决定哪 top-k 个专家处理每个词元。朴素路由器会将过多工作集中在少数专家上，其他专家闲置。标准解决方案：添加惩罚负载不均衡的辅助损失项。这有效但会略微降低主任务性能。

DeepSeek-V3 引入了无辅助损失的方案：在路由 logits 上添加逐专家偏置项，在训练期间按简单规则调整——若专家 `e` 过载则降低 `bias_e`，若负载不足则增加。没有额外损失项，训练保持干净，专家负载保持均衡。

对主损失的影响：不可测量。对 MoE 架构的影响：更简洁，没有需要调优的辅助损失超参数。

### MTP：更密集的训练 + 免费草稿

从第18课你已知道 DeepSeek-V3 添加了 D=1 的 MTP 模块，预测两个位置后的词元。推理时，训练好的模块被复用为接受率超过 80% 的投机解码草稿。训练时，每个隐藏状态在 D+1=2 个目标上被监督，提供更密集的信号。

参数量：在 671B 主模型之上额外 140 亿参数，开销 2.1%。

### 训练：DualPipe

从第19课你已知道 DualPipe 是一种双向流水线，将前向和反向分块与跨节点 all-to-all 通信重叠。在 DeepSeek-V3 的 2048 块 H800 规模下，大约恢复了 1F1B 会因流水线气泡损失的 24.5 万 GPU 小时。

### 配置字段详解

DeepSeek-V3 配置（简化版）：

```
hidden_size: 7168
intermediate_size: 18432   （密集 MLP 隐藏大小，用于前几层）
moe_intermediate_size: 2048 （专家 MLP 隐藏大小）
num_hidden_layers: 61
first_k_dense_layers: 3    （前 3 层使用密集 MLP）
num_attention_heads: 128
num_key_value_heads: 128   （MLA 下形式上等于 num_heads，
                           真正的压缩在 kv_lora_rank 中）
kv_lora_rank: 512          （MLA 潜在维度）
num_experts: 256            （每个块的 MoE 专家数量）
num_experts_per_tok: 8      （top-8 路由）
shared_experts: 1           （每个块始终激活的共享专家）
max_position_embeddings: 163840
rope_theta: 10000.0
vocab_size: 129280
mtp_module: 1               （深度 1 处的 1 个 MTP 模块）
```

解析：

- `hidden_size=7168`：嵌入维度
- `num_hidden_layers=61`：总块深度
- `first_k_dense_layers=3`：前 3 个块使用大小 18432 的密集 MLP，其余 58 个使用 MoE
- `num_attention_heads=128`：128 个查询头
- `kv_lora_rank=512`：K 和 V 被压缩到此潜在维度，然后逐头解压
- `num_experts=256, num_experts_per_tok=8`：每个 MoE 块有 256 个专家，路由 top-8
- `shared_experts=1`：在 256 个路由专家之上，1 个始终激活的专家处理每个词元——可以理解为"密集底层"，确保每个词元都能获得可靠的基础
- `moe_intermediate_size=2048`：每个专家 MLP 的隐藏大小，比密集 MLP 小，因为有 256 个专家

### 参数统计

完整计算在 `code/main.py` 中。关键数据：

- 嵌入：`vocab * hidden = 129280 * 7168 ≈ 9.3 亿`
- 前 3 个密集块：带 MLA 的注意力（每块约 1.44 亿）+ 密集 MLP（每块约 2.6 亿）+ 归一化，共约 12 亿
- 58 个 MoE 块：带 MLA 的注意力（1.44 亿）+ 256 个专家（每个 3000 万）+ 1 个共享专家（3000 万）+ 归一化，含所有专家约每块 79.5 亿，58 个 MoE 块共 4610 亿
- MTP 模块：140 亿

合计：核心架构约 4760 亿 + 140 亿 MTP + 发布的 6710 亿数字还包含额外的结构参数（偏置张量、专家特定组件、共享专家缩放等）。计算器的复现数字与发布值相差 3-5%，差距来自 DeepSeek 报告附录第 2 节记录的细粒度统计。

每次前向传播的激活参数：

- 注意力：每层 1.44 亿 × 61 层 = 88 亿（所有层都激活）
- MLP 激活：前 3 层密集（3 × 2.6 亿 = 7.8 亿），58 个 MoE 层各激活 8 个路由专家 + 1 个共享专家 + 路由开销，每层激活 MLP 约 2.6 亿，共 3 × 2.6 亿 + 58 × 2.6 亿 ≈ 159 亿
- 嵌入 + 归一化：12 亿
- 总激活：约 260 亿核心 + 140 亿 MTP（训练但推理时不总是运行）≈ 370 亿

### 671B / 37B 比例

18 倍稀疏比（激活参数是总参数的 5.5%）。DeepSeek-V3 是发布了开放权重中最稀疏的前沿 MoE 模型。Mixtral 8x7B 的 13/47（28%）密集得多，Llama 4 Maverick 的 17B/400B（4.25%）与之相当。DeepSeek 的押注：在前沿规模下，更多专家加更低激活比例，相比每激活 FLOP 能产生更好的质量。

### DeepSeek-V3 的定位

| 模型 | 总参数 | 激活参数 | 比例 | 注意力 | 创新点 |
|-----|-------|---------|-----|-------|-------|
| Llama 3 70B | 70B | 70B | 100% | GQA 64/8 | — |
| Llama 4 Maverick | 400B | 17B | 4.25% | GQA | — |
| Mixtral 8x22B | 141B | 39B | 27% | GQA | — |
| DeepSeek V3 | 671B | 37B | 5.5% | MLA 512 | MLA + MTP + 无辅助损失 + DualPipe |
| Qwen 2.5 72B | 72B | 72B | 100% | GQA 64/8 | YaRN 扩展 |

### 后续：R1、V4

DeepSeek-R1（2025）是在 V3 主干上进行的推理训练运行，使用相同的架构。改变的是后训练方案（在可验证任务上的大规模强化学习），而非预训练架构。

DeepSeek-V4（如果发布）预计将保留 MLA + MoE + MTP，并添加 DSA（DeepSeek 稀疏注意力），即第17课中 NSA 的后继。传承稳定：架构层面的创新积累，每个版本调整额外的旋钮。

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|-----------|---------|
| MLA | "多头潜在注意力" | 将 K 和 V 压缩进共享的低秩潜在（kv_lora_rank，通常 512），在计算时逐头解压；KV 缓存只存储潜在 |
| kv_lora_rank | "MLA 压缩维度" | K 和 V 共享潜在的大小；DeepSeek-V3 使用 512 |
| 前 k 个密集层 | "早期层保持密集" | 最初几个 MoE 模型层跳过 MoE 路由器，运行密集 MLP 以确保稳定性 |
| num_experts_per_tok | "Top-k 路由" | 每个词元激活多少个路由专家；DeepSeek-V3 使用 8 |
| 共享专家 | "始终激活的专家" | 无论路由如何，处理每个词元的专家；DeepSeek-V3 使用 1 个 |
| 无辅助损失路由 | "偏置调整负载均衡" | 训练期间调整的逐专家偏置项，使专家负载均衡而不添加损失项 |
| MTP 模块 | "额外预测头" | 从 h^(1) 和 E(t+1) 预测 t+2 的 Transformer 块；更密集的训练，免费的投机解码草稿 |
| DualPipe | "双向流水线" | 将前向/反向计算与跨节点 all-to-all 重叠的训练调度 |
| 激活参数比例 | "稀疏度" | active_params / total_params；DeepSeek-V3 达到 5.5% |
| FP8 训练 | "8 位训练" | 训练存储和许多计算操作使用 FP8；相比 BF16 大约减半内存，质量代价很小 |
