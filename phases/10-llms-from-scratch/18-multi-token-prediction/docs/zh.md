# 多词元预测（MTP）

> 从 GPT-2 到 Llama 3，每个自回归 LLM 在每个位置训练一个损失：预测下一个词元。DeepSeek-V3 在每个位置添加了第二个损失：预测之后的词元。这额外的 140 亿参数（在 6710 亿参数模型上）通过梯度流蒸馏回主模型，训练好的 MTP 头在推理时被复用为投机解码的起草器，接受率超过 80%。生成吞吐量免费获得 1.8× 提升。本课从头构建 DeepSeek 技术报告中的顺序 MTP 模块，计算损失和共享头参数布局，并解释为何 MTP 保持了因果链，而 Gloeckle 等人最初的并行 MTP 打破了它。

**类型：** 构建
**语言：** Python（标准库）
**前置条件：** Phase 10 · 第04课（预训练微型 GPT），Phase 10 · 第15课（投机解码）
**时间：** 约 60 分钟

## 学习目标

- 阐述 MTP 训练目标，并推导跨预测深度的联合损失
- 解释 Gloeckle 等人的并行 MTP 头（2024）与 DeepSeek-V3 的顺序 MTP 模块之间的区别，以及为何顺序设计保留了因果链
- 计算向预训练运行中添加 MTP 模块的参数和内存开销
- 从头实现一个 MTP 模块：共享嵌入、每深度 Transformer 块、投影和共享输出头

## 问题所在

下一词元预测是标准的 LLM 训练目标。每个隐藏状态被监督以预测恰好一件事：紧随其后的词元。这是一个出人意料弱的信号。序列中的大部分信息延伸到一个词元之外——结构、连贯性、事实性、算术流。模型必须通过在数万亿词元上积累许多单词元信号来学习这些。

MTP 提出的问题是：如果每个隐藏状态同时被监督以预测多个未来词元会怎样？Gloeckle 等人（Meta，2024）证明这有帮助。他们的实现在主干上放了几个独立的输出头，每个预测不同的偏移量。并行、简单，但各头看到相同的隐藏状态而没有任何层次化细化——预测没有因果链接，所以不能用于投机解码。

DeepSeek-V3（2024 年 12 月）将 MTP 重新设计为顺序模块，在每个预测深度保持因果链。模型从 `h_i^(0)` 预测 `t+1`，然后从将 `h_i^(0)` 与 `E(t+1)` 嵌入结合的新隐藏状态 `h_i^(1)` 预测 `t+2`，以此类推。每个深度都是自己的小型 Transformer 块。共享的嵌入和共享的输出头使参数开销保持适中。在 DeepSeek-V3 的规模下，MTP 模块在 6710 亿主模型权重之上额外增加了 140 亿参数。这 2% 的开销换来了更密集的训练信号，以及推理时现成的投机解码草稿。

本课从头构建单个 MTP 模块和 D 深度损失。数学简洁，实现约 150 行。

## 概念讲解

### 顺序 MTP 配方

DeepSeek-V3 在主模型之上添加 `D` 个 MTP 模块。每个模块 `k`（`k = 1..D`）预测深度 `k` 处的词元——即给定直到位置 `i` 的前缀，预测 `t_{i+k}`。

模块 `k` 由以下组成：

- 带自身注意力和 MLP 的 Transformer 块 `T_k`
- 投影矩阵 `M_k`，将上一深度的隐藏状态与下一深度真实词元的嵌入结合
- 共享嵌入 `E`（与主模型相同）
- 共享输出头 `Out`（与主模型相同）

训练时，对于直到位置 `i` 的前缀，每深度隐藏状态为：

```
h_i^(0) = 位置 i 处的主模型主干
h_i^(k) = T_k( M_k * concat(RMSNorm(h_i^(k-1)), RMSNorm(E(t_{i+k}))) )   k >= 1
```

每深度预测为：

```
logits_{i+k} = Out(h_i^(k-1))   k = 1..D
```

每深度损失为对真实 `t_{i+k}` 的交叉熵：

```
L_k = CE(logits_{i+k}, t_{i+k})
```

跨深度的联合损失：

```
L_MTP = (lambda / D) * sum_{k=1..D} L_k
```

`lambda` 是一个小的权重因子——DeepSeek-V3 在训练前 10% 使用 0.3，之后使用 0.1。总训练损失为 `L_main + L_MTP`。

### 为何是顺序而非并行

Gloeckle 的原始并行 MTP 有 D 个输出头，每个直接应用于 `h_i^(0)`。每个头从相同的主干隐藏状态预测 `t_{i+k}`，这样训练没问题，但预测不相互条件化——你不能用 `head_1` 的输出帮助 `head_2`，各头并行触发。

DeepSeek-V3 的顺序设计从 `h_i^(k-1)` 加上实际下一词元嵌入 `E(t_{i+k})` 构建 `h_i^(k)`。这保留了因果链：要预测 `t_{i+k+1}`，深度 `k+1` 处的模块能看到 `t_{i+k}` 处是什么。这在结构上与自回归解码器消费自身输出完全相同——使得 MTP 模块可以直接用作投机解码的起草器。

推理时：将 `h_i^(k-1)` 和起草的 `t_{i+k}` 输入模块 `k+1`，获得对 `t_{i+k+1}` 的预测，然后重复。这正是 EAGLE 风格的草稿，使用训练好的 MTP 模块作为草稿网络。DeepSeek-V3 报告第一个 MTP 模块的接受率超过 80%，加速约 1.8×。

### 参数统计

对于隐藏维度 `h`、词汇量 `V` 的模型：

- 主模型：数十亿参数，加上大小为 `V * h` 的一个输出头
- 共享输出头：复用主模型的头，无额外参数
- 共享嵌入：复用主模型的嵌入，无额外参数
- 每 MTP 模块：
  - 投影 `M_k`：`(2h) * h = 2h^2`
  - Transformer 块 `T_k`：注意力（MHA 约 `4h^2`）加 MLP（SwiGLU 比例 8/3 约 `8h^2`），约每块 `12h^2`

每模块总额外参数：约 `14h^2`。对于 DeepSeek-V3 的 `h = 7168`，D=1 模块：约 `14 * 7168^2 ≈ 7.2 亿` 参数（理论上）。DeepSeek-V3 报告 140 亿——差距主要是 MTP 模块中的 Transformer 块也采用 MoE 结构。

### 投机解码的回报

预训练期间，MTP 模块使训练速度降低约 10%（更多前向计算、额外损失）。回报是双重的：

1. **更密集的训练信号**：每个隐藏状态看到 D+1 个监督目标。在 MMLU、GSM8K、MATH、HumanEval 上的实测效果：DeepSeek-V3 消融研究中一致出现几个百分点的改善。

2. **推理时免费获得投机解码草稿**：MTP 模块已经被训练来预测接下来的几个词元。复用为草稿网络，接受率超过 80%。在这个水平上，N=3 或 N=5 的投机解码给出 1.8× 吞吐量。10% 的训练时间代价在第一次推理时就得到了回报。

### 与 EAGLE 的关系

EAGLE 在预训练之后分别训练一个小型草稿模型。MTP 将草稿融入预训练。两种方法在类似的接受率上收敛，但通过不同的流程：

| 维度 | EAGLE-3 | MTP（DeepSeek-V3）|
|-----|---------|-----------------|
| 训练时机 | 预训练后 | 预训练期间 |
| 与现有权重向后兼容 | 是 | 否（需要重新训练）|
| 草稿参数 | 1-2 个 Transformer 层 | 1 个 Transformer 块 + 投影 |
| 接受率 | 0.88-0.92 | 深度1 超过 0.80 |
| 速度之外的收益 | 仅投机解码 | 更密集的训练信号 + 加速 |

## 动手实践

`code/main.py` 端到端构建单个 MTP 模块：共享嵌入、投影、Transformer 块、共享输出头。然后在短合成序列上计算每深度交叉熵损失，并按组件打印参数量。32 个词元的玩具词汇表使数字可读。

### 第 1 步：共享嵌入表

单个 `vocab_size x hidden` 表被主模型和每个 MTP 模块在每个深度使用，不是第二个副本，而是字面上同一个张量。

### 第 2 步：每深度组合

```python
def combine(prev_hidden, next_token_embed, M_k):
    # 沿特征维度拼接，然后投影到 hidden 维度
    concat = rms_norm(prev_hidden) + rms_norm(next_token_embed)  # 向量加法替代
    projected = matvec(M_k, concat)
    return projected
```

真实 DeepSeek-V3 将两个 RMSNorm 后的向量拼接为 `[2h]`，并用 `h x 2h` 矩阵投影。玩具实现为简洁使用向量加法。

### 第 3 步：深度 k 处的 Transformer 块

自注意力加 MLP。玩具中，单层线性注意力块和 SwiGLU MLP 在不依赖 numpy 的情况下保持结构可见。

### 第 4 步：共享输出头

复用主模型的输出投影，输出词汇表上的 logits。

### 第 5 步：每深度损失

softmax(logits) 对偏移 `k` 处真实词元的交叉熵，用 `lambda / D` 缩放因子跨深度聚合。

### 第 6 步：参数统计

打印总参数量、共享（嵌入、头）参数量和每模块额外参数量，显示 MTP 额外参数与主模型大小的比率。

## 实际运用

MTP 集成在 DeepSeek-V3（2024 年 12 月）和 DeepSeek-R1 系列中。推理时：

- DeepSeek 自己的服务栈开箱即用地将 MTP 模块用作投机解码器
- vLLM 和 SGLang 截至 2026 年 4 月已有 DeepSeek-V3 MTP 的集成路径
- AMD 的 ROCm SGLang 教程展示了特定的 MTP 投机解码配置，在 V3 检查点上实测 1.8× 加速

何时在新的预训练运行中使用 MTP：

- 你掌控完整的预训练管线，并希望获得更密集的训练信号
- 你知道将在规模上服务该模型，并希望免费获得投机解码
- 你的隐藏大小至少为 4096——在 10 亿参数规模下，开销的代价大于收益

何时不适合：

- 微调现有预训练密集模型——MTP 模块未经训练
- 你想要干净基线进行对比的研究模型——MTP 改变了架构

## 练习

1. 运行 `code/main.py`，证明随着合成信号增强，每深度损失单调递减。修改合成序列使用固定模式，验证深度1和深度2的损失均收敛。

2. 计算带 D=1 MTP 模块的密集 70B 模型（hidden 8192，80 层）的参数开销，与 DeepSeek-V3 报告的 140 亿开销对比。解释 DeepSeek 数字更高的原因：MTP Transformer 块也继承了相同的 MoE 结构，使每模块参数量膨胀。

3. 在玩具中实现 D=2：添加第二个 MTP 模块，取 h^(1) 预测 `t_{i+2}`，验证联合损失和参数统计与 DeepSeek 论文的方程 19-21 一致。

4. 将玩具切换为并行 MTP（Gloeckle 风格）：在主隐藏状态上添加 D 个输出头，每个预测不同的偏移量。在相同合成信号上测量每深度损失与顺序版本的对比。顺序版本对于 k > 1 的深度损失应更低，因为它以中间预测为条件。

5. 将训练好的 MTP 模块用作 EAGLE 风格的草稿：在推理时调用模块 k 来提出 `t_{i+k}`，在留出序列上测量这些草稿词元对主模型预测的接受率。若玩具上达到 50%+，你已经复现了 MTP 作为草稿的经验性质。

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|-----------|---------|
| MTP 模块 | "额外损失块" | 预测主模型前 `k` 个位置词元的小型 Transformer 块加投影 |
| 预测深度 | "哪个偏移量" | 整数 `k`，使得模块 `k` 从直到位置 `i` 的前缀预测 `t_{i+k}` |
| 并行 MTP | "Gloeckle 风格" | 相同主干隐藏状态上的 D 个独立头，无条件链 |
| 顺序 MTP | "DeepSeek-V3 风格" | 每个模块以上一深度的隐藏状态加下一词元嵌入为条件；保留因果链 |
| 共享输出头 | "复用主头" | MTP 模块调用主模型的 LM 头，而非单独的输出投影 |
| 共享嵌入 | "复用主表" | 到处使用相同的词汇嵌入表；无重复参数 |
| 投影矩阵 M_k | "组合隐藏+下一词元" | 将上一隐藏状态和目标词元嵌入折叠到下一深度输入的 `h x 2h` 线性层 |
| 联合损失 L_MTP | "平均额外损失" | 每深度交叉熵损失的算术平均，乘以 `lambda` 缩放 |
| 深度1接受率 | "MTP 草稿正确的频率" | D=1 MTP 模块的 top-1 预测与主模型 top-1 预测一致的比率；DeepSeek-V3 上超过 80% |
| Lambda 权重 | "额外损失重要性" | 每深度缩放因子；DeepSeek-V3 训练开始时为 0.3，后期为 0.1 |
