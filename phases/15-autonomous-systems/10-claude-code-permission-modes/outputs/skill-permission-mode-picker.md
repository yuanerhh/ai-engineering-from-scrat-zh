给定一项拟议的 Claude Code 任务，选择权限模式、设定预算，并在允许智能体开始前指定所需的最低隔离措施。

产出内容：

1. **任务概要。** 一句话描述任务内容，一句话描述出错时的爆炸半径。
2. **模式建议。** 选择以下之一：`plan`、`default`、`acceptEdits`、`acceptExec`、`autoMode`、`yolo`、`bypassPermissions`。引用爆炸半径用一句话说明理由。
3. **预算数字。** 为 `max_turns`、`max_budget_usd` 和任何每工具上限设定具体值。对于超过一小时的无人值守运行，指定不超过你愿意为无法回滚的人为错误付出的金额的美元上限。
4. **隔离要求。** 文件系统范围（仅限项目目录、临时目录、临时容器）。网络策略（无出口流量、仅白名单、完全开放）。凭据面（无凭据、有范围令牌、宽泛令牌）。对于 `bypassPermissions` 或 `yolo`，运行必须在临时容器内进行，且不得挂载生产凭据。
5. **轨迹审计计划。** 运行后人工如何审查轨迹？`autoMode`、`yolo` 以及超过 30 分钟时间跨度的任务必须有此计划。

硬性拒绝：
- 针对有未提交变更仓库使用 `bypassPermissions`。
- 没有预算上限的 `autoMode`。
- 在环境中有宽泛凭据（AWS、GCP、具有 repo 范围的 GitHub PAT）时使用高于 `acceptEdits` 的任何模式。
- 超过一小时且未安排轨迹审计的无人值守运行。
- 声称自动模式分类器单独就足以应对新任务分布的说法。

拒绝规则：
- 如果用户无法说明失败的爆炸半径，拒绝并要求在开始前提供明确的最坏情况描述。
- 如果用户在可触及生产数据库凭据的工作区中请求 `autoMode`，拒绝并要求先使用有范围凭据或临时容器。
- 如果拟议的预算上限超过用户愿意在一次糟糕运行中损失的金额，拒绝并要求降低上限。

输出格式：

返回一页运行卡，包含：
- **任务摘要**（一句话）
- **爆炸半径**（一句话，最坏情况）
- **模式**（明确指定）
- **预算**（`max_turns`、`max_budget_usd`、每工具上限）
- **隔离**（文件系统范围、网络策略、凭据面）
- **审计计划**（谁审查轨迹、何时、按照什么标准）
