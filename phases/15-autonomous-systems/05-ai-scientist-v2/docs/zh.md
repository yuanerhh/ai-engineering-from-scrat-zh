# AI Scientist v2——研讨会级别的自主研究

> Sakana 的 AI Scientist v2（Yamada 等人，arXiv:2504.08066）运行完整的研究循环：假设、代码、实验、图表、论文撰写、投稿。这是第一个由生成的论文通过 ICLR 2025 研讨会同行评审的系统。独立评估（Beel 等人）发现 42% 的实验因编码错误而失败，文献综述经常将已有概念错误标记为新颖。Sakana 自己的文档警告说该代码库执行 LLM 编写的代码，并建议使用 Docker 隔离。这幅图景的两面都是本课的重点。

**类型：** 学习
**编程语言：** Python（标准库，研究循环状态机玩具程序）
**前置知识：** Phase 15 · 03（AlphaEvolve）、Phase 15 · 04（DGM）
**预计时间：** 约 60 分钟

## 问题背景

研究是一项开放式任务。与 AlphaEvolve 的算法搜索或 DGM 以基准为界的自我修改不同，研究结果没有机器可验证的正确性标准。论文由审稿人而非单元测试来评判。这使得循环更难闭合——但如果闭合了则更有价值，因为研究是复合进步所在之处。

AI Scientist v1（Sakana，2024 年）通过从人工编写的模板开始闭合了循环。LLM 在固定脚手架中填写实验。AI Scientist v2（Yamada 等人，2025 年）通过使用带有视觉语言模型批评循环的智能体树搜索消除了模板要求。该系统生成想法、实现实验、制作图表、撰写论文，并根据审稿人反馈进行迭代。

同行评审结论：一篇 v2 生成的论文在 ICLR 2025 研讨会上被接受（附披露声明）。独立评估结论：该系统远非可靠。两者都是真实的。

## 核心概念

### 架构

1. **想法生成。** LLM 根据主题和先前文献提出研究想法。v1 使用模板；v2 在假设空间上使用智能体搜索。
2. **新颖性检查。** 文献检索步骤检查想法是否已发表。这是 Beel 等人的评估发现错误标记的步骤——已有方法经常被归类为新颖。
3. **实验计划。** 智能体起草实验方案并编写代码。
4. **执行。** 代码在沙盒中运行。失败被反馈到重试循环中。在 Beel 等人的测量中，42% 的实验在此阶段因编码错误而失败。
5. **图表生成。** 视觉语言模型读取生成的图表并重写以提高清晰度。这是 v2 的关键技术添加。
6. **撰写。** LLM 起草论文，与内部审稿人迭代。
7. **可选：投稿。** 论文提交到会议。

### 研讨会接受结果意味着什么

一篇 v2 生成的论文通过了 ICLR 2025 研讨会的同行评审。作者向程序委员会披露了论文的来源。这次接受是一个数据点；这不是声称该系统"能做研究"的许可。

重要背景：研讨会论文的门槛低于主会议论文。同行评审存在噪声；任何给定日期都有一小部分投稿被接受。一次成功是概念验证，不是可靠性声明。2026 年的 Nature 论文记录了端到端循环，并由人类研究人员共同撰写；这不是"系统写了一篇 Nature 论文"。

### 独立评估发现了什么

Beel 等人（arXiv:2502.14297）进行了外部评估。主要发现：

- **实验失败。** 42% 的实验因编码错误（错误的导入、形状不匹配、未定义的变量）而失败。重试循环捕获了一些，但不是全部。
- **新颖性错误标记。** 文献检索步骤经常将已有概念标记为新颖。这是研究中幻觉的等价物。
- **呈现质量差距。** 视觉语言图表批评产生了发表级别的视觉效果，掩盖了底层实验的弱点。

最后一个发现对本阶段最重要。一个产生令人信服的输出但没有进行令人信服研究的系统，比明显失败的系统更危险，而不是更安全。评估必须触及底层声明，而不是停留在图表上。

### 沙盒逃逸顾虑

Sakana 自己的仓库 README 警告：

> 由于该软件执行 LLM 生成的代码，我们无法保证安全性。存在危险包、不受控制的网络访问和产生意外进程的风险。使用风险自担，考虑使用 Docker 隔离。

这是自主性在未验证领域中的操作形态。LLM 编写代码；代码运行；代码可以做进程被允许做的任何事情。没有硬限制文件系统、网络和进程操作的沙盒，任何自主研究智能体都可以窃取数据、消耗算力或重写自身。

AlphaEvolve 的沙盒情况更简单，因为其评估器很严格。AI Scientist v2 的循环以开放式目标运行开放式代码。这就是为什么它需要更强的隔离（Docker 是最低要求；seccomp / gVisor 更好）以及在每篇投稿离开系统之前进行人工审查。

### v2 在前沿技术栈中的位置

| 系统 | 目标 | 输出类型 | 评估器 | 已知故障 |
|------|------|---------|--------|---------|
| AlphaEvolve | 算法 | 代码 | 单元 + 基准 | 受评估器严谨性限制 |
| DGM | 智能体脚手架 | 代码 | SWE-bench | 奖励黑客攻击 |
| AI Scientist v2 | 研究论文 | 文本 + 代码 + 图表 | 同行评审（弱） | 实验失败、错误标记、润色掩盖弱点 |

v2 三者中自动评估器最弱、输出面最宽、到公共产物的路径最短。操作控制（沙盒、审查、披露）承担了大部分安全工作。

## 动手实践

`code/main.py` 将 v2 循环模拟为状态机：想法 → 新颖性检查 → 实验 → 图表 → 撰写 → 审查 → 接受或迭代。每个状态都有从 Beel 等人发现中提取的可配置失败概率。运行 N 个循环的模拟器并计数：

- 有多少个想法到达投稿阶段。
- 有多少篇投稿会有润色后的论文所隐藏的关键实验缺陷。
- 重试预算如何在质量与产出之间进行权衡。

## 产出技能

`outputs/skill-ai-scientist-sandbox-review.md` 是研究循环智能体产生的任何内容在离开沙盒之前的两关审查清单。

## 练习

1. 使用默认参数运行 `code/main.py`。什么比例的循环运行产生"干净"的论文？什么比例产生带有图表批评润色过的实验失败缺陷的论文？

2. 默认值已经使用了 Beel 等人的 42% / 25%。使用 `--experiment-failure 0.20 --novelty-mislabel 0.10` 重新运行，然后使用 `--experiment-failure 0.60 --novelty-mislabel 0.40`。经过润色但有缺陷的比例在两次运行之间如何变化？

3. 阅读 Sakana AI Scientist v2 仓库 README 关于沙盒要求的内容。为多天自主运行命名两个额外的限制（除 Docker 外）。

4. 阅读 Beel 等人第 4 节关于呈现质量差距的内容。设计一个额外的评估器，能够捕获看起来精良但实验上有缺陷的论文。

5. 提出一个研究智能体输出的人工审查协议，比"博士读每篇论文"扩展性更好。找出瓶颈并围绕它设计。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| AI Scientist v1 | "Sakana 的模板化研究智能体" | 在固定脚手架中填写实验 |
| AI Scientist v2 | "无模板研究智能体" | 带 VLM 图表批评的智能体树搜索 |
| 智能体树搜索 | "分支研究智能体" | 并行扩展多个实验计划；由内部评判者修剪 |

| 润色掩盖 | "精美论文，破碎研究" | 呈现质量超过实验质量；隐藏弱点 |
| 沙盒逃逸 | "LLM 代码突破限制" | 智能体执行的代码做了循环设计者未预期的事情 |
